# cMind для брокеров cTrader 🏦

Вы управляете брокерской конторой cTrader. Ваши клиенты уже могут торговать — но и клиенты каждого другого брокера могут. **cMind позволяет вам дать своим трейдерам полную платформу операций по торговле с ИИ, брендированную как вашу**, поэтому они создают, выполняют бэктесты, запускают, копируют и мониторят стратегии внутри *вашей* экосистемы вместо дрейфа к третьему инструменту. Это липкие клиенты, больше объема и реальное преимущество перед брокерами, предлагающими только терминал.

:::tip TL;DR
Запустите белый ярлык cMind для своих клиентов. Ограничьте счета **вашей** брокерской конторе, включите ИИ и копирование торговли и поставляйте это под вашим брендом. → [Белый ярлык для бизнеса](./white-label-for-business.md)
:::

## Преимущество, которое вы получаете над другими брокерами

- **Дифференцируйте с помощью инструментов, не только спредов.** Предоставляйте клиентам генерацию ИИ cBot, бэктестирование на управляемом кластере, копирование торговли и испытания проприетарной компании — возможности, которые большинство брокеров просто не предлагают.
- **Держите клиентов в своей экосистеме.** Когда трейдеры создают и запускают свои стратегии внутри вашей брендированной платформы, они остаются. Удержание — вся игра.
- **Под вашим брендом, на вашем доменом.** Имя, логотип, цвета, favicon, даже устанавливаемое приложение для телефона — всё ваше. Никто не видит «cMind». → [Функция белого ярлыка](./features/white-label.md)

## Служите только вашим счетам (список допускаемых брокеров)

Запускаете белый ярлык для *ваших* клиентов? Ограничьте, какие счета трейдеров брокеров пользователи могут добавить, поэтому ваше развертывание служит только вашей книге:

```json
{
  "App": {
    "Accounts": {
      "AllowedBrokers": ["Имя вашей брокерской конторы"]
    }
  }
}
```

Когда список отфильтрован, cMind проверяет каждый счет, который пользователь пытается добавить — как через cTrader Open API, так и через ручную учетную запись cID (проверяется путем чтения реального имени брокера счета) — и отклоняет любой счет, который не в вашем списке. Оставьте пусто и каждый брокер разрешен (по умолчанию). Смотрите [документ функции белого ярлыка](./features/white-label.md#список-допускаемых-брокеров) для полной механики.

## Поставляйте одно приложение Open API для всех ваших пользователей

Пропустите забот на пользователя: предоставьте **одно приложение cTrader Open API** и каждый клиент авторизирует свои счета через него — клиент никогда не регистрирует свой собственный. Зарегистрируйте один URL переадресации, сбросьте учетные данные в config или в параметрах владельца, и общий режим включается для всех. Договорились о более высоком лимите сообщений cTrader? Отрегулируйте **пределы скорости клиента для каждого типа сообщений** (или отключите скоростное ограничение). → [Приложение общего доступа Open API и ограничения скорости](./features/open-api-shared-app.md)

## Новые способы монетизации

- **ИИ, без трения для клиентов.** Предоставьте ключ поставщика ИИ по умолчанию на уровне развертывания и каждый клиент мгновенно получает функции ИИ — нет подписки в другом месте. Наценка или включение в премиум-уровни. Клиенты всё ещё могут принести свой ключ. → [Функция ИИ](./features/ai.md)
- **Вызовы проприетарной компании.** Запустите испытания финансируемого трейдера с отслеживанием капитала в реальном времени и применяемыми правилами и взимайте за записи. → [Правила проприетарной компании](./features/prop-firm.md)
- **Бизнес копирования торговли.** Комиссии за производительность и рынок поставщиков превращают копирование торговли в доход. → [Комиссии за производительность](./features/copy-performance-fees.md) · [Рынок поставщиков](./features/copy-provider-marketplace.md)
- **Уровни функций.** Решайте, какие возможности видит каждый сегмент клиентов с помощью [переключателей функций](./features/feature-toggles.md).

## Регулируется, проверяется, мультитенантно

- **[Соответствие](./features/compliance.md)** логи дают вам аудиторский след, который запросит ваш регулятор.
- **[Двухфакторная аутентификация](./features/two-factor-auth.md)** может быть сделана обязательной для развертывания.
- **Брендирование для каждого клиента** — запустите отдельный брендированный экземпляр для каждого сегмента, управляемый из вашей собственной плоскости управления. → [Мультитенантное брендирование](./white-label-for-business.md#мультитенантное-брендирование-для-каждого-клиента)

## Как начать

1. Прочитайте [Белый ярлык для бизнеса](./white-label-for-business.md) для 60-секундного перебрендирования.
2. Установите `App:Accounts:AllowedBrokers` вашей брокерской конторе и выберите ваш [набор функций](./features/feature-toggles.md).
3. [Развертайте](./deployment/cloud.md) — Docker, Kubernetes, Azure или AWS.

Не хотите самостоятельно запускать инфраструктуру? Хостинг-провайдер может управлять управляемым cMind для вас — укажите им на [Для облачных и VPS провайдеров](./for-cloud-providers.md).

## Формируйте дорожную карту

cMind открыт. Брокеры, которые его используют, получают непропорциональное влияние на то, куда он идет — запрашивают интеграции и элементы управления, которые вам нужны, и внесите их через [руководство для участников](./contributing.md).
