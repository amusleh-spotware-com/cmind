# Strategy Health & Alpha Decay

Каждое преимущество деградирует — исследования однозначно показывают, что half-life количественной стратегии сократилась с лет до месяцев, поэтому *адаптация важнее открытия*. Монитор Strategy Health сообщает вам, по истории доходностей стратегии, осталось ли ещё преимущество.

Открыть **cBots → Strategy Health** (`/quant/health`).

## Что делает

По серии доходностей (или кривой капитала, от старых к новым), вычисляет:

- разбивает историю на **более раннюю** и **недавнюю** половины и сравнивает их коэффициенты Шарпа;
- запускает **CUSUM change-point** сканирование для нахождения наблюдения, где среднее наиболее явно сдвинулось (точка изменения режима), сообщается только если отклонение статистически значимо;
- возвращает вердикт:

| Вердикт | Значение |
|---|---|
| **Healthy** | Недавняя производительность соответствует (или лучше) более раннему результату. |
| **Degrading** | Недавний Шарп существенно слабее более раннего — следите внимательно. |
| **Decayed** | Преимущество фактически исчезло в недавнем окне — рассмотрите паузу. |
| **Unknown** | Недостаточно истории для оценки. |

```http
POST /api/quant/health
{ "returns": [...] }   // или { "equity": [...] }
```

## Почему это надёжно

Это чистый, детерминированный доменный код (`Core.Health`) без инфраструктурных зависимостей и внешних вызовов — модульные тесты покрывают случаи decayed, degrading, healthy и слишком короткой истории, а также локализацию change-point. Это мануальный спутник всегда включённых проверок здоровья, которые поддерживают автономных агентов: те же статистика驱动ят автоматический выключатель, снижающий риск живой стратегии, чьё преимущество угасает.
