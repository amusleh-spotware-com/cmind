# 소유자 설정의 화이트레이블 옵션

배포가 구성(`appsettings`/env)을 통해 설정할 수 있는 모든 화이트레이블 옵션은 재배치 없이 **런타임에 앱 소유자가 설정**할 수도 있으며, **Settings → Deployment**에서 가능합니다. 소유자 오버라이드가 구성 **보다 우선**합니다; 지우면 옵션이 배포의 구성된(또는 기본 제공 기본값) 값으로 돌아갑니다.

이것은 화이트레이블 *배포*가 제품을 구성하는 방식을 반영합니다 — 동일한 노브, 동일한 효과 — 그래서 운영자가 라이브로 브랜딩, 게이트 및 정책을 조정하고 즉시 결과를 확인할 수 있습니다.

## 위치

- **UI:** 설정 대화상자의 소유자 전용 **Deployment** 섹션 및 딥 링크 가능 페이지 **`/settings/deployment`**. 옵션은 **카테고리별 탭**으로 그룹화됩니다(Branding, Theme, Features, Registration, Accounts, Email, AI, Open API, Prop firm), 모바일 우선, 데스크톱에서는 창화된 대화상자, 핸드폰에서는 전체 화면.
- **API:** `/api/whitelabel`(소유자 전용, 기능 게이트 아님):
  - `GET /api/whitelabel` — 각 옵션과 유효 값, 출처(`Config` / `Owner` / `Default`) 및 오버라이드가 설정되었는지 여부. **비밀번호는 마스킹됩니다**(값이 절대 반환되지 않음).
  - `PUT /api/whitelabel/{key}` `{ "value": "…" }` — 오버라이드 설정(옵션 종류별 검증). **비밀**에서 빈 값은 기존 비밀을 유지합니다.
  - `DELETE /api/whitelabel/{key}` — 하나 오버라이드 지우기(구성으로 되돌리기).
  - `POST /api/whitelabel/reset` — 모든 오버라이드 지우기(배포를 순수 구성으로 되돌리기).

## 오버라이드가 적용되는 방식

소유자 오버라이드는 암호화 필요한 곳은 암호화하여 `AppSetting` 행으로 저장되고 바운드 `AppOptions`의 위에 레이어됩니다. 모든 소비자가 이미 해당 모니터를 통해 옵션을 읽으므로 오버라이드는 앱 전체에 **라이브로 적용**됩니다 — 테마, 페이지 제목, MFA 게이트, AI 공급자 게이트, 중개，允许list, 등록 정책, 이메일 전송 설정 등이 다음 읽기 시 업데이트됩니다(테마/브랜딩은 즉시 다시 렌더링). 데이터베이스가 잠시 사용할 수 없는 경우 레이어는 구성된 기준값으로 **열림**으로 실패하므로 오버라이드 읽기가 앱을 절대 중단시킬 수 없습니다.

**기능 플래그**는 동일한 표면의 일부이지만 기존 기능 오버라이드 저장소(`IFeatureGate`)를 통해 지속되므로 Features 탭과 독립 실행형 기능 토글이 절대 발산하지 않습니다.

**비밀번호**(SMTP 비밀번호, CAPTCHA 비밀번호, 프로비저닝 비밀번호)는 저장에서 암호화됩니다(`ISecretProtector`, 목적 `whitelabel.secret`), UI에서 쓰기 전용이며 API에서 절대 반환되지 않습니다.

## 위임 옵션

**공유 Open API 애플리케이션** 자격 증명 및 **퍼메시지 유형별 속도 제한**은 **Open API** 설정 섹션에서 관리됩니다(카피트레이딩 / Open API 문서 참조). 동기화 보장이 적용되는 한 중복을 피하기 위해 읽기 전용으로 표시되며 해당 카탈로그에 covered으로 카운트됩니다.

## 항상 동기화 (강제)

구성에 새로운 화이트레이블 옵션을 추가 **해야 합니다** 같은 커밋에서 소유자 설정에 표면화합니다. 이것은 `WhiteLabelCatalogParityTests`에 의해 적용됩니다: 모든 화이트레이블 옵션-레코드 속성을 반영하고 속성이 `Core/WhiteLabel/WhiteLabelCatalog`에 등록되지 않았거나 이유와 함께 `IntentionallyExcluded`에 명시적으로 나열되지 않으면 빌드를 실패시킵니다. `CLAUDE.md`의 의무 10 참조.

## 참고

- **이메일 미구성으로 시작한 배포에서 SMTP을 활성화하면 재시작이 필요합니다**(발신자 유형이 시작 시 선택됨); 이미 구성된 발신자의 호스트/자격 증명은 라이브로 업데이트됩니다.
- 옵션 **레이블/설명**은 데이터로 표시되는 기술적 구성 노브 식별자입니다; 탭 레이블과 모든 대화 chrome은 완전히 지역화됩니다.
