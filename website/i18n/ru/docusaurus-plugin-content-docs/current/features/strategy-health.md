# Strategy Health & Alpha Decay

Каждое преимущество затухает — исследования ясно показывают, что период полураспада количественной стратегии сократился с лет до месяцев, поэтому *адаптация важнее открытия*. Монитор Strategy Health подскажет вам, основываясь на собственной истории доходности стратегии, остаётся ли преимущество.

Откройте **cBots → Strategy Health** (`/quant/health`).

## What it does

Имея ряд доходностей (или кривую собственного капитала, от старого к новому), он:

- разбивает историю на **ранний** и **недавний** периоды и сравнивает их коэффициенты Sharpe;
- выполняет **CUSUM change-point** сканирование, чтобы найти наблюдение, где среднее значение наиболее явно сместилось (разрыв режима), и сообщает только при статистически примечательном отклонении;
- возвращает вердикт:

| Вердикт | Значение |
|---|---|
| **Healthy** | Недавнее производство находится в соответствии с (или лучше, чем) история стратегии. |
| **Degrading** | Недавний Sharpe заметно слабее, чем ранняя запись — следите внимательно. |
| **Decayed** | Преимущество практически исчезло в последнем окне — рассмотрите паузу. |
| **Unknown** | Недостаточно истории для оценки. |

- **Прямо из прогона бэктеста — без копирования.** Каждый завершённый бэктест показывает иконку **Check strategy health** в строке списка **Backtest** и в представлении деталей экземпляра; один клик запускает монитор на сохранённой кривой собственного капитала прогона и показывает вердикт в диалоге. Иконка отключена до завершения бэктеста и создания отчёта, поэтому никогда не является мёртвым элементом управления. Под капотом это `POST /api/quant/health/backtest/{instanceId}`, который читает кривую собственного капитала сохранённого отчёта.

```http
POST /api/quant/health
{ "returns": [...] }   // или { "equity": [...] }
```

## Why it is reliable

Это чистый, детерминированный доменный код (`Core.Health`) без зависимостей инфраструктуры и внешних вызовов — модульно протестированный для случаев затухания, деградации, здоровья и слишком короткой истории, а также для локализации change-point. Это ручной компаньон к постоянно работающим проверкам здоровья, которые поддерживают автономные агенты: одна и та же статистика управляет выключателем, который снижает риск активной стратегии, чьё преимущество угасает.
