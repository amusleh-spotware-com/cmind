# 백테스트 무결성 랩

소매 플랫폼은 백테스트의 Sharpe 또는 순이익을 보여주고 거기서 멈춥니다. 기관은 원시 백테스트를 절대 신뢰하지 않습니다 — 결과가 **선택 편향 및 시도한 구성 수의修正**을 버틸지 묻습니다. 백테스트 무결성 랩이 cMind에 이를 가져옵니다. **결정론적 수학** (AI 없음, 외부 호출 없음)이므로 판정이 재현 가능하고 모든 숫자를 설명할 수 있습니다.

**cBots → Integrity** (`/quant/integrity`)에서 엽니다.

## 무엇을 계산하는가

수익률 시리즈(또는 Equity/잔액 곡선)와 이를 얻기 위해 시도한 파라미터 세트 수를 주면 분석기가 보고합니다:

- **Sharpe 비율** — 기간별 및 연간 (제곱근-시간).
- **Probabilistic Sharpe Ratio (PSR)** — 추적 기록 길이, 왜도 및 첨도를 고려하여 *실제* Sharpe가 벤치마크를 능가하는 확신 (Bailey & López de Prado, 2012). 짧거나 fat-tailed 기록은 그것을 낮춥니다.
- **Deflated Sharpe Ratio (DSR)** — **deflated 벤치마크**에 대해 측정된 PSR: *N개의 무작위 시험 중 최고*에서 기대할 Sharpe (False Strategy Theorem). 시도한 구성이 많을수록 기준이 높아집니다 — 이것이 과적합을 잡는 것입니다.
- **평균 수익의 t-통계**. Harvey, Liu & Zhu를 따르면 진짜 에지는 **t ≥ 3.0**을クリア해야 합니다, 교과서 2.0이 아니라.
- **수익률의 왜도/첨도**, PSR/DSR修正에 공급됩니다.

## 판정

| 판정 | 의미 | 규칙 |
|---|---|---|
| **Robust** | 에지가 실행한 시험을 버팁니다. | DSR ≥ 95% **그리고** PSR ≥ 95% **그리고** \|t\| ≥ 3.0 |
| **Fragile** | 통계적으로 살아 있지만 확신하기 어려움 — 이것만으로 규모를 늘리지 마세요. | 둘 사이 |
| **Overfit** | 가장 가능성 있는 것은 선택 편향의 산물이 아니라 진짜 에지입니다. | DSR < 90% |

모든 결과에는 일반 영어 근거가携带되므로 "왜"가 절대 숨겨지지 않습니다.

## 백테스트 과적확 확률 (시험 전반)

시험 *카운트*를 제공하는 것은 좋습니다; 시도한 **각 구성의 실제 샘플 외 시리즈**를 제공하는 것이 더 좋습니다. 선택적 **시험 그리드**에 붙여넣고 (한 줄당 하나의 시리즈) cMind가 **Combinatorially-Symmetric Cross-Validation** (Bailey, Borwein, López de Prado & Zhu, 2015)를 실행합니다: 관찰을 그룹으로 분할하고, 샘플 내 최고 구성을 선택하고 샘플 외 하단의 절반에 착지하는지 확인하는 모든 방식으로 각 분할에 대해. **백테스트 과적확 확률 (PBO)**은 승자가 일반화에 실패한 분할의 비율입니다. PBO가 0에 가까우면 최상의 구성이 진짜로 최상입니다; PBO가 0.5 이상이면 선택 프로세스가 노이즈를 선택하고 있습니다 — 판정이 어떻게 보여도 **과적합**이 됩니다.

```http
POST /api/quant/ppo
{ "trials": [[...], [...], ...] }
```

네이티브 cTrader Console 옵티마이저가 착륙하면 cMind는 여기에 전체 시험 표면을 자동으로 공급합니다.

## 시험 — 중요한 숫자

`Trials`는 **이것을 선택하기 전에 테스트한 파라미터 세트 수**입니다. 하나의 전략을 테스트하는 것과 10,000개를 테스트하고 최고를 유지하는 것은 완전히 다른 것입니다: 두 번째는 우연히 높은 샘플 내 Sharpe를製造합니다. 정직한 시험 수를 제공하는 것이 요점입니다 — 이것이 defilation을 높이고 "훌륭한" 백테스트를 **과적합**으로 이동시킬 수 있습니다. 네이티브 cTrader Console 옵티마이저가 착륙하면 cMind는 스위프의 실제 그리드 크기를 자동으로 공급합니다.

## 입력

- **주기 수익** — 기간당 하나의 숫자 (예: `0.01` = +1%). 최소 두 개.
- **Equity / 잔액 곡선** — cMind가 연속 단순 수익을 파생합니다.
- 또는 완료된 백테스트에서 직접 실행: `POST /api/quant/integrity/backtest/{instanceId}`는 저장된 보고서의 Equity 곡선을 읽습니다.

## API

```http
POST /api/quant/integrity
{ "returns": [0.006, 0.004, 0.006, ...], "trials": 250 }
```

판정, 모든 메트릭 및 근거를 반환합니다. `POST /api/quant/integrity/backtest/{id}`는 소유한 완료된 백테스트에서 동일한 분석을 실행합니다.

## 왜 신뢰할 수 있는가

통계는 게시된 공식에 대해 golden-vector 단위 테스트로 고정된 인프라 의존성 없이 도메인 핵심 (`Core.Quant`)의 순수 함수입니다. 정규 CDF/역은 닫힌 형식 근사치 (Abramowitz-Stegun / Acklam)이므로 동일한 입력이 항상 동일한 판정을 산출합니다.
