# ダッシュボード

サインイン時に表示される最初の画面であり、実際に1日中開いておくことになるページです。
ランディングページ（`/`、`Components/Pages/Index.razor`）は、サインインしたユーザーの
cBot実行、バックテスト、リソース、（管理者向け）ノードクラスタ全体のアクティビティのための
**ライブ、モバイルファーストコマンドセンター**です。
自動更新され、phoneでも美しく表示され、F5を押さえる必要が永远不会。

## 表示内容

優先度順（phoneではすべてのブロックが全幅スタックアイテム、タブレ/desktopではレスポンシブグリッド）：

1. **ヘッダー** — タイトル、ライブインジケーター（実際のpulsesするドット; `prefers-reduced-motion`下では静的）、
   最終更新時刻、**期間トグル**（`1H · 24H · 7D · 30D`）— KPIとチャートを驱动。
2. **ヒーローKPI** — 4つの一剑arellaカード、それぞれ大きな数字 + インラインSVGスパークライン +
   （意味のある場所では）**前期間との差分**：
   - **Active now** — 現在開始/実行中の実行 + バックテスト。
   - **Success rate** — 期間中の完了 ÷（完了 + 失敗）; 差分はパーセンテージポイント。
   - **Completed** — この期間の終了した実行/バックテスト; 前期間との差分。
   - **Failed** — この期間の失敗; 差分（少ないほど良ため、減少は緑色で表示）。
3. **アクティビティチャート** — 開始/完了/失敗の各時間バケットのApexChartsエリアタイムライン。
4. **インスタンスステータスリング** — 実行/バックテスト/保留/完了/失敗のドーナツ、中心に合計。
5. **バックテスト** — 3タイルスナップショット（実行中/完了/失敗）、クリックで`/backtest`へ。
6. **コピー取引** — ライブステータスドット、宛先数、実行中プロファイルの**Live**バッジ付き
   コピー取引プロファイル; クリックで`/copy-trading`へ。
7. **AIエージェント** — 実行状態（アーキタイプ · ステータス）と最終アクション時刻付きペルソナ駆動取引エージェント;
   クリックで`/agent-studio`へ。
8. **ライブアクティビティフィード** — 最新の20イベント（最新順）、ステータスカラーのドットと
   相対タイムスタンプ付き。
9. **クラスタヘルス**（管理者のみ） — アクティブvs合計ノードと容量使用量ゲージ。
10. **リソースタイル** — cBot、取引アカウント、cTrader ID、MCPキー（各ページにクリックthrough）。

## ダッシュボードのカスタマイズ

上記のすべてのブロックは**あなたがコントロールするウィジェット**です。
ヘッダーの右上にある**Customize**を押してダイアログを開き、ウィジェートの**表示/非表示**を切り替え、
上下矢印で**並べ替え**ができます。**デフォルトにリセット**するとカタログ順序が復元されます。
あなたの選択は**サーバー側でユーザーごとに永続化**されるため、ブラウザやデバイス間をまたいで追跡 —
このタブだけでなく。

- 機能ゲートと管理者のみのウィジェット（コピー取引、AIエージェント、クラスタヘルス）は、
  デプロイ/ロールがそれらを使用できる場合にのみダイアログに表示されます。
- ウィジェットカタログは`Core/Dashboard/DashboardWidgets.cs`の単一の情報源;
  表示（ラベル + アイコン + 可用性）は`Components/Dashboard/DashboardWidgetMeta.cs`にあります。

## ライブ状態を維持する方法

ページは10秒ごとに`GET /api/dashboard/overview?period=<1h|24h|7d|30d>`をポーリングし、
ウィジェットをインプレースで再描画 — 手動再読み込みなし。
一時的なフェッチ失敗は飲み込まれ、次のティックでリトライ; disposeでループはクリーンに停止。
最初の読み込みではスケルトンが表示; 持続的な失敗は**Retry**付きの錯誤カードを表示;
データのないユーザーはゼロのKPIと空状態copy能看到。

## バックエンド

- `Endpoints/DashboardEndpoints.cs`が`/overview`をマッピング（古いスカラー`/stats`も維持）。
  ユーザーごと、`ICurrentUser`で管理者にゲート; クロックは`TimeProvider`から。
  `GET/PUT /api/dashboard/layout`もマッピング — ユーザーのウィジェットレイアウト、
  ページ開始時に読み込まれ、Customizeダイアログから保存。
- **レイアウト永続化**は`UserDashboard`集約（`Core/Dashboard/UserDashboard.cs`）：
  ユーザーごとに1つのボード（`UserId`で一意）、順序付きウィジェット設定リスト
  （visible + order）を`jsonb`カラムとして所有。
  順序付きリストは`Apply` / `Reset`を通じてのみ突然変異し、
  すべてのキーを`DashboardWidgets`カタログに対して検証し、コレクションを完全で重複なしのまま維持。
  不明なキーは`DomainException` → `400`で拒否されます。
- `Endpoints/DashboardQuery.cs`が複合`DashboardOverview`読み取りモデルを構築：
  オールタイムステータススナップショット（グループカウント）、
   материализован onceのウィンドウ化されたインスタンスセット、
  リソース/ノードカウント。
  インスタンスステータスと终端タイムスタンプはTPHサブタイプに存在（カラムではなく）、
  そのため行は共有`InstanceEndpoints.GetStartedAt/GetStoppedAt` помощникによってメモリで読み取られます。
  イベント時刻 = `stopped ?? started ?? created`。
- `Endpoints/DashboardModels.cs`がDTO保持、期間→（ウィンドウ、バケットカウント）プラン、
  `DashboardMath` — 純粋で決定論的なバケッティング + KPI/差分数学（I/Oなし、`now`が渡される）。

KPI差分は現在のウィンドウと即座に先行するウィンドウを比較（そのためにクエリはダブルウィンドウをフェッチ）。
**ライブアカウントP&Lフィードはありません** — プラットフォームはバックテストとprop-firmトラッキングの
株式のみを持つため、ダッシュボードは意図的に*運用*(アクティビティ、スループット、成功率)、
而不是经纪公司残고 ticker。

## デザインとトークン

すべての颜色はデザイートークン（`var(--app-success|-warning|-error|-info|-primary|-text*)`）から来て、
因此ホワイトラベルパレットが無料でフロー — チャート含む、
そのシリーズ颜色は実行時に`window.appReadTokens`から解決されたトークンを読み取ります
（SVGはCSS変数を直接的消费できない）。ダッシュボードのどこにもハードコードされたhexなし。
[../ui-guidelines.md](../ui-guidelines.md)を参照。

## 「Powered by cMind」リンク

ダッシュボードにはこのドキュメントサイトヘの小さな tastefulな**「Powered by cMind」**リンクが表示されます。
これは**デフォルトで表示** — 私たちはプロジェクトに誇りを持っており、
他のトレーダーがそれを見つけるのに役立ちます — が完全にあなたの判断次第です。
完全にホワイトラベルされたインスタンスを実行するリセラは`App:Branding:ShowSiteLink`を`false`に
flipすと消えます。
[ホワイトラベルブランディング](./white-label.md#powered-by-link)を参照。

## テスト

- **ユニットスタイル**（`tests/IntegrationTests/DashboardMathTests.cs`）—
  バケティング、成功率、前期間差分、期間解析、空/境界（`now`のイベント、ゼロ除算ガード）。
- **ユニット**（`tests/UnitTests/Dashboard/UserDashboardTests.cs`）—
  `UserDashboard`集約：デフォルトシード、order/visibility適用、欠落のappended、重複collapse、
  不明なキー拒否、リセット。
- **統合**（`tests/IntegrationTests/DashboardQueryTests.cs`、`DashboardLayoutTests.cs`）—
  実際のPostgresに対する読み取りモデル（ステータス/KPI/アクティビティ/リソース、管理者ノードヘルス、
  空ユーザーパス）、新しいバックテスト/コピープロファイル/エージェントセクション、
  レイアウト**ラウンドトリップ**（カスタムレイアウト保存 → 再読み込み → order + visibilityが持続）。
- **E2E**（`tests/E2ETests/DashboardTests.cs`、`DashboardCustomizeTests.cs`）—
  desktop + mobile：KPIカード、チャート、リング、フィードが描画; 期間トグルがアクティブ期間を切り替え、
  再読み込み; KPIが`/run`にドリル-through; **ウィジェットを非表示にすると再読み込み後に持続**、
  **リセット**が復元、Customizeダイアログが電話で動作（水平オーバーフローなし）。
  `/`は`PageSmokeTests`、`MobileLayoutTests`（シェル + 非オーバーフロー）、
  `MobileJourneyTests`にも含まれる。
