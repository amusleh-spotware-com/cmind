# cMind за cTrader берзе 🏦

Водите cTrader берзу. Ваши клијенти већ могу трговати — али могу и клијенти сваке друге берзе. **cMind вам омогућава да своје трговце опремите са комплетном AI-напајаном платформом за трговачке операције, означене као ваша сопствена**, тако да граде, тестирају, покрећу, копирају и надзиру стратегије унутар *вашег* екосистема умјесто да одлазе према алатима трећих страна. То значи траженије клијенте, већу запремину и праву предност над берзама које нуде ништа осим терминала.

:::tip[РЕЗИМЕ]
Покрените белу ознаку cMind-а за своје клијенте. Ограничите налоге на **вашу** берзу, укључите AI и копирање трговања, и испоручите је под вашом марком. → [Беларабежена за посао](./white-label-for-business.md)
:::

## Предност коју имате над другим берзама

- **Разликујте се по алатима, не само по спредовима.** Дајте клијентима генерисање AI cBot-а, тестирање на управљаном кластеру, копирање трговања и изазове проп-фирме — способности које већина берза једноставно не нуди.
- **Задржите клијенте у вашем екосистему.** Када трговци граде и покрећу своје стратегије унутар ваше означене платформе, остају. Задржавање је цела игра.
- **Под вашом марком, на вашем домену.** Име, логотип, боје, фавикон, чак и инсталирива телефонска апликација — све ваше. Нико не види „cMind." → [Беларабежена функција](./features/white-label.md)

## Послужите само ваше налоге (листа дозвољених берзи)

Водите белу ознаку за *ваше* клијенте? Ограничите које налоге берзи корисници могу да додају како би ваша примена служила само вашој књизи:

```json
{
  "App": {
    "Accounts": {
      "AllowedBrokers": ["Your Brokerage Name"]
    }
  }
}
```

Када је листа дозвољених постављена, cMind проверава сваки налог који корисник покуша да дода — како преко cTrader Open API тако и преко ручног cID пријављивања (верификовано читањем правог имена берзе налога) — и одбија сваки налог који није на вашој листи. Оставите празно и свака берза је дозвољена (подразумевано). Видите [док белабежене функције](./features/white-label.md#broker-allowlist) за комплетну механику.

## Испоручите једну Open API апликацију за све своје кориснике

Прескочите мукотрпну перерту по кориснику: обезбедите **једну cTrader Open API апликацију** и сваки клијент аутентификује своје налоге кроз њу — клијент никада не региструје своју. Региструјте једну URL редиректа, пустите верификације у конфигурацији или подешавањима власника, и подељени режим се укључује за све. Негоцирани виши cTrader лимит поруке? Подесите **по типу поруке лимите брзине рачунара клијента** (или деактивирајте паце). → [Подељена Open API апликација & лимити брзине](./features/open-api-shared-app.md)

## Нови начини за монетизацију

- **AI, без икаквих трења за клијенте.** Обезбедите подразумевану AI клавијатуру на нивоу примене и сваки клијент добиј AI функције одмах — без регистрације на другом месту. Обележите је, или је комбинујте у премијумске слојеве. Клијенти још увек могу донети своју клавијатуру. → [AI функција](./features/ai.md)
- **Проп-фирме изазови.** Покрените финансиране трговачке изазове са живом одрживањем капитала и наведеним правилима, и наплаћујте уводе. → [Правила проп-фирме](./features/prop-firm.md)
- **Посао копирања трговања.** Комисије за перформансе и тржиште добављача претварају копирање трговања у приход. → [Комисије за перформансе](./features/copy-performance-fees.md) ·
  [Тржиште добављача](./features/copy-provider-marketplace.md)
- **Слојеви функција.** Одлучите које способности сваки сегмент клијента види са
  [прекидачима функција](./features/feature-toggles.md).

## Регулирано, проверљиво, мултинајмерско

- **[Усклађеност](./features/compliance.md)** дневници вам дају записник за ревизију коју ће ваш регулатор питати.
- **[Двофакторска аутентификација](./features/two-factor-auth.md)** може бити направљена обавезном по примени.
- **Брендирање по клијенту** — водите одвојену означену инстанцу по сегменту, вођену из ваше сопствене контролне равни. → [Брендирање мултинајмерског](./white-label-for-business.md#multi-tenant-per-customer-branding)

## Како почети

1. Прочитајте [Беларабежена за посао](./white-label-for-business.md) за 60-секундну ребранд.
2. Поставите `App:Accounts:AllowedBrokers` на вашу берзу и одаберите ваш [скуп функција](./features/feature-toggles.md).
3. [Развоја](./deployment/cloud.md) то — Docker, Kubernetes, Azure или AWS.

Не желите да водите инфраструктуру сами? Добављач услуга хостинга може да управља управљаним cMind-ом за вас — упутите их на [За облак и VPS добављаче](./for-cloud-providers.md).
