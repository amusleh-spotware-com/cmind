# Build & backtest cBots

Build, run, backtest cTrader cBots (C# **и** Python, оба .NET) из in-browser Monaco
IDE, работает на официальном образе `ghcr.io/spotware/ctrader-console`.

## Build

- Страница **Builder** содержит Monaco editor; `CBotBuilder` компилирует проект с
  `dotnet build` **в одноразовом контейнере** (`AppOptions.BuildImage`, рабочая директория bind-mount
  в `/work`), поэтому недоверенные пользовательские MSBuild-таргеты не имеют доступа к файловой системе хоста. NuGet restore кэшируется
  между сборками через shared volume. Веб-хост требует доступ к Docker socket.
- C# + Python стартовые шаблоны живут в `src/Nodes/Builder/Templates/`.

## Run & backtest

- **Instances** = иерархия состояний TPH (`Run`/`Backtest` × `Pending`/`Scheduled`/`Starting`/
  `Running`/`Stopping`/`Stopped`/`Failed`). Переход заменяет сущность (смена id),
  id контейнера сохраняется.
- `NodeScheduler` выбирает наименее загруженный подходящий узел; `ContainerDispatcherFactory` маршрутизирует к
  удалённому HTTP-агенту узла или локальному Docker-диспетчеру.
- Completion poller'ы сверяют вышедшие контейнеры (бэктест-контейнеры выходят сами через
  `--exit-on-stop`); отчёт есть → completed (сохраняется `ReportJson`), отсутствует → failed.
- Логи работающего контейнера стримятся в браузер через SignalR; кривые капитала бэктестов парсятся из
  отчёта и чартятся.

## Заметки о CLI cTrader Console

Бэктестам нужен `--data-mode` (по умолчанию `m1`), даты как `dd/MM/yyyy HH:mm`, и
`params.cbotset` JSON positional arg; `run` отклоняет `--data-dir` (только для бэктеста). См.
`ContainerCommandHelpers`.

## Nodes & scale

Вместимость исполнения масштабируется добавлением агентов-узлов (саморегистрация + heartbeat). См.
[node discovery](../operations/node-discovery.md) и [scaling](../deployment/scaling.md).

## Запуск из редактора кода

Нажатие **Запустить** в редакторе кода открывает диалог вместо слепого, жёстко заданного запуска:

- **Торговый счёт** (обязательно) — счёт cTrader, к которому подключается cBot.
- **Набор параметров** (необязательно) — выберите существующий набор или оставьте пустым, чтобы запустить со **значениями параметров cBot по умолчанию**. Кнопка **+** рядом с селектором создаёт новый набор параметров прямо здесь (см. ниже) и выбирает его.
- **Символ / Таймфрейм** по умолчанию `EURUSD` / `h1` и их можно изменить; **Отмена** или **Запустить**.

При **Запуске** редактор сохраняет и собирает текущий исходный код, запускает экземпляр на выбранном счёте с выбранными параметрами, затем выводит логи контейнера в реальном времени. (Поток логов передаёт cookie аутентификации вошедшего пользователя в SignalR-хаб `/hubs/logs`, поэтому он подключается, а не завершается ошибкой `Invalid negotiation response received`.)

## Наборы параметров

**Набор параметров** — это именованный переиспользуемый набор переопределений параметров cBot, хранимый как плоский объект JSON, сопоставляющий каждое имя параметра со скалярным значением, напр. `{"Period": 14, "Label": "trend"}`. Во время запуска/бэктеста он преобразуется в файл cTrader `params.cbotset` (`{ "Parameters": { … } }`). Набор можно создать/изменить как чистый JSON в диалоге **Наборы параметров** cBot или прямо в диалоге запуска.

JSON **проверяется** при сохранении: это должен быть один плоский объект, все значения которого скалярные (строка / число / bool). Корень не-объект, массив, вложенный объект, значение `null` или некорректный JSON отклоняются (понятная ошибка в диалоге, `400 Bad Request` в API). Пустой объект `{}` разрешён и означает «без переопределений».
