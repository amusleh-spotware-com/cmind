# Rekod Keputusan Seni Bina

Ini merekodkan keputusan reka bentuk yang **tidak dapat disimpulkan daripada kod** — pertukaran, jalan tidak diambil, dan mengapa. Setiap adalah pendek: *Konteks → Keputusan → Akibat*. Keputusan struktur baru → tambah ADR di sini (nombor seterusnya) jadi jurutera seterusnya (manusia atau AI) mewarisi penalaran, bukan hanya hasilnya.

| [0001](./0001-strict-ddd-pure-core.md) | DDD Ketat dengan `Core` tulen |
| [0002](./0002-tph-instance-replaces-entity.md) | Keadaan contoh adalah TPH; peralihan menggantikan entiti |
| [0003](./0003-external-nodes-http-jwt.md) | Nod CLI cTrader adalah HTTP + JWT, tiada SSH/cangkul |
| [0004](./0004-cbotbuilder-on-web-host.md) | `CBotBuilder` berjalan di hos web dalam bekas kotak pasir |
| [0005](./0005-anthropic-raw-http.md) | Klien AI menggunakan HTTP tulen, bukan SDK Anthropic |
| [0006](./0006-copy-profile-db-lease.md) | Pengehosan salinan diselaraskan oleh pajakan DB atom |
