# 0005 — ไคลเอ็นต์ AI ใช้ HTTP ดิบ ไม่ใช่ Anthropic SDK

## บริบท

ทุกฟีเจอร์ AI (การสร้างกลยุทธ์ การซ่อมแซมตนเอง ยาม ความเสี่ยง post-mortems) เรียก Anthropic API การพึ่งพา SDK เพิ่มพื้นผิวการถ่ายทำขี่ที่เราไม่ควบคุม จับระยะการปล่อยของเรากับของพวกเขา และซ่อนสัญญาลวดที่แน่นอนที่เราต้องการให้เหตุผลสำหรับการยืดหยุ่นและต้นทุน

## การตัดสินใจ

`IAiClient` เรียก Anthropic ผ่าน **HTTP ดิบ** ผ่าน `HttpClient` ที่พิมพ์แล้ว — โดยตั้งใจ **ไม่ใช่** SDK `AiFeatureService` เป็นผู้จัดการเดี่ยวที่ใช้ร่วมกันโดยจุดปลายทาง Web `AiTools` ของ MCP และ `AiRiskGuard` พื้นผิวทั้งหมดได้รับการ **กัน้บน `AppOptions.Ai.ApiKey`**: ไม่มีคีย์ ทุกฟีเจอร์ส่งคืน `AiResult.Fail` และแอปจะทำงานโดยไม่เปลี่ยนแปลง

## ผลที่ตามมา

- ไม่จำเป็นต้องมีคีย์สำหรับการสร้าง การทดสอบ หรือ E2E — CI และ dev ในพื้นที่เรียกใช้แอปแบบเต็มโดยไม่มี AI
- เราเป็นเจ้าของรูปร่างคำขอ/การตอบสนอง นโยบาย retry/timeout และการบัญชีโทเค็นโดยชัดแจ้ง
- ฟีเจอร์ Anthropic ใหม่ต้องเดินสายด้วยมือ; เราแลกเปลี่ยนความสะดวกสำหรับการควบคุมและพื้นผิวการขึ้นต่อกันที่เล็กลง ดูข้อมูลอ้างอิง `claude-api` สำหรับรหัสแบบจำลองปัจจุบันและพารามิเตอร์
