# 戦略ヘルスチェック & アルファ減衰

すべてのエッジは減衰します — 研究は、クオンツ戦略のハーフライフが数年から数ヶ月に短縮したことを明確に示しており、*適応が発見に勝ります*。戦略ヘルスチェック機能は、戦略自体のリターン履歴から、そのエッジがまだ存在するかどうかを判定します。

**cBots → Strategy Health** (`/quant/health`) を開きます。

## What it does

リターン系列（または最も古いものから順のエクイティカーブ）が与えられた場合、以下を実行します：

- 履歴を**前半**と**最近半**に分割し、それらのSharpeレシオを比較します；
- **CUSUM変化点**スキャンを実行して、平均が最も明確にシフトした観測値（レジーム破綻）を特定し、偏差が統計的に注目に値する場合のみ報告します；
- 判定を返します：

| 判定 | 意味 |
|---|---|
| **Healthy** | 最近のパフォーマンスは過去の記録と一致しているか、それ以上です。 |
| **Degrading** | 最近のSharpeは過去の記録よりも著しく低下しています — 注視してください。 |
| **Decayed** | エッジは最近のウィンドウでは実質的に消失しています — 一時停止を検討してください。 |
| **Unknown** | 判定に十分な履歴がありません。 |

- **バックテスト実行から直接 — コピーペーストは不要。** すべての完了したバックテストは、**Backtest** リスト行とそのインスタンス詳細ビューにハート **Check strategy health** アイコンを表示します。1 回のクリックで、そのリアルのストアされたエクイティカーブでモニターを実行し、判定をダイアログに表示します。アイコンはバックテストが完了して報告書を生成するまで無効です。そのため、決してデッドコントロールではありません。内部的には、これは `POST /api/quant/health/backtest/{instanceId}` であり、保存されたレポートのエクイティカーブを読み取ります。

```http
POST /api/quant/health
{ "returns": [...] }   // or { "equity": [...] }
```

## Why it is reliable

これは、インフラストラクチャの依存性がなく、外部呼び出しもない純粋で決定論的なドメインコード（`Core.Health`）です。減衰、低下、健全、および短すぎる場合と変化点の局所化についてユニットテストされています。これは、自律エージェントをサポートする常時オンのヘルスチェックに対する手動の相手役です。同じ統計は、エッジが薄れている可視化戦略のリスクを軽減するサーキットブレーカーを駆動します。
