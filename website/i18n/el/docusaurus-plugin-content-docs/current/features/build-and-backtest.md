# Build & backtest cBots

Δημιουργία, εκτέλεση, backtest cTrader cBots (C# **και** Python, και τα δύο .NET) από ενσωματωμένο Monaco
IDE, εκτέλεση στην επίσημη εικόνα `ghcr.io/spotware/ctrader-console`.

## Build

- **Σελίδα Builder** που φιλοξενεί τον επεξεργαστή Monaco· το `CBotBuilder` μεταγλωττίζει το έργο με
  `dotnet build` **σε container που θα απορρίπτεται** (`AppOptions.BuildImage`, ο κατάλογος εργασίας bind-mount
  στο `/work`), έτσι ώστε τα στοχεύματα MSBuild δεν χρειάζονται πρόσβαση στον κεντρικό υπολογιστή. Η αποθήκευση της αποκατάστασης NuGet
  αποθηκεύεται μεταξύ των δημιουργιών μέσω κοινόχρηστου όγκου. Ο διακομιστής web χρειάζεται πρόσβαση σε υποδοχή Docker.
- Τα πρότυπα ενεργοποίησης C# + Python βρίσκονται στο `src/Nodes/Builder/Templates/`.

## Run & backtest

- **Instances** = TPH κατάσταση ιεραρχία (`Run`/`Backtest` × `Pending`/`Scheduled`/`Starting`/
  `Running`/`Stopping`/`Stopped`/`Failed`). Η μετάβαση αντικαθιστά την οντότητα (αλλαγή id),
  το id του container μεταφέρεται.
- Το `NodeScheduler` επιλέγει τον λιγότερο φορτωμένο κατάλληλο κόμβο· το `ContainerDispatcherFactory` δρομολογεί προς
  τον απομακρυσμένο κόμβο HTTP agent ή τον τοπικό Docker dispatcher.
- Οι ολοκληρωτικοί polling αναμορφώνουν τα containers που έχουν εξέλθει (τα containers backtest εξέρχονται αυτόματα μέσω
  `--exit-on-stop`); την αναφορά παρούσα → ολοκληρωμένη (αποθήκευση `ReportJson`), απούσα → απέτυχε.
- Τα live logs του container ρέουν στο πρόγραμμα περιήγησης μέσω SignalR· οι καμπύλες ιδίας αξίας backtest αναλύονται από
  την αναφορά + διαγράφονται.

## Backtest market data is cached per account

Το cTrader Console κατεβάζει ιστορικά δεδομένα tick/bar στο `--data-dir`. Αυτός ο κατάλογος είναι
**σταθερή, ανθεκτική μνήμη cache με κλειδί το λογαριασμό διαπραγμάτευσης** (τον αριθμό λογαριασμού του) — bind-mounted από
τον δίσκο του κόμβου στη δική του διαδρομή container (`/mnt/data`), ένα **ξεχωριστό, μη ένθετο mount** από τον
κατάλογο εργασίας ανά παράδειγμα. Έτσι κάθε backtest στον ίδιο λογαριασμό **επαναχρησιμοποιεί** τα ήδη κατεβασμένα δεδομένα
αντί να τα κατεβάζει ξανά σε κάθε εκτέλεση. (Παλαιότερα ο κατάλογος δεδομένων βρισκόταν κάτω από τον κατάλογο εργασίας ανά παράδειγμα, του οποίου
το id αλλάζει σε κάθε εκτέλεση, το οποίο ανάγκαζε μια νέα λήψη σε κάθε backtest.) Ο εφήμερος κατάλογος εργασίας ανά παράδειγμα εξακολουθεί να διατηρεί
το αλγόριθμο, τις παραμέτρους, τον κωδικό πρόσβασης και την αναφορά· η κοινόχρηστη μνήμη cache δεδομένων καταμετράται στη χρήση δεδομένων backtest ενός κόμβου
και διαγράφεται με την ενέργεια node-clean.

## Backtest settings

Το διάλογο **Backtest** εκθέτει τις ρυθμίσεις backtest του cTrader Console που μπορείτε να ρυθμίσετε, ώστε να μην χρειάζεται ποτέ να
αγγίξετε μια γραμμή εντολών:

- **Symbol / Timeframe** — το timeframe είναι ένα **dropdown κάθε περιόδου cTrader** (`t1`…`t1000`,
  `m1`…`m45`, `h1`…`h12`, `D1`/`D2`/`D3`, `W1`, `Month1`, και τις περιόδους Renko/Range/Heikin), στο
  κανονικό σημείωση της κονσόλας, ώστε να επιλέγετε πάντα ένα έγκυρο `--period`.
- **From / To** — το παράθυρο backtest (`--start` / `--end`).
- **Data mode** — ένα από τα τρία modes του cTrader (`--data-mode`): **Tick data** (`tick`, ακριβές),
  **m1 bars** (`m1`, γρήγορα), ή **Open prices only** (`open`, ταχύτερα).
- **Starting balance** — προεπιλογή `10000` (`--balance`). Ένα **ισοζύγιο 0 δεν τοποθετεί κανένα trade και κάνει
  το cTrader να εκπέμψει μια κενή αναφορά που κατόπιν κρασάρει** ("Message expected"), οπότε ένα ισοζύγιο που δεν είναι μηδέν
  αποστέλλεται πάντα.
- **Commission** — `--commission`.
- **Spread** — `--spread`, ένα **αριθμητικό πεδίο σε pips που δεν μπορεί να πάει κάτω από 0**. Είναι **κρυμμένο σε Tick
  data mode**, όπου το cTrader προέρχεται το spread από τα ίδια τα δεδομένα tick (δεν αποστέλλεται `--spread`).

Ο κατάλογος δεδομένων (`--data-file` / `--data-dir`) διαχειρίζεται από την ίδια την εφαρμογή (μια μνήμη cache ανά λογαριασμό, βλ.
παραπάνω), όχι εκτεθειμένη στο διάλογο.

:::note cTrader crashes on an empty backtest
Εάν ένα backtest δεν παράγει **κανένα αποτέλεσμα** — καμία διαπραγμάτευση ή καμία δεδομένα αγοράς για τις επιλεγμένες ημερομηνίες/σύμβολο —
ο συγγραφέας αναφοράς του cTrader Console ρίχνει `Message expected` και εξέρχεται χωρίς αναφορά. Η εφαρμογή δεν μπορεί
να καθηλώσει αυτό το upstream bug, αλλά το ανιχνεύει και σημειώνει την παράδειγμα **Failed** με ένα κατανοητό λόγο
("no backtest results for the selected range…") αντί ενός raw stack trace. Επιλέξτε ένα ευρύτερο εύρος ημερομηνιών
που έχει διαθέσιμα δεδομένα αγοράς και δοκιμάστε ξανά.
:::

## Instance detail page

Η άνοιγμα μιας παράδειγμας (`/instance/{id}`) εμφανίζει την κατάστασή της σε ζωντανή λήψη, τα logs και — για ένα backtest — την καμπύλη ιδίας αξίας. Ο **τίτλος της σελίδας του προγράμματος περιήγησης** αντικατοπτρίζει την ειδική παράδειγμα (**όνομα cBot · είδος · σύμβολο**, π.χ.
`TrendBot · Backtest · EURUSD`) έτσι ώστε μια σελίδα live-run και μια σελίδα backtest είναι διακρίσιμες με μια ματιά.
Ένα run και ένα backtest του ίδιου cBot παρακολουθούνται ως ξεχωριστές **lineages** (ένα σταθερό lineage id που μεταφέρεται
κατά τις μεταβάσεις κατάστασης), οπότε η σελίδα ακολουθεί ακριβώς μια παράδειγμα και ποτέ δεν αναμιγνύει τα δεδομένα ενός run με ένα backtest.

## Instance lifecycle controls

Κάθε σειρά παράδειγμας (και η σελίδα λεπτομερειών της) έχει ελέγχους που ταιριάζουν στην κατάσταση. Μια **ενεργή** παράδειγμα δείχνει
**Stop**; μια **τελική** (Stopped / Completed / Failed) δείχνει **Start (▶)** για να την επανεκκινήσετε με
την ίδια cBot, λογαριασμό, σύμβολο, timeframe, σύνολο παραμέτρων και εικόνα (ένα run επανεκκινεί ως run, ένα
backtest ως backtest). Το κλικ στο Stop εμφανίζει μια ειδοποίηση "Stopping…" και απενεργοποιεί το εικονίδιο μέχρι να επιλυθεί,
και ένα νεοσυσταθέν run εμφανίζεται αμέσως στη λίστα — χωρίς ανανέωση σελίδας.

Τα logs της κονσόλας είναι **μονιμοποιημένα όταν ολοκληρώνεται μια παράδειγμα** — για ένα run (σε Stop) και για ένα
**backtest** (σε ολοκλήρωση) ομοίως — έτσι ώστε τα logs του τελευταίου run να παραμένουν προβάσιμα στη σελίδα λεπτομερειών και,
μέσω της γραμμής εργαλείων log, **αντιγράφονται στο πρόχειρο** (εικονίδιο Copy logs) ή **λήφθηκαν** (εικονίδιο Download logs)
ακόμη και μετά το container είναι σβησμένο. Και τα δύο δρουν στο πλήρες log κονσόλας της παράδειγμας, όχι μόνο στο
tail στην οθόνη.

Ένα **ανεβασμένο** `.algo` δεν κατασκευάστηκε ποτέ εδώ, οπότε η **Last Build** στήλη του σε σελίδα cBots αφήνεται
κενή (εμφανίζει έναν χρόνο δημιουργίας μόνο για cBots που δημιουργείτε στο πρόγραμμα περιήγησης).

## Edit & re-run a stopped instance

Μια **σταματημένη** παράδειγμα (run ή backtest) έχει έναν έλεγχο **Edit** — ένα εικονίδιο στη σειρά του στη λίστα **και**
δίπλα στο Start/Stop στη σελίδα λεπτομερειών — που ανοίγει ένα διάλογο **προ-γεμάτο** με την τρέχουσα ρύθμισή του.
Μπορείτε να αλλάξετε τον **λογαριασμό διαπραγμάτευσης, σύμβολο, timeframe, σύνολο παραμέτρων και tag εικόνας** (και, για ένα
backtest, το **παράθυρο και όλες τις ρυθμίσεις backtest** παραπάνω), στη συνέχεια **Save & start** την επανεκκινεί με τις
νέες ρυθμίσεις (αντικαθιστώντας τη σταματημένη παράδειγμα). Ο έλεγχος είναι **απενεργοποιημένος ενώ η παράδειγμα είναι ενεργή** —
μόνο μια σταματημένη παράδειγμα μπορεί να επεξεργαστεί.

## Run from the code editor

Το κλικ στο **Run** στον επεξεργαστή κώδικα ανοίγει ένα διάλογο αντί να ξεκινήσει ένα τυφλό, hard-coded run:

- **Trading account** (required) — ο λογαριασμός cTrader που συνδέεται η cBot.
- **Parameter set** (optional) — επιλέξτε ένα υπάρχον σύνολο, ή αφήστε το κενό για να εκτελεστεί με την **προεπιλεγμένες τιμές παραμέτρων** της cBot. Ένα **+** κουμπί δίπλα στον επιλογέα δημιουργεί ένα νέο σύνολο παραμέτρων
  inline (βλ. παρακάτω) και το επιλέγει.
- **Symbol / Timeframe** προεπιλογή `EURUSD` / `h1` και μπορούν να αλλαχθούν· **Cancel** ή **Run**.

Σε **Run** ο επεξεργαστής αποθηκεύει + δημιουργεί την τρέχουσα πηγή, ξεκινά την παράδειγμα στον επιλεγμένο λογαριασμό
με τις επιλεγμένες παραμέτρους, στη συνέχεια ουράες τα live container logs. (Το log stream προωθεί το
καταχωρημένο cookie auth του χρήστη στο hub SignalR `/hubs/logs`, άρα συνδέεται αντί να αποτύχει με
`Invalid negotiation response received`.)

## Parameter sets

Ένα **parameter set** είναι ένα ονοματισμένο, επαναχρησιμοποιούμενο σύνολο παραμέτρων cBot που αποθηκεύονται ως ένα
ενιαίο αντικείμενο JSON που αντιστοιχεί κάθε όνομα παραμέτρου σε μια κλιμακωτή τιμή, π.χ. `{"Period": 14, "Label": "trend"}`. Κατά
τη διάρκεια του run/backtest μετατρέπεται στο αρχείο cTrader `params.cbotset`
(`{ "Parameters": { … } }`). Μπορείτε να δημιουργήσετε/επεξεργαστείτε ένα σύνολο ως raw JSON από το διάλογο **Parameter
sets** της cBot ή inline από το διάλογο Run.

Κάθε σύνολο παραμέτρων **ανήκει σε μια cBot**: το διάλογο New Parameter Set απαριθμεί όλες τις cBots σας και πρέπει
**να επιλέξετε μία** — η δημιουργία αποκλείεται μέχρι να επιλεγεί μια cBot. Το **όνομα ενός συνόλου είναι μοναδικό ανά cBot**:
η δημιουργία ή μετονομασία ενός συνόλου σε ένα όνομα που ένα άλλο σύνολο της ίδιας cBot χρησιμοποιεί ήδη αποδοκιμάζεται (ένα
σαφές λάθος στο διάλογο, `409 Conflict` στο API). Το ίδιο όνομα μπορεί να χρησιμοποιηθεί ξανά σε μια **διαφορετική** cBot.

Το JSON είναι **επικυρωμένο** σε αποθήκευση: πρέπει να είναι ένα ενιαίο ενιαίο αντικείμενο του οποίου οι τιμές είναι όλες κλιμακωτές
(string / number / bool). Ένα μη-object root, ένας πίνακας, ένα ένθετο αντικείμενο, μια `null` τιμή, ή ακατάστατο
JSON αποδοκιμάζεται (ένα σαφές λάθος στο διάλογο, `400 Bad Request` στο API). Ένα κενό αντικείμενο `{}`
είναι επιτρεπτό και σημαίνει "no overrides".

## cTrader Console CLI notes

Τα backtests χρειάζονται `--data-mode` (προεπιλογή `m1`), ημερομηνίες ως `dd/MM/yyyy HH:mm`, και
`params.cbotset` JSON ορίσματος positional; ο `run` απορρίπτει `--data-dir` (backtest-only). Δες
`ContainerCommandHelpers`.

## Nodes & scale

Η χωρητικότητα εκτέλεσης κλιμακώνεται με την προσθήκη κόμβων agents (αυτο-εγγραφή + heartbeat). Δες
[node discovery](../operations/node-discovery.md) και [scaling](../deployment/scaling.md).

## A trading account is required

Η εκτέλεση ή το backtest μιας cBot χρειάζεται ένα λογαριασμό διαπραγμάτευσης cTrader για σύνδεση. Μέχρι να προσθέσετε
έναν κάτω από **Trading accounts**, τα κουμπιά **Run New cBot** / **Backtest New cBot** είναι απενεργοποιημένα (με
ένα tooltip) και η σελίδα εμφανίζει μια προτροπή που σύνδεσμο προς την ρύθμιση λογαριασμού — δεν χτυπάτε
πλέον ένα raw σφάλμα `stream connect failed` από ένα bot χωρίς λογαριασμό.
