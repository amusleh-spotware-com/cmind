# บันทึกการตัดสินใจสถาปัตยกรรม

สิ่งเหล่านี้บันทึกการตัดสินใจออกแบบที่ **คุณไม่สามารถอนุมานได้จากโค้ด** — การแลกเปลี่ยน เส้นทางที่ไม่ได้ใช้ และเหตุผล แต่ละเรื่องมีความสั้น: *บริบท → การตัดสินใจ → ผลที่ตามมา* การตัดสินใจโครงสร้างใหม่ → เพิ่ม ADR ที่นี่ (หมายเลขถัดไป) เพื่อให้วิศวกรคนต่อไป (มนุษย์หรือ AI) สืบทอดเหตุผล ไม่ใช่แค่ผลลัพธ์

| # | การตัดสินใจ |
|---|---|
| [0001](./0001-strict-ddd-pure-core.md) | DDD ที่เข้มงวดกับ `Core` บริสุทธิ์ |
| [0002](./0002-tph-instance-replaces-entity.md) | สถานะอินสแตนซ์คือ TPH; การเปลี่ยนแปลงแทนที่เอนทิตี |
| [0003](./0003-external-nodes-http-jwt.md) | โหนด cTrader CLI เป็น HTTP + JWT ไม่มี SSH/shell |
| [0004](./0004-cbotbuilder-on-web-host.md) | `CBotBuilder` ทำงานบนโฮสต์เว็บในคอนเทนเนอร์แซนด์บ็อกซ์ |
| [0005](./0005-anthropic-raw-http.md) | ไคลเอ็นต์ AI ใช้ HTTP ดิบ ไม่ใช่ Anthropic SDK |
| [0006](./0006-copy-profile-db-lease.md) | การโฮสต์คัดลอกได้รับการประสานงานโดยลีสฐานข้อมูลอะตอมิก |
