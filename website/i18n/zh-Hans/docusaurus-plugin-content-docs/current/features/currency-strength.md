# AI 宏观货币强度与前向展望

cMind 提供**AI 辅助、数学确定性的**宏观货币强度引擎。它对可配置的货币宇宙进行排名 — 8 大主要货币加新兴市场和小众货币 — 按**当前**基本面强度，并为选定时间范围内的每个货币对投射**前向方向展望**（1M / 3M / 6M / 12M）。每个排名、每个货币对偏差和每个数字都由域核心中的纯确定性数学计算；LLM 仅*收集*数据无法发布的前向输入，并用清晰英文*解释*结果。它从不虚构排名、方向或数字。

> **诚实的局限。**基本面可以很好地预测中长期价值，短期价值预测较差。将其视为仓位 / 汇聚过滤器，**而非**短期时机信号。接近高影响发布的读数（NFP/CPI/央行）带有噪声。不构成财务建议。

## 工作原理

1. **当前基本面来自经济日历，而非 LLM。**硬数字 — 政策利率、CPI vs 目标、GDP、就业、贸易余额 — 和它们的**意外 z 分数**是**某个时间点**从[经济日历](./economic-calendar.md)模块（FRED/BLS/BEA/ECB 和央行日程表）源获的。历史快照从不泄露前向信息。
2. **LLM 仅收集日历无法发布的内容** — 按货币：**前向**轨迹（预期政策利率路径（bp）、通胀趋势 vs 目标、增长动能）和**地缘政治**展望（风险偏好、关税、财政/债务、选举），加上日历缺少的任何新兴市场/小众当前数字。严格 JSON、分层感知验证、网络搜索启用。
3. **域确定性地计算排名和前向矩阵。**每个驱动因素被评分为**分层内 z 分数**（所以 50% 通胀小众不会扭曲主要货币），winsorized，权重加总成复合，并按最强→最弱排名，ISO 并列打破稳定。前向层沿其轨迹带动每个复合 —
   `projected = current + horizonScale · Σ trajectoryDriver·weight` — 并将每个货币对的投射差异映射到**方向偏差**（▲ 升值 / ▬ 中立 / ▼ 贬值）及置信度。
4. **LLM 用清晰语言解释**排名和顶部货币对看法。

## 驱动因素

| 驱动因素 | 对强度的影响 | 注释 |
|---|---|---|
| 政策利率与轨迹 | 更高 / 鹰派 ⇒ 更强 | 最高权重；央行分化驱动最大的差距。 |
| 通胀（CPI vs 目标） | 高于目标 ⇒ 更弱 | 反向评分（购买力拖累）。 |
| GDP 增长 | 相对增长更高 ⇒ 更强 | 与面板对比。 |
| 就业 | 更强的劳动力 ⇒ 更强 | 助推政策路径。 |
| 贸易余额 / 经常项 | 顺差 ⇒ 更强 | 结构性需求。 |
| 政策立场 | 鹰派 ⇒ 更强 | 主要的长期驱动因素。 |
| 意外动能 | 最近超预期 ⇒ 更强 | 来自日历的意外 z 分数。 |
| 地缘政治 / 风险 | 风险厌恶 ⇒ 安全港（USD/JPY/CHF）更强 | 有界前向风险增量。 |
| 实际收益率 / 利差 *（新兴市场/小众）* | 正实际利率 ⇒ 更强 | 平静制度中的主导新兴市场驱动因素。 |
| 外部脆弱性 *（新兴市场/小众）* | 赤字 / 低储备 / USD 债务 ⇒ 更弱 | 结构性贬值压力。 |
| 贸易条件 *（商品出口国）* | 出口价格上升 ⇒ 更强 | BRL、ZAR、CLP、NOK、AUD、CAD。 |
| 政治 / 制度风险 *（新兴市场/小众）* | 不稳定 ⇒ 更弱 | 更宽的死区、有限的置信度。 |

## 分层宇宙（主要 + 新兴市场 + 小众）

宇宙**部署可配置**（`App:CurrencyStrength:Universe`）— 添加货币是配置，不是代码。每种货币携带**层级**（`Major` / `EmergingMarket` / `Exotic`）来调整权重、死区宽度和置信度上限：

- **主要货币** — USD、EUR、GBP、JPY、AUD、NZD、CAD、CHF（利率水平主导）。
- **新兴市场** — CNH、INR、BRL、MXN、ZAR、KRW、SGD、PLN（+ 斯堪的纳维亚 NOK/SEK）；利差 + 风险 +
  外部脆弱性权重提升，中等置信度。
- **小众** — TRY、HUF、CZK、加上 USD 挂钩的 HKD/SAR；低置信度、更宽的死区、有限的置信度。**挂钩 / 高度受管理的**货币（HKD、SAR、CNH）被标记，其轨迹被降权，其货币对展望被限制于`Neutral`，所以挂钩从不被理解为自由浮动信号。

因为官方新兴市场/小众统计数据频率较低、有修订且有时不透明，AI 收集的数字携带**每层置信度**显示为可靠性徽章。

## 优雅降级

| 日历 | AI | 结果 |
|---|---|---|
| ✅ | ✅ | 完整排名 + 前向投射 + 叙述（`CalendarAndAi`）。 |
| ✅ | ❌ | 仅日历当前排名，无前向投射（`CalendarOnly`）。 |
| ❌ | ✅ | AI 收集的当前数字 + 前向，较低置信度（`AiOnly`）。 |
| ❌ | ❌ | 无快照 — 小部件隐藏，页面显示空状态。 |

应用无论哪种方式运行保持不变。AI 依赖于 AI 键；日历部分遵重其自身的
白标签门 + 运行时切换。

## 使用方法

- **启用 AI**（设置 → AI）并通过您自己的仪表板从**自定义**对话框**启用小部件**
  （"货币强度" — 选择性，默认隐藏）。小部件显示顶部强/弱货币和顶部 3M 货币对看法；它链接到完整页面。
- **完整页面** — `/ai/currency-strength`：时间范围选择器（1M/3M/6M/12M）、层级过滤器
  （全部/主要/新兴市场/小众）、当前排名、前向预测、货币对展望矩阵（偏差 +
  置信度，挂钩/低置信度标记）和 AI 叙述。按**立即刷新**（所有者）重新生成。后台工作者（`App:CurrencyStrength:RefreshEnabled`，**默认`true`**）按日程刷新，所以页面开箱即用；部署或所有者关闭它（或禁用 AI / 经济日历功能，刷新器会通过降级到无快照来遵重）。

## 程序化访问

一个共享读模型（`ICurrencyStrengthQuery`）可通过三种方式访问：

- **应用内 AI** — 直接注入（进程内）到 AI 功能。
- **MCP** — `currency_strength` 工具（参数 `horizon`、`tier`）用于 AI 客户端/代理。
- **cBot REST** — `GET /api/market/v1/currency-strength/{latest,history,pair/{base}/{quote}}`，由**相同的** `CalendarJwt` 机制保护，如[日历 cBot API](./calendar-cbot-api.md)，加上**`market:read`**范围。cBot 用 `market:read` 注册 API 客户端，在 `POST /api/calendar/v1/token` 交换其 id + 密钥以获得短期 JWT，并用 `Bearer` 令牌调用端点。无第二个 JWT 方案，无第二个密钥 — 泄露的令牌是只读、市场范围、短期和可撤销的。

有关令牌流和复制粘贴示例，请参阅[日历 cBot API](./calendar-cbot-api.md)。
