# Transaction Cost Analysis (TCA)

Το execution alpha είναι μικροσκοπικό ανά συναλλαγή και τεράστιο σε χιλιάδες από αυτές — είναι μεγάλο μέρος του πώς οι τράπεζες και τα prop desks διατηρούν το πλεονέκτημά τους. Το TCA μετρά πόσο μακριά η τιμή που πετύχατε πραγματικά αποκλίθηκε από την τιμή όταν *αποφασίσατε* να διαπραγματευτείτε.

Ανοίξτε **cBots → Execution Cost** (`/quant/tca`).

## Τι μετρά

Δεδομένης της **τιμής άφιξης (decision price)**, της **πλευράς** και των **fills** σας (τιμή × ποσότητα), αναφέρει:

- **Μέση τιμή fill (VWAP)** — η σταθμισμένη κατά τον όγκο τιμή που πετύχατε πραγματικά.
- **Slippage (bps)** — η απόκλιση από την άφιξη στη VWAP σε basis points, **προσημασμένη ώστε ένας θετικός αριθμός να είναι
  κόστος** (αγορά πάνω από την άφιξη ή πώληση κάτω από αυτήν) και ένας αρνητικός αριθμός να είναι βελτίωση τιμής.
- **Implementation shortfall** — αυτό το κόστος εκφρασμένο σε όρους τιμή × ποσότητα: τα χρήματα που σας κόστισε η απόκλιση σε αυτή την εντολή.

```http
POST /api/quant/tca
{ "arrivalPrice": 1.1000, "side": "Buy",
  "fills": [ { "price": 1.1010, "quantity": 100 }, { "price": 1.1020, "quantity": 100 } ] }
```

## Έξυπνο slicing (Almgren-Chriss)

Πέρα από τη μέτρηση του κόστους, το cMind μπορεί να σχεδιάσει μια μεγάλη εντολή για να *ελαχιστοποιήσει* το κόστος της. **cBots → Execution Schedule**
(`/quant/execution`) χτίζει ένα **βέλτιστο πρόγραμμα εκτέλεσης Almgren-Chriss**: δεδομένης της συνολικής ποσότητας,
ενός αριθμού φετών, της διακριτικότητας κινδύνου σας, της μεταβλητότητας και της προσωρινής αντίκτυπης της αγοράς, επιστρέφει το μέγεθος που θα διαπραγματευτεί σε κάθε φέτα. Η υψηλότερη διακριτικότητα κινδύνου **φορτώνει μπροστά** το πρόγραμμα (περικόπτοντας τον κίνδυνο χρονισμού)· μηδενική διακριτικότητα κινδύνου ισοπεδώνει σε ομοιόμορφο **TWAP**. Οι φέτες αθροίζονται πάντα στο σύνολο.

```http
POST /api/quant/execution-schedule
{ "totalQuantity": 100, "slices": 5, "riskAversion": 2, "volatility": 0.02, "temporaryImpact": 0.1 }
```

## Γιατί είναι αξιόπιστο

Καθαρός, ντετερμινιστικός domain code (`Core.Execution`) χωρίς εξάρτηση υποδομής και χωρίς εξωτερικές κλήσεις
— unit-tested για το πρόσημο κόστους αγοράς/πώλησης, τη βελτίωση τιμής, το μηδενικό slippage, τη συνάθροιση VWAP και
τους guards εισόδου. Αυτό είναι το μισό της μέτρησης της ποιότητας εκτέλεσης· είναι η ίδια μετρική shortfall που χρησιμοποιεί η μηχανή copy για να κρίνει (και, με το έξυπνο slicing, να μειώσει) το κόστος των καθρεφτισμένων εντολών.
