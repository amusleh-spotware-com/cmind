# 0002 — 实例状态是 TPH；转换替换实体

## 背景

运行/回测实例通过状态移动（待处理 → 已调度 → 启动中 → 运行中 → 终端）。
我们使用 EF Core **表-每-层级 (TPH)** 建模状态：每个状态是一个子类型
（`StartingRunInstance`、`RunningRunInstance`、…）。EF 的 TPH 判别器列在
现有行上**无法更改**。

## 决策

状态转换**用新的子类型实例替换实体**，而不是变更状态
字段。因为行被替换，**实例 id 在**启动中 → 运行中 → 终端中**变化**。
**容器 id 稳定**，跨越转换被携带；HTTP 节点代理由
容器 id 键控用于状态/报告/停止/日志。

## 后果

- 每个状态是一个不同的类型，仅包含该状态有效的字段和方法——非法
  转换和无意义的字段访问是编译错误，不是运行时检查。
- 调用者**不得**在转换中缓存实例 id；使用容器 id 作为稳定
  句柄用于任何跨越状态的内容。
- 转换逻辑位于 `InstanceTransitions`；id 变化是有意的，不是错误。
