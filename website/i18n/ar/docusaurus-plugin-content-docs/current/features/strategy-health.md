# صحة الاستراتيجية والتحلل الألفا

كل حافة تتحلل — البحث واضح بأن نصف العمر لاستراتيجية كمية انهار من سنوات إلى أشهر، لذا *التكيف يتفوق على الاكتشاف*. يخبرك مراقب Strategy Health، من سجل العوائد الخاص بالاستراتيجية الخاصة بك، ما إذا كانت الحافة لا تزال موجودة.

افتحه على **cBots → Strategy Health** (`/quant/health`).

## ما يحسبه

بالنظر إلى سلسلة العوائد (أو منحنى الأسهم/الرصيد، الأقدم أولاً)، فإنه:

- يقسم السجل إلى نصف **سابق** و**حديث** ويقارن نسب Sharpe الخاصة بهما؛
- يشغل فحص **CUSUM change-point** لتحديد الملاحظة التي تحول المتوسط بوضوح فيها (انقطاع النظام)، يتم الإبلاغ عنها فقط عندما يكون الانحراف ملحوظاً إحصائياً؛
- يعيد حكماً:

| الحكم | المعنى |
|---|---|
| **Healthy** | الأداء الحديث متوافق مع (أو أفضل من) السجل السابق. |
| **Degrading** | Sharpe الحديث أضعف بشكل ملموس من السجل السابق — راقب عن كثب. |
| **Decayed** | اختفت الحافة بشكل فعلي في النافذة الحديثة — فكر في الإيقاف. |
| **Unknown** | لا توجد نقاط بيانات كافية للحكم. |

- **مباشرة من backtest run — بدون نسخ لصق.** كل backtest مكتمل يعرض رمز **Check strategy health** على صف قائمة **Backtest** وعلى عرض تفاصيل الحالة الخاصة به؛ انقر واحد يشغل المراقب على منحنى الأسهم المخزن لتلك الحالة ويظهر الحكم في صندوق حوار. يتم تعطيل الرمز حتى يكتمل backtest وينتج تقريراً، لذا فهو لا يكون أبداً عنصر تحكم ميت. تحت الغطاء هذا هو `POST /api/quant/health/backtest/{instanceId}`، الذي يقرأ منحنى الأسهم من التقرير المخزن.

```http
POST /api/quant/health
{ "returns": [...] }   // أو { "equity": [...] }
```

## لماذا هو موثوق

إنه كود مجال نقي وحتمي (`Core.Health`) بدون أي اعتماديات على البنية التحتية وبدون استدعاءات خارجية — مختبر على الحالات المتحللة والمتدهورة والصحية وقصيرة جداً وتحديد موقع change-point. إنه الرفيق اليدوي لفحوصات الصحة دائماً على الإنترنت التي تدعم الوكلاء المستقلين: نفس الإحصاءات تدفع قاطع الدارة الذي يقلل من مخاطر الاستراتيجية الحية التي تتلاشى حافتها.
