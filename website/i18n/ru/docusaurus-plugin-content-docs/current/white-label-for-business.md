# Белый ярлык cMind для вашего бизнеса 🏢

Запускаете проприетарную компанию, брокерский стол или сервис копирования торговли? cMind был построен с первого дня, чтобы быть **перепроданным как вашу собственную продукцию**. Каждая поверхность — имя, логотип, favicon, цвета, даже устанавливаемое приложение для телефона — изгибается под ваш бренд. Ваши клиенты видят *вашу* компанию. Нет изменений кода, нет разветвления, просто config.

:::tip[TL;DR]
Укажите `App:Branding` на ваше имя, цвета и логотип. Перезагрузитесь. Готово. Полный технический справочник живет в [документе функции белого ярлыка](./features/white-label.md).
:::

## Что вы можете перебрендировать

| Поверхность | Что изменяется |
|---|---|
| **Имя продукта** | Текст строки приложения + название вкладки браузера |
| **Логотип и favicon** | Ваши отметки везде, включая вкладку браузера |
| **Цвета** | Полная палитра — первичная, поверхности, цвета статуса — течет через весь пользовательский интерфейс *и* собственный CSS приложения через токены дизайна |
| **Устанавливаемое приложение (PWA)** | Имя add-to-home-screen, значок и заставка используют ваш бренд |
| **Meta / SEO** | Описание и URL поддержки ваши |
| **Пользовательский CSS** | Инъектируйте собственный полировку для последних 5% |

Всё по умолчанию для идентичности стандартного cMind, поэтому вы переопределяете только то, что вам нужно.

## 60-секундный перебрендинг

Установите эти параметры в вашем развертывании (JSON config или переменные окружения):

```json
{
  "App": {
    "Branding": {
      "ProductName": "AcmeFX",
      "CompanyName": "Acme Markets Ltd",
      "SupportUrl": "https://support.acme.example",
      "LogoUrl": "/branding/acme-logo.svg",
      "FaviconUrl": "/branding/acme.ico",
      "PrimaryColor": "#2D7FF9",
      "SecondaryColor": "#1E63C8",
      "ShowSiteLink": false
    }
  }
}
```

Форма переменной окружения: `App__Branding__ProductName=AcmeFX`. Цвета проверяются при запуске — плохое значение hex отказывает загрузку с четким сообщением вместо отрисовки сломанной страницы. Хорошо и громко, ровно когда вы этого хотите.

## Ссылка «Создано cMind»

По **умолчанию** приборная панель показывает небольшую, тактичную ссылку **«Создано cMind»**, которая направляет посетителей обратно на эот сайт. Это по умолчанию, потому что мы гордимся проектом и это помогает другим трейдерам его найти — но это **ваше решение**.

- **Оставьте это** (по умолчанию): хорошая кредит-ссылка на приборной панели. Стоит вам ничего, помогает проекту.
- **Скройте это**: установите `App__Branding__ShowSiteLink=false` и это полностью исчезает — идеально для полностью белого ярлыка развертывания, где продукт неоспоримо *ваш*.

Смотрите [документ функции белого ярлыка](./features/white-label.md#ссылка-создано) для ровно где это отрисовывается.

## Мультитенантное, брендирование для каждого клиента

Потому что брендирование — это просто config развертывания, каждое развертывание клиента может нести свою собственную личность. Запустите отдельный экземпляр для каждого клиента или управляйте брендированием из вашей собственной плоскости управления — приложение читает это из `IOptionsMonitor`, поэтому это может даже перестроить тему в реальном времени, когда параметры изменяются.

Комбинируйте это с:

- **[Переключатели функций](./features/feature-toggles.md)** — решайте, какие возможности видит каждый клиент.
- **[Правила проприетарной компании](./features/prop-firm.md)** — применяйте правила испытаний с отслеживанием капитала в реальном времени.
- **[Комиссии за производительность](./features/copy-performance-fees.md)** + **[рынок поставщиков](./features/copy-provider-marketplace.md)** — монетизируйте копирование торговли.
- **[Соответствие](./features/compliance.md)** — сохраняйте аудиторский след, который запросит ваш регулятор.

## Активы и хостинг

Сбросьте ваш логотип/favicon в `wwwroot/branding/` Web приложения (или укажите `LogoUrl`/`FaviconUrl` на любой абсолютный URL). Развертайте как вы считаете — [Docker](./deployment/local.md), [Kubernetes](./deployment/kubernetes.md), [Azure](./deployment/cloud-azure.md) или [AWS](./deployment/cloud-aws.md).
