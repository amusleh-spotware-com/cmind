# بناء واختبار cBots

بناء واختبار cBots من cTrader (C# **و** Python، كلاهما .NET) من محرر Monaco المدمج في المتصفح، التشغيل على صورة `ghcr.io/spotware/ctrader-console` الرسمية.

## البناء

- صفحة **Builder** تستضيف محرر Monaco؛ يقوم `CBotBuilder` بترجمة المشروع باستخدام
  `dotnet build` **في حاوية مؤقتة** (`AppOptions.BuildImage`، مع ربط دليل العمل
  في `/work`)، بحيث لا تستطيع أهداف MSBuild غير الموثوقة الوصول إلى المضيف. يتم تخزين استعادة NuGet مؤقتاً
  عبر عمليات البناء من خلال مجلد مشترك. يحتاج المضيف على الويب إلى إمكانية الوصول إلى مقبس Docker.
- توجد قوالب بدء C# و Python في `src/Nodes/Builder/Templates/`.

## التشغيل والاختبار

- **Instances** = تسلسل هرمي حالة TPH (`Run`/`Backtest` × `Pending`/`Scheduled`/`Starting`/
  `Running`/`Stopping`/`Stopped`/`Failed`). يؤدي الانتقال إلى استبدال الكيان (تغيير المعرف)،
  مع نقل معرف الحاوية.
- يختار `NodeScheduler` العقدة الأقل تحميلاً المؤهلة؛ `ContainerDispatcherFactory` يوجه إلى
  وكيل HTTP العقدة البعيدة أو موزع Docker المحلي.
- توفق أدوات الإكمال الحاويات الخارجة (حاويات الاختبار تخرج تلقائياً عبر
  `--exit-on-stop`)؛ تقرير موجود → مكتمل (تخزين `ReportJson`)، غير موجود → فشل.
- تتدفق سجلات الحاوية المباشرة إلى المتصفح عبر SignalR؛ يتم تحليل منحنيات الأسهم من
  التقرير ورسمها بيانياً.

## بيانات سوق الاختبار مخزنة مؤقتاً لكل حساب

يقوم cTrader Console بتنزيل بيانات التجزئة/الأعمدة التاريخية إلى دليل `--data-dir`. هذا الدليل هو **ذاكرة تخزين مؤقت مستقرة ومستمرة مفتاحة حساب التداول** (رقم الحساب الخاص به) — مربوطة من قرص العقدة في مسار الحاوية الخاص به (`/mnt/data`)، **جبل منفصل وغير متداخل** من دليل العمل لكل مثيل. وبالتالي فإن كل اختبار على نفس الحساب **يعاد استخدام** البيانات المحملة بالفعل بدلاً من إعادة تحميلها في كل تشغيل. (في السابق كان دليل البيانات يعيش تحت دليل العمل لكل مثيل، الذي يتغير معرفه في كل تشغيل، مما أجبر على تنزيل جديد في كل اختبار.) لا يزال دليل العمل المؤقت لكل مثيل يحتفظ بالخوارزمية والمعاملات وكلمة المرور والتقرير؛ يتم حساب ذاكرة التخزين المؤقت للبيانات المشتركة في استخدام بيانات الاختبار للعقدة وتنظيفها بواسطة إجراء تنظيف العقدة.

## إعدادات الاختبار

يعرض الحوار **Backtest** كل إعدادات التي يقبلها CLI لـ cTrader Console، لذلك لا تضطر أبداً إلى لمس سطر أوامر:

- **From / To** — نافذة الاختبار (`--start` / `--end`).
- **Data mode** — `m1` (أعمدة لمدة دقيقة واحدة) أو `tick` (`--data-mode`).
- **Starting balance** — الافتراضي هو `10000` (`--balance`). **رصيد 0 لا يضع أي صفقات ويجعل cTrader ينبعث تقريراً فارغاً يتعطل** ("Message expected")، لذا يتم إرسال رصيد غير صفري دائماً.
- **Commission** و **Spread** (`--commission` / `--spread`، spread بالنقاط).
- **Advanced options** — صندوق `name=value` بدون تنسيق لكل سطر لأي خيار اختبار آخر يدعمه cTrader (مثل `applyCommissionAutomatically=true`)؛ كل سطر يصبح وسيطة CLI `--name value`.

## صفحة تفاصيل المثيل

يؤدي فتح مثيل (`/instance/{id}`) إلى عرض حالته الحية وسجلاته و — لاختبار — منحنى الأسهم. **عنوان علامة التبويب في المتصفح** يعكس المثيل المحدد (**اسم cBot · نوع · رمز**، على سبيل المثال
`TrendBot · Backtest · EURUSD`) بحيث يكون علامة تبويب التشغيل المباشر وعلامة تبويب الاختبار مميزة بسهولة. يتم تتبع عملية تشغيل واختبار لنفس cBot كـ **lineages** مختلفة (معرف lineage مستقر محمول عبر تحولات الحالة)، لذا تتبع الصفحة بالضبط مثيل واحد ولا تخلط أبداً بيانات التشغيل مع بيانات الاختبار.

## عناصر تحكم دورة حياة المثيل

يحتوي كل صف مثيل (وصفحة التفاصيل الخاصة به) على عناصر تحكم صحيحة الحالة. يُظهر مثيل **نشط** **Stop**؛ واحد **طرفي** (Stopped / Completed / Failed) يظهر **Start (▶)** لإعادة إطلاقه بنفس cBot والحساب والرمز والإطار الزمني ومجموعة المعاملات والصورة (إعادة تشغيل كتشغيل، اختبار كاختبار). يؤدي النقر فوق Stop إلى ظهور إشعار "Stopping…" وتعطيل الرمز حتى يتم حله، وتظهر عملية تشغيل تم إنشاؤها حديثاً في القائمة فوراً — بدون إعادة تحميل الصفحة.

يتم **حفظ سجلات Console عند إنهاء المثيل** — لعملية تشغيل (عند التوقف) ولـ **اختبار** (عند الإكمال) على حد سواء — بحيث تبقى سجلات آخر تشغيل قابلة للعرض على صفحة التفاصيل و، عبر شريط الأدوات السجل، **المنسوخة إلى الحافظة** (رمز نسخ السجلات) أو **تم التنزيل** (رمز تنزيل السجلات) حتى بعد اختفاء الحاوية. يعمل كلاهما على سجل وحدة التحكم الكامل للمثيل، وليس فقط الذيل على الشاشة.

لم يتم بناء `.algo` **المرفوع** أبداً هنا، لذا عمود **Last Build** الخاص به على صفحة cBots يُترك فارغاً (يظهر وقت البناء فقط لـ cBots التي تبنيها في المتصفح).

## تحرير وإعادة تشغيل مثيل متوقف

مثيل **متوقف** (تشغيل أو اختبار) لديه عنصر تحكم **Edit** — رمز على صفه في القائمة **و** بجانب Start/Stop على صفحة التفاصيل الخاصة به — يفتح حواراً **مملوءاً مسبقاً** بتكوينه الحالي. يمكنك تغيير **حساب التداول والرمز والإطار الزمني ومجموعة المعاملات وعلامة الصورة** (و، لاختبار، **النافذة وجميع إعدادات الاختبار** أعلاه)، ثم يؤدي **Save & start** إلى إعادة إطلاقه بالإعدادات الجديدة (استبدال المثيل المتوقف). عنصر التحكم **معطل أثناء نشاط المثيل** — يمكن تحرير مثيل متوقف فقط.

## التشغيل من محرر الأكواد

يؤدي النقر فوق **Run** في محرر الأكواد إلى فتح حوار بدلاً من إطلاق تشغيل عمياء مشفرة:

- **Trading account** (مطلوب) — حساب cTrader الذي يتصل به cBot.
- **Parameter set** (اختياري) — اختر مجموعة موجودة، أو اتركها فارغة للتشغيل بـ **قيم المعاملات الافتراضية** لـ cBot. يؤدي زر **+** بجانب المحدد إلى إنشاء مجموعة معاملات جديدة مضمنة (انظر أدناه) وتحديدها.
- **Symbol / Timeframe** الافتراضي إلى `EURUSD` / `h1` ويمكن تغييره؛ **Cancel** أو **Run**.

عند **Run** يحفظ المحرر + ينشئ المصدر الحالي، يبدأ المثيل على الحساب المختار بالمعاملات المختارة، ثم يعقب سجلات الحاوية المباشرة. (يعيد تدفق السجل ملف تعريف الارتباط للمصادقة للمستخدم الموقع إلى مركز `/hubs/logs` SignalR، لذا فإنه يتصل بدلاً من الفشل مع `Invalid negotiation response received`.)

## مجموعات المعاملات

**مجموعة معاملات** هي مجموعة مسماة وقابلة لإعادة الاستخدام من تجاوزات معاملات cBot المخزنة كائن JSON مسطح يعين كل اسم معامل إلى قيمة عددية، على سبيل المثال `{"Period": 14, "Label": "trend"}`. في وقت التشغيل/الاختبار يتم تحويله إلى ملف `params.cbotset` لـ cTrader
(`{ "Parameters": { … } }`). يمكنك إنشاء/تحرير مجموعة كـ JSON خام من حوار **مجموعات المعاملات** لـ cBot أو مضمناً من حوار التشغيل.

كل مجموعة معاملات **تنتمي إلى cBot**: يسرد حوار New Parameter Set جميع cBots الخاصة بك و **يجب عليك اختيار واحد** — يتم منع الإنشاء حتى يتم تحديد cBot. **اسم المجموعة فريد لكل cBot**:
إنشاء أو إعادة تسمية مجموعة بأسماء مجموعة أخرى من نفس cBot بالفعل مستخدمة يتم رفضها (خطأ واضح في الحوار، `409 Conflict` في API). قد تُعاد استخدام نفس الاسم على **cBot مختلف**.

يتم **التحقق من صحة** JSON عند الحفظ: يجب أن يكون أول كائن مسطح واحد جميع قيمه عددية
(string / number / bool). لا يتم قبول جذر غير كائن أو مصفوفة أو كائن متداخل أو قيمة `null` أو JSON معطل (خطأ واضح في الحوار، `400 Bad Request` في API). يُسمح بكائن فارغ `{}` ويعني "لا تجاوزات".

## ملاحظات CLI لـ cTrader Console

تحتاج الاختبارات إلى `--data-mode` (افتراضي `m1`)، التواريخ كـ `dd/MM/yyyy HH:mm`، و
وسيطة `params.cbotset` JSON موضعية؛ يرفض `run` العلم `--data-dir` (اختبار فقط). انظر
`ContainerCommandHelpers`.

## العقد والقياس

يتم قياس سعة التنفيذ بإضافة وكلاء العقد (التسجيل الذاتي + النبض). انظر
[node discovery](../operations/node-discovery.md) و [scaling](../deployment/scaling.md).

## حساب التداول مطلوب

يتطلب تشغيل أو اختبار cBot حساب تداول cTrader للاتصال به. حتى تضيف واحداً تحت
**Trading accounts**، يتم تعطيل أزرار **Run New cBot** / **Backtest New cBot** (مع تلميح أداة) وتعرض الصفحة موجه يرتبط بإعداد الحساب — لن تصادف خطأ خام
`stream connect failed` من إنسان آلي بدون حساب.
