# تحليل_تكلفة_المعاملة_(TCA)

alpha_التنفيذ_صغير_لكل_صفقة_وعظيم_عبر_آلاف_منها_—_إنه_جزء_كبير_من_كيف_
تحتفظ_البنوك_ومكات_التداول_oprop_بحافتها._TCA_يقيس_مدى_ابتعاد_السعر_
الذي_حقققته_فعلياً_عن_السعر_عندما_ *قررت*_التداول.

افتح **cBots → Execution Cost** (`/quant/tca`).

## ماذا_يقيس

بالنظر_إلى_ **سعر_الوصول_(القرار)**،_ **الجانب**،_و **الملءات** (السعر_×_الكمية)_،
يُبلغ:

- **متوسط_سعر_الملء_(VWAP)** —_السعر_الموزون_بالحجم_الذي_حصلته_فعلياً.
- **الانزلاق_(bps)** —_الانحراف_من_الوصول_إلى_VWAP_بالنقاط_الأساسية_،
  **مووقيع_لذا_الرقم_الإيجابي_هو_تكلفة** (شراء_فوق_الوصول_أو_بيع_تحته)_
  والرقم_السالب_هو_تحسن_في_السعر.
- **عجز_التنفيذ** —_هذه_التكلفة_معبراً_عنها_بمصطلحات_السعر_×_الكمية:_المال_
  الذي_كلّفه_الانحراف_على_هذا_الأمر.

```http
POST /api/quant/tca
{ "arrivalPrice": 1.1000, "side": "Buy",
  "fills": [ { "price": 1.1010, "quantity": 100 }, { "price": 1.1020, "quantity": 100 } ] }
```

## التقطيع_الذكي_(Almgren-Chriss)

بعد_قياس_التكلفة،_cMind_يمكنها_التخطيط_لأمر_كبير_لـ *تصغير*_التكلفة._
 **cBots → Execution Schedule** (`/quant/execution`)_يبني_ **جدول_تنفيذ_أمثل_Almgren-Chriss**:
 بالنظر_إلى_الكمية_الإجمالية_،_عدد_الشرائح_،_تجنب_المخاطر_،_التقلب_والتأثير_
 السوق_المؤقت_،_يُرجع_الحجم_التداول_في_كل_شريحة._تجنب_المخاطر_الأعلى_
  **يُحمّل_أماماً** _الجدول_(يقطع_مخاطر_التوقيت)؛_تجنب_المخاطر_صفر_يُسوّي_
 إلى_ **TWAP**_متساوٍ._الشرائح_دائماً_مجموع_الإجمالي.

```http
POST /api/quant/execution-schedule
{ "totalQuantity": 100, "slices": 5, "riskAversion": 2, "volatility": 0.02, "temporaryImpact": 0.1 }
```

## لماذا_هو_موثوق

كود_مجال_نقي_وحتمي_(`Core.Execution`)_بدون_اعتماد_على_البنية_التحتية_
ولا_مكالمات_خارجية_—_مختبر_بالوحدة_لعلامة_تكلفة_الشراء/البيع_،
تحسن_السعر،_الانزلاق_صفري،_تجميع_VWAP_،_وحراسة_المدخلات._هذا_هو_نصف_
قياس_جودة_التنفيذ؛_إنه_نفس_مقياس_العجز_الذي_يستخدمه_محرك_النسخ_
للمحاكمة_(و،_مع_التقطيع_الذكي_،_لتقليل)_تكلفة_الأوامر_المُنسوخة.
