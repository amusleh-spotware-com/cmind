# Commitment of Traders (COT)

cMind มีรายงาน **Commitment of Traders** ที่สร้างมาแล้ว — การแบ่งรายละเอียดของ CFTC รายสัปดาห์ว่าใครกำลังซื้อและขาย
ในตลาดฟิวเจอร์สของสหรัฐฯ (ผู้ป้องกันความเสี่ยงทางการค้า, นักเก็งกำไรขนาดใหญ่, กองทุน), พร้อมแผนภูมิเชิงโต้ตอบ
ที่มีประวัติศาสตร์, **ดัชนี COT** ที่ทำให้เป็นมาตรฐาน, REST API ที่ผ่านการตรวจสอบสำหรับ cBots และ MCP tools 
สำหรับไคลเอนต์ AI ข้อมูลมาจาก **ชุดข้อมูล Socrata สาธารณะของ CFTC** โดยตรง — ไม่มีคีย์ API, ไม่มีตัวรวบรวมข้อมูล 
เช่นเดียวกับปฏิทินเศรษฐศาสตร์ เป็นโมดูลที่แยกจากกันซึ่งสามารถปิดใช้งานได้โดยไม่มีผลกระทบต่อแกนการซื้อขาย

## สิ่งที่มันให้

- **ทั้งสามตระกูลรายงาน, เฉพาะฟิวเจอร์สและรวมกันฟิวเจอร์ส+ตัวเลือก:**
  - **มรดก** — ไม่ใช่เชิงพาณิชย์ (นักเก็งกำไรขนาดใหญ่), เชิงพาณิชย์ (ผู้ป้องกันความเสี่ยง), ไม่สามารถรายงาน
  - **แยกแยะ** — ผู้ผลิต/พ่อค้า, ผู้ค้าสวอป, เงินที่จัดการ, ผู้รายงานอื่น ๆ
  - **ผู้ค้าในฟิวเจอร์สทางการเงิน (TFF)** — ผู้จัดการ, ผู้จัดการสินทรัพย์, กองทุนที่มีการใช้คันโยก, ผู้รายงานอื่น ๆ
- **แค็ตตาล็อกตลาดที่คัดสรรมา** — ผู้เล่นฟอเร็กซ์, ทองคำ/เงิน/ทองแดง, ดิบและก๊าซธรรมชาติ, พันธบัตร, ดัชนี
  ทุน, คริปโตและธัญพืช/สินค้าที่นุ่ม — แต่ละรายการจับคู่กับรหัสสัญญา CFTC ที่เสถียรและ, ในกรณีที่ชัดเจน,
  กับสัญลักษณ์ที่สามารถซื้อขายได้ (เช่น Euro FX → `EURUSD`, ทองคำ → `XAUUSD`)
- **ดัชนี COT (0–100)** — ตำแหน่งสุทธิของนักเก็งกำไรในปัจจุบันอยู่ภายในช่วงประวัติศาสตร์ของมัน
  (ค่าเริ่มต้น ~3 ปีย้อนกลับ) การอ่านค่าใกล้กับสุดขั้วบ่งบอกถึงการวางตำแหน่งที่แออัดซึ่งมักนำหน้าการกลับตัว;
  รายงานกำหนดป้ายกำกับ **สุดขั้วที่ยาว** (≥80) หรือ **สุดขั้วที่สั้น** (≤20)
- **ความถูกต้องของจุดเวลา** รายงานรายสัปดาห์วัดในวันอังคาร แต่กลายเป็นสาธารณะเพียงในวันศุกร์ที่ตามมา;
  การอ่านแต่ละครั้งให้เกียรติแก่ช่วงเวลารับออกแบบนั้น ดังนั้นสัญญาณการวางตำแหน่งที่ได้รับการทดสอบในอดีตจึงไม่
  เห็นรายงานก่อนที่จะเผยแพร่ (ไม่มีการมองไปข้างหน้า)

## การใช้หน้า

เปิด **Commitment of Traders** จากการนำทางด้านซ้าย เลือก **ตลาด**, **ประเภทรายงาน** (มรดก /
แยกแยะ / การเงิน) และสลับ **ฟิวเจอร์ส + ตัวเลือก** เพื่อสลับระหว่างเฉพาะฟิวเจอร์สและตัวแปรรวม 
หน้านี้แสดง:

- **การวางตำแหน่งสุทธิเมื่อเวลาผ่านไป** — แผนภูมิเส้นแบบโต้ตอบของตำแหน่งสุทธิของหมวดหมู่ผู้ค้าแต่ละรายการ
  (ยาว - สั้น) ตลอดหน้าต่างประวัติศาสตร์
- **ดัชนี COT** — แผนภูมิเส้นของดัชนี 0–100 พร้อมการอ่านค่าล่าสุดและป้ายกำกับขั้นสูง
- **ภาพรวมล่าสุด** — ตารางยาว / สั้น / สุทธิ / % ของดอกเบี้ยเปิดต่อหมวดหมู่ผู้ค้า บวกกับ
  ดอกเบี้ยที่เปิดทั้งหมดและวันที่ของรายงาน

แต่ละแผนภูมิมีปุ่มแถบเครื่องมือ **ซูมเข้า/ออก** (และรีเซ็ต) และคุณสามารถลากไปยังแกนเวลาเพื่อซูมได้ **ส่งออก CSV** ดาวน์โหลดประวัติศาสตร์รายสัปดาห์ที่สมบูรณ์ของตลาดและประเภทรายงานที่เลือกไว้เป็นไฟล์ที่พร้อมสำหรับสเปรดชีต ใช้ **เปรียบเทียบตลาด** เพื่อวางตลาดหลายตลาดบนแผนภูมิเดียว — แผนภูมิเปรียบเทียบวาดตำแหน่งสุทธิของนักเก็งกำไรและดัชนี COT ของแต่ละตลาดที่เลือกไว้ข้างเคียง เพื่อให้คุณสามารถอ่านตำแหน่งระหว่างตลาดได้ในทันที

## วิธีการไหลของข้อมูล

ฐานข้อมูลคือแคช คนงานการบริหารรายสัปดาห์ดึงชุดข้อมูล CFTC หกชุดสำหรับตลาดที่ติดตามไว้ upserts แค็ตตาล็อกตลาด และเพิ่มรายงานใหม่แต่ละฉบับ **idempotently** (การรันใหม่ไม่เคยซ้ำภาพถ่าย) นอกจากนี้ข้อมูลจะ **โหลดตามความต้องการ**: ครั้งแรกที่ขอตลาดจะดึงมาจากแหล่งข้อมูล CFTC และเก็บไว้ และคำขออื่น ๆ ทั้งหมดจะได้รับบริการโดยตรงจากฐานข้อมูล แคช **รีเฟรชเมื่อรายงานรายสัปดาห์ใหม่ออกมา** — เมื่อรายงานที่เก็บไว้ล่าสุดมีอายุมากกว่าหนึ่งสัปดาห์ คำขออื่น ๆ จะดึงและเพิ่มข้อมูลล่าสุดโดยโปร่งใส (จำกัดเพื่อไม่ให้แหล่งข้อมูลได้รับผลกระทบ) การโหลดครั้งแรกป้อนประวัติศาสตร์ของหลายปี; การหยุดของแหล่งข้อมูลจะเสื่อมสภาพเพื่อให้บริการข้อมูลแคชที่ดีที่สุด ทุกอย่างทำงานจากกล่องนั้นโดยไม่มีคีย์; โทเค็นแอปพลิเคชัน Socrata ที่เป็นตัวเลือกจะเพียงแค่ยกขีด จำกัด อัตรา

## การกำหนดค่า

ทั้งหมด ปุ่มอยู่ภายใต้ `App:Cot` (ดู [สวิตช์คุณสมบัติ](./feature-toggles.md) และ
[การตั้งค่าเจ้าของป้ายขาว](./white-label-owner-settings.md)):

| ปุ่ม | ค่าเริ่มต้น | วัตถุประสงค์ |
|-----|---------|---------|
| `IngestionEnabled` | `true` | ว่าคนงานการบริหารรายสัปดาห์ทำงาน |
| `PollInterval` | `6h` | ความถี่ของคนงานสำรวจชุดข้อมูล CFTC |
| `BackfillYears` | `5` | ปีของประวัติศาสตร์ดึงมาในการรันครั้งแรก |
| `ReconcileLookbackWeeks` | `4` | สัปดาห์ล่าสุดสอดคล้องกันอีกครั้งในแต่ละรอบเพื่อจับการแก้ไข |
| `SocrataAppToken` | — | โทเค็นที่เป็นตัวเลือกที่ยกขีด จำกัด อัตรานิรนาม |
| `CotIndexLookbackWeeks` | `156` | รายงานรายสัปดาห์ที่ใช้เป็นช่วง COT-index (~3 ปี) |

## การล็อก

การมองเห็นคือประตูสองชั้น เหมือนกับปฏิทินเศรษฐศาสตร์: ประตูการสร้าง `App:Branding:EnableCot`
(ระดับการสร้าง) **และ** สลับเวลาทำงาน `App:Features:Cot` การเชื่อมต่อ nav ขาหน้า, หน้า, REST API 
และ MCP tools หายไปทั้งหมด (API ส่งคืน `404`) เนื่องจากแหล่งข้อมูลไม่มีคีย์ จึงไม่มีการล็อกคีย์แหล่งข้อมูล — 
เปิดใช้งานหมายถึงมองเห็น

## สำหรับนักพัฒนา

- โดเมน: `Core.Cot` — รวมตัวของ `CotMarket` และ `CotReport`, วัตถุมูลค่า `CotPositions`,
  บริการโดเมน `CotIndexCalculator`, และพอร์ต `ICotReports` / `ICotSource`
- สถาปัตยกรรม: `Infrastructure.Cot` — ตัววิเคราะห์ต่อต้านการคอร์รัปชันของ `CftcSocrataSource`, 
  ประตูอัตรา, บริการเขียนเพียงเพิ่มเติม, อ่านด้าน, และคนงานบริหารรายสัปดาห์ (EF schema `cot`)
- cBot & การเข้าถึง AI: [COT cBot API](./cot-cbot-api.md) (REST, `market:read` JWT) และ MCP tools
  `CotMarkets`, `CotLatest`, `CotHistory`, `CotHealth`.
