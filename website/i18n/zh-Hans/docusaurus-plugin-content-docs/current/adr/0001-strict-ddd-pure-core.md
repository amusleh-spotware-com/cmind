# 0001 — 严格 DDD 与纯 `Core`

## 背景

这个应用处理真实资金。散布在端点、后台服务和 Razor 组件中的业务规则会腐烂成不可测试的、不一致的行为——正是这样的地方，一个错误可能让用户损失资本。

## 决策

域逻辑存在**于聚合体、值对象和域服务**上，位于 `src/Core`，它编译时**零基础设施依赖**（无 EF、HttpClient、Docker 或 ASP.NET）。端点、MCP 工具、组件和 `BackgroundService` **组织** ——它们从不决策。规则：

- 无公开设置器；状态变更通过保护不变量的意图显现方法进行。
- 聚合体通过**强 ID** 引用彼此，而不是导航属性。
- 一个 `SaveChanges` 变更**一个**聚合体；跨聚合体流使用域事件。
- 跨越域边界的基元包装在值对象中。
- 不变量违规抛出 Core `DomainException`，不是框架异常。

## 后果

- 域规则可在没有数据库或 web 主机的情况下进行单元测试。
- `Core` 纯度由 `ArchitectureGuardTests` 机器强制执行，如果破坏会使构建失败。
- 与贫血模型相比有更多仪式（值对象、强 ID、域事件）——这是保持资金流动规则正确且集中的有意成本。
