# Создание и тестирование cBots

Создавайте, запускайте, тестируйте cBots cTrader (C# **и** Python, оба на .NET) из встроенного IDE Monaco
в браузере, запускайте на официальном образе `ghcr.io/spotware/ctrader-console`.

## Создание

- Страница **Builder** содержит редактор Monaco; `CBotBuilder` компилирует проект с помощью
  `dotnet build` **в одноразовом контейнере** (`AppOptions.BuildImage`, рабочая директория монтируется
  в `/work`), поэтому ненадежные целевые объекты MSBuild пользователя не могут получить доступ к хосту. Восстановление NuGet кэшируется
  между сборками через общий том. Веб-хост нуждается в доступе к сокету Docker.
- Стартовые шаблоны C# и Python находятся в `src/Nodes/Builder/Templates/`.

## Запуск и тестирование

- **Instances** = иерархия состояния TPH (`Run`/`Backtest` × `Pending`/`Scheduled`/`Starting`/
  `Running`/`Stopping`/`Stopped`/`Failed`). Переход заменяет сущность (изменение id),
  id контейнера сохраняется.
- `NodeScheduler` выбирает наименее загруженный подходящий узел; `ContainerDispatcherFactory` маршрутизирует на
  удаленный HTTP агент узла или локальный диспетчер Docker.
- Полировщики завершения согласуют вышедшие контейнеры (контейнеры тестирования автоматически выходят через
  `--exit-on-stop`); отчет присутствует → завершено (сохранить `ReportJson`), отсутствует → не удалось.
- Живые логи контейнера транслируются в браузер через SignalR; кривые справедливости тестирования анализируются из
  отчета и отображаются на диаграмме.

## Данные тестирования кэшируются по учетной записи

cTrader Console загружает историческое данные tick/bar в свою `--data-dir`. Эта директория —
**стабильный, постоянный кэш, ключи которого — торговая учетная запись** (номер ее счета) — монтируется из
диска узла по его собственному пути контейнера (`/mnt/data`), **отдельное, не вложенное монтирование** от
директории работы на экземпляр. Поэтому каждое тестирование на одном и том же счете **переиспользует** уже загруженные данные
вместо их повторной загрузки при каждом запуске. (Ранее
директория данных находилась под директорией работы на экземпляр, чей id изменяется при каждом запуске, что вынуждало свежую
загрузку при каждом тестировании.) Эфемерная директория работы на экземпляр по-прежнему содержит алгоритм, параметры, пароль
и отчет; кэш общих данных учитывается в использовании данных тестирования узлом и очищается действием очистки узла.

## Параметры тестирования

Диалог **Backtest** предоставляет все параметры, которые принимает CLI тестирования cTrader Console, поэтому вам никогда
не нужно трогать командную строку:

- **From / To** — окно тестирования (`--start` / `--end`).
- **Data mode** — `m1` (бары в 1 минуту) или `tick` (`--data-mode`).
- **Starting balance** — по умолчанию `10000` (`--balance`). **Баланс 0 не размещает сделки и заставляет
  cTrader выводить пустой отчет, затем падает** ("Message expected"), поэтому ненулевой баланс
  всегда отправляется.
- **Commission** и **Spread** (`--commission` / `--spread`, спред в пипсах).
- **Advanced options** — свободное поле `name=value` на строку для любого другого параметра тестирования, который поддерживает cTrader
  (например, `applyCommissionAutomatically=true`); каждая строка становится аргументом CLI `--name value`.

## Страница деталей экземпляра

При открытии экземпляра (`/instance/{id}`) отображается его текущее состояние, логи и — для тестирования — кривая справедливости.
**Заголовок вкладки браузера** отражает конкретный экземпляр (**имя cBot · вид · символ**, например
`TrendBot · Backtest · EURUSD`), поэтому вкладка live-run и вкладка тестирования различимы с первого взгляда.
Запуск и тестирование одного и того же cBot отслеживаются как отдельные **линии** (стабильный id линии, переносимый
через переходы состояния), поэтому страница следует ровно одному экземпляру и никогда не смешивает данные запуска с данными
тестирования.

## Элементы управления жизненным циклом экземпляра

Каждая строка экземпляра (и его страница деталей) имеет элементы управления, правильные для состояния. **Активный** экземпляр показывает
**Stop**; **терминальный** (Stopped / Completed / Failed) показывает **Start (▶)**, чтобы перезапустить его с
тем же cBot, учетной записью, символом, таймфреймом, набором параметров и образом (запуск перезапускается как запуск, тестирование как тестирование). Нажатие Stop отображает уведомление "Stopping…" и отключает значок до его завершения, и
новый запуск сразу же появляется в списке — без перезагрузки страницы.

Логи консоли **сохраняются при завершении экземпляра** — для запуска (при Stop) и для
**тестирования** (при завершении) — поэтому логи последнего запуска остаются просматриваемыми на странице деталей и,
через панель инструментов логов, **скопированы в буфер обмена** (значок Copy logs) или **загружены** (значок Download logs)
даже после того, как контейнер исчез. Оба работают с полным логом консоли экземпляра, а не только
видимой хвостовой частью.

**Загруженный** `.algo` никогда не создавался здесь, поэтому его столбец **Last Build** на странице cBots остается
пустым (он отображает время сборки только для cBots, которые вы создаете в браузере).

## Редактирование и повторный запуск остановленного экземпляра

**Остановленный** экземпляр (запуск или тестирование) имеет элемент управления **Edit** — значок на его строке в списке **и**
рядом с Start/Stop на его странице деталей — который открывает диалог, **предварительно заполненный** его текущей конфигурацией.
Вы можете изменить **торговую учетную запись, символ, таймфрейм, набор параметров и тег образа** (и для
тестирования **окно и все параметры тестирования** выше), затем **Save & start** перезапускает его с
новыми параметрами (заменяя остановленный экземпляр). Элемент управления **отключен, пока экземпляр активен** —
только остановленный экземпляр может быть отредактирован.

## Запуск из редактора кода

Нажатие **Run** в редакторе кода открывает диалог вместо вслепую запуска с жестко закодированными параметрами:

- **Trading account** (требуется) — учетная запись cTrader, к которой подключается cBot.
- **Parameter set** (опционально) — выберите существующий набор или оставьте его пустым для запуска с
  **значениями параметров по умолчанию** cBot. Кнопка **+** рядом с селектором создает новый набор параметров
  встроенно (см. ниже) и выбирает его.
- **Symbol / Timeframe** по умолчанию `EURUSD` / `h1` и могут быть изменены; **Cancel** или **Run**.

При нажатии **Run** редактор сохраняет + создает текущий источник, запускает экземпляр на выбранной учетной записи
с выбранными параметрами, затем отслеживает живые логи контейнера. (Поток логов пересылает cookie аутентификации
вошедшего пользователя на концентратор SignalR `/hubs/logs`, поэтому он подключается вместо того, чтобы не удаться с
`Invalid negotiation response received`.)

## Наборы параметров

**Набор параметров** — это именованный, переиспользуемый набор переопределений параметров cBot, сохраняемый как плоский объект JSON
сопоставляющий каждое имя параметра со скалярным значением, например `{"Period": 14, "Label": "trend"}`. При
запуске/тестировании он преобразуется в файл cTrader `params.cbotset`
(`{ "Parameters": { … } }`). Вы можете создать/отредактировать набор как сырой JSON из диалога **Parameter
sets** cBot или встроенно из диалога Run.

Каждый набор параметров **принадлежит cBot**: диалог New Parameter Set перечисляет все ваши cBots и вы
**должны выбрать один** — создание блокируется до тех пор, пока не будет выбран cBot. **Имя** набора уникально на cBot:
создание или переименование набора на имя, которое уже использует другой набор того же cBot, отклоняется (четкая
ошибка в диалоге, `409 Conflict` в API). Одно и то же имя может быть переиспользовано на **другом** cBot.

JSON **проверяется** при сохранении: он должен быть единственным плоским объектом, значения которого — все скаляры
(string / number / bool). Не-объектный корень, массив, вложенный объект, значение `null` или неправильный
JSON отклоняется (четкая ошибка в диалоге, `400 Bad Request` в API). Пустой объект `{}`
разрешен и означает "нет переопределений".

## Заметки CLI cTrader Console

Тестирования нужны `--data-mode` (по умолчанию `m1`), даты как `dd/MM/yyyy HH:mm` и
`params.cbotset` JSON позиционный аргумент; `run` отклоняет `--data-dir` (только для тестирования). Смотрите
`ContainerCommandHelpers`.

## Узлы и масштабирование

Емкость выполнения масштабируется путем добавления агентов узлов (самостоятельная регистрация + heartbeat). Смотрите
[node discovery](../operations/node-discovery.md) и [scaling](../deployment/scaling.md).

## Требуется торговая учетная запись

Запуск или тестирование cBot требует торговую учетную запись cTrader для подключения. Пока вы не добавите ее в
**Trading accounts**, кнопки **Run New cBot** / **Backtest New cBot** отключены (с
подсказкой) и страница отображает подсказку со ссылкой на настройку учетной записи — вы больше не получите ошибку
`stream connect failed` от бота без учетной записи.
