# Backtest Integrity Lab

リテールプラットフォームはバックテストのSharpeまたは純利益を 보여주고そこで止まります。機関は生のバックテストを信頼しません — *選択バイアスと試行した設定数の補正*を生き延びるかどうか尋ねます。Backtest Integrity LabがそのチェックをcMindにもたらします。これは**決定論的数学**（AIなし、外部呼び出しなし）なので、判定は再現可能ですべての数字が説明可能です。

**cBots → Integrity**（`/quant/integrity`）で開きます。

## 何が計算されるか

リターンシリーズ（またはエクイティ/バランスカーブ）とそれに到達するために試行したパラメータセットの数が与えられると、アナライザーは報告します：

- **Sharpeレシオ** — 期間ごとと年率（平方根時間）。
- **Probabilistic Sharpe Ratio（PSR）** — トラックレコードの長さ、歪度、尖度を考慮して、*真の*Sharpeがベンチマークを打っている confidence（Bailey & López de Prado、2012）。短いまたはファット尾の記録はそれを低下させます。
- **Deflated Sharpe Ratio（DSR）** — *N個のランダム試行の最良*に対するベンチマークに対して測定されたPSR（Nullの下、FALSE戦略定理）。試行した設定が多いほどバーが高くなります — これが過学習を検出します。
- **平均リターンのt統計量。** Harvey、 Liu & Zhuに続き、本当のedgeは**t ≥ 3.0**をクリアする必要があります（教科書の2.0ではなく）。
- **リターンの歪度/尖度**、PSR/DSR補正に使用。

## 判定

| 判定 | 意味 | ルール |
|---|---|---|
| **Robust** | edgeは実行した試行を生き延びます。 | DSR ≥ 95% **かつ** PSR ≥ 95% **かつ** |t| ≥ 3.0 |
| **Fragile** | 統計的に生存しているが説得力がない — これだけでサイズアップしないでください。 | その間 |
| **Overfit** | おそらく選択バイアスのアーティファクトであり、本当のedgeではない。 | DSR < 90% |

すべての結果には平易な英語の理論的根拠が伴うため、「なぜ」が隠されることは決してありません。

## 試行間でのバックテスト過学習の確率

試行数*を与える*ことは良い；あなたが試行した*各設定の実在するアウトオブサンプルシリーズ*を与える*ことは更好。オプションの**試行グリッド**に貼り付ける（1行に1シリーズ）と、cMindは**Combinatorially-Symmetric Cross-Validation**（Bailey、Borwein、López de Prado & Zhu、2015）を実行：観察をグループに分割し、半分のサンプルを選択するすべての方法についてサンプル内最良の設定を選択し、勝者がアウトオブサンプルで下半分の底部にあるかどうかをチェック。**バックテスト過学習の確率（PBO）**は、勝者が汎化できなかった分割の割合です。PBOが0に近い場合、最良の設定は真有功に最良です；PBOが0.5以上の場合、選択プロセスはノイズを選択しています — 勝者がどのように見えたかに関係なく判定は**Overfit**になります。

```http
POST /api/quant/pbo
{ "trials": [[...], [...], ...] }
```

ネイティブcTrader Consoleオプティマイザーが到着すると、cMindはここにフル試行 поверхностルを自動的に供給します。

## 試行 — 重要な数字

`Trials`は**この設定を選ぶ前にテストしたパラメータセットの数*です。1つの戦略をテストすることと、1万をテストして最良を保持することは大きく異なる：2番目は偶然に高いサンプル内Sharpeを製造します。正直な試行数を供給することは要点です — デフレを上げ、「素晴らしい」バックテストを**Overfit**に移動させる可能性があります。ネイティブcTrader Consoleオプティマイザーが到着すると、cMindはスイープの実際のグリッドサイズを自動的に供給します。

## 入力

- **periodic returns** — 期間ごとの1つの数字（例： `0.01` = +1%）。最低2つ。
- **エクイティ/バランスカーブ** — cMindは連続単純リターンを派生させます。
- または完了したバックテストで直接実行： `POST /api/quant/integrity/backtest/{instanceId}`保存されたレポートのエクイティカーブを読み取ります。

## API

```http
POST /api/quant/integrity
{ "returns": [0.006, 0.004, 0.006, ...], "trials": 250 }
```

判定、すべてのメトリクス、理論的根拠を返します。 `POST /api/quant/integrity/backtest/{id}`所有する完了したバックテストで同じ分析を実行します。

## これが信頼できる理由

統計はドメインコア（`Core.Quant`）の純粋な関数であり、インフラ依存ゼロ — ネットワークブリップで downed にすることはできず、公開式に対する金種ベクトルユニットテストでピン留めされています。正規CDF/逆数は閉形式近似（Abramowitz-Stegun / Acklam）なので、同じ入力は常に同じ判定を 生成します。
