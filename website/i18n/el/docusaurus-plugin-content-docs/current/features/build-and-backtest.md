# Build & backtest cBots

Δημιουργήστε, εκτελέστε, κάντε backtest cTrader cBots (C# **και** Python, και τα δύο .NET) από το in-browser Monaco
IDE, εκτελέστε στο επίσημο `ghcr.io/spotware/ctrader-console` image.

## Build

- **Builder** page φιλοξενεί το Monaco editor; `CBotBuilder` μεταγλωττίζει το project με
  `dotnet build` **σε throwaway container** (`AppOptions.BuildImage`, work dir bind-mount
  στο `/work`), ώστε τα untrusted user MSBuild targets να μην έχουν πρόσβαση στο host. Η NuGet restore κρυώνεται
  διαδρόμων builds μέσω shared volume. Το Web host χρειάζεται Docker socket access.
- C# + Python starter templates ζουν στο `src/Nodes/Builder/Templates/`.

## Run & backtest

- **Instances** = TPH state hierarchy (`Run`/`Backtest` × `Pending`/`Scheduled`/`Starting`/
  `Running`/`Stopping`/`Stopped`/`Failed`). Η transition αντικαθιστά την entity (id change),
  container id μεταφέρεται.
- `NodeScheduler` επιλέγει το λιγότερο φορτωμένο eligible node; `ContainerDispatcherFactory` δρομολογεί
  στον remote node HTTP agent ή local Docker dispatcher.
- Completion pollers συμφιλιώνουν exited containers (backtest containers self-exit μέσω
  `--exit-on-stop`); report present → completed (store `ReportJson`), missing → failed.
- Τα live container logs ρέουν στο browser μέσω SignalR; τα backtest equity curves αναλύονται από
  report + charted.

## cTrader Console CLI σημειώσεις

Τα Backtests χρειάζονται `--data-mode` (default `m1`), dates ως `dd/MM/yyyy HH:mm`, και
`params.cbotset` JSON positional arg; `run` απορρίπτει `--data-dir` (backtest-only). Δείτε
`ContainerCommandHelpers`.

## Nodes & scale

Η execution capacity κλιμακώνεται με την προσθήκη node agents (self-register + heartbeat). Δείτε
[node discovery](../operations/node-discovery.md) και [scaling](../deployment/scaling.md).

## Εκτέλεση από τον επεξεργαστή κώδικα

Κάνοντας κλικ στο **Εκτέλεση** στον επεξεργαστή κώδικα ανοίγει ένα παράθυρο διαλόγου αντί να ξεκινά μια τυφλή, σταθερά κωδικοποιημένη εκτέλεση:

- **Λογαριασμός συναλλαγών** (υποχρεωτικό) — ο λογαριασμός cTrader στον οποίο συνδέεται το cBot.
- **Σύνολο παραμέτρων** (προαιρετικό) — επιλέξτε ένα υπάρχον σύνολο ή αφήστε το κενό για εκτέλεση με τις **προεπιλεγμένες τιμές παραμέτρων** του cBot. Ένα κουμπί **+** δίπλα στον επιλογέα δημιουργεί ένα νέο σύνολο παραμέτρων επιτόπου (δείτε παρακάτω) και το επιλέγει.
- **Σύμβολο / Χρονικό πλαίσιο** είναι προεπιλεγμένα `EURUSD` / `h1` και μπορούν να αλλάξουν· **Άκυρο** ή **Εκτέλεση**.

Κατά την **Εκτέλεση** ο επεξεργαστής αποθηκεύει και χτίζει τον τρέχοντα πηγαίο κώδικα, ξεκινά το στιγμιότυπο στον επιλεγμένο λογαριασμό με τις επιλεγμένες παραμέτρους και στη συνέχεια παρακολουθεί τα ζωντανά αρχεία καταγραφής του κοντέινερ. (Η ροή καταγραφής προωθεί το cookie ελέγχου ταυτότητας του συνδεδεμένου χρήστη στον διανομέα SignalR `/hubs/logs`, ώστε να συνδέεται αντί να αποτυγχάνει με `Invalid negotiation response received`.)

## Σύνολα παραμέτρων

Ένα **σύνολο παραμέτρων** είναι ένα ονομασμένο, επαναχρησιμοποιήσιμο σύνολο παρακάμψεων παραμέτρων του cBot, αποθηκευμένο ως επίπεδο αντικείμενο JSON που αντιστοιχίζει κάθε όνομα παραμέτρου σε μια βαθμωτή τιμή, π.χ. `{"Period": 14, "Label": "trend"}`. Κατά την εκτέλεση/backtest μετατρέπεται στο αρχείο cTrader `params.cbotset` (`{ "Parameters": { … } }`). Μπορείτε να δημιουργήσετε/επεξεργαστείτε ένα σύνολο ως ακατέργαστο JSON από το παράθυρο διαλόγου **Σύνολα παραμέτρων** του cBot ή επιτόπου από το παράθυρο διαλόγου Εκτέλεση.

Το JSON **επικυρώνεται** κατά την αποθήκευση: πρέπει να είναι ένα μόνο επίπεδο αντικείμενο του οποίου όλες οι τιμές είναι βαθμωτές (συμβολοσειρά / αριθμός / bool). Μια ρίζα που δεν είναι αντικείμενο, ένας πίνακας, ένα ένθετο αντικείμενο, μια τιμή `null` ή κακοσχηματισμένο JSON απορρίπτεται (σαφές σφάλμα στο παράθυρο διαλόγου, `400 Bad Request` στο API). Ένα κενό αντικείμενο `{}` επιτρέπεται και σημαίνει «καμία παράκαμψη».

## Στοιχεία ελέγχου κύκλου ζωής στιγμιότυπου

Κάθε γραμμή στιγμιότυπου (και η σελίδα λεπτομερειών της) έχει στοιχεία ελέγχου σύμφωνα με την κατάσταση. Ένα **ενεργό** στιγμιότυπο εμφανίζει **Διακοπή**· ένα **τερματικό** (Διακομμένο / Ολοκληρωμένο / Απέτυχε) εμφανίζει **Εκκίνηση (▶)** για επανεκκίνηση με το ίδιο cBot, λογαριασμό, σύμβολο, χρονικό πλαίσιο, σύνολο παραμέτρων και εικόνα (μια εκτέλεση επανεκκινεί ως εκτέλεση, ένα backtest ως backtest). Κάνοντας κλικ στη Διακοπή εμφανίζεται μια ειδοποίηση «Διακοπή…» και απενεργοποιείται το εικονίδιο μέχρι να ολοκληρωθεί· μια νέα εκτέλεση εμφανίζεται αμέσως στη λίστα — χωρίς επαναφόρτωση της σελίδας.

Τα αρχεία καταγραφής κονσόλας **διατηρούνται όταν ένα στιγμιότυπο τερματίζεται** — τόσο για μια εκτέλεση (κατά τη διακοπή) όσο και για ένα **backtest** (κατά την ολοκλήρωση) — ώστε τα αρχεία καταγραφής της τελευταίας εκτέλεσης να παραμένουν ορατά στη σελίδα λεπτομερειών και με δυνατότητα λήψης μέσω του εικονιδίου **Λήψη αρχείων καταγραφής**, ακόμη και αφού εξαφανιστεί το κοντέινερ.
