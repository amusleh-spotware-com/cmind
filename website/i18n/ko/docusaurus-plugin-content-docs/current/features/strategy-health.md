# 전략 상태 & 알파 감쇠

모든 엣지는 감쇠한다 — 연구는 정량적 전략의 반감기가 몇 년에서 몇 개월로 붕괴했음을 명확히 보여주므로 *적응이 발견을 능가한다*. 전략 상태 모니터는 전략 자체의 수익 기록에서 엣지가 여전히 유효한지 알려준다.

**cBots → Strategy Health** (`/quant/health`)를 열자.

## What it does

수익 시리즈(또는 자산곡선, 가장 오래된 것부터)가 주어지면:

- 기록을 **이전**과 **최근** 절반으로 분할하고 Sharpe 비율을 비교한다;
- **CUSUM 변곡점** 스캔을 실행하여 평균이 가장 명확하게 이동한 관찰(체제 전환)을 찾으며, 편차가 통계적으로 주목할 만할 때만 보고한다;
- 판정을 반환한다:

| 판정 | 의미 |
|---|---|
| **Healthy** | 최근 성과가 이전 기록과 일치하거나 더 좋다. |
| **Degrading** | 최근 Sharpe가 이전 기록보다 현저히 약하다 — 주의 깊게 관찰한다. |
| **Decayed** | 최근 기간에서 엣지가 사실상 사라졌다 — 중지를 고려한다. |
| **Unknown** | 판단할 충분한 기록이 없다. |

- **백테스트 실행에서 직접 — 복사-붙여넣기 없음.** 모든 완료된 백테스트는 **Backtest** 목록 행과 인스턴스 상세 보기에 **Check strategy health** 아이콘을 표시한다; 한 번 클릭하면 해당 실행의 저장된 자산곡선에서 모니터를 실행하고 판정을 대화상자에 표시한다. 아이콘은 백테스트가 완료되고 보고서를 생성할 때까지 비활성화되므로 죽은 컨트롤이 아니다. 내부적으로 이는 `POST /api/quant/health/backtest/{instanceId}`이며, 저장된 보고서의 자산곡선을 읽는다.

```http
POST /api/quant/health
{ "returns": [...] }   // or { "equity": [...] }
```

## Why it is reliable

인프라 의존성이 없고 외부 호출이 없는 순수하고 결정론적인 도메인 코드(`Core.Health`)이다 — 감쇠, 악화, 건강함 및 너무 짧은 경우와 변곡점 위치화에 대해 단위 테스트된다. 자율 에이전트를 지원하는 항상 켜진 상태 검사에 대한 수동 동반자이다: 동일한 통계가 엣지가 퇴색되는 라이브 전략의 위험을 제거하는 회로 차단기를 구동한다.
