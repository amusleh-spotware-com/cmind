# سوق مزود النسخ (المرحلة 4)

دليل قابل للتصفح لاستراتيجيات النسخ. يقوم المزود بـ **نشر** ملف تعريف النسخ كقائمة مع شارة **verified-live** (حساب مصدر الاستراتيجية تتاجر بأموال حقيقية، وليس عرض توضيحي) بالإضافة إلى رسم الأداء. يتصفح المتابعون السوق، ترتيب حسب درجة الأداء المتوقعة من بيانات logs execution-transparency.

## النموذج

- `CopyProviderListing` = إجمالي: `UserId`, `ProfileId`, display name, description, performance fee, `VerifiedLive`, `Published` + `PublishedAt`. قائمة واحدة لكل ملف تعريف (فهرس فريد).
- **Verified-live** مشتق في وقت النشر من ملف التعريف المصدر `TradingAccount.IsLive` — لا يمكن للمزود التأكيد ذاتياً.
- إحصائيات الأداء **لم يتم تخزينها على القائمة** — نموذج قراءة الإسقاط على سجل logs execution-transparency (معدل الملء، متوسط الكمون، متوسط الانزلاق المحقق)، لذا تعكس السوق دائماً جودة التنفيذ الحية.

## الترتيب

`CopyEndpoints.MarketplaceScore(fillRate, avgLatencyMs, avgSlippagePoints, verifiedLive)` → درجة 0–100: معدل الملء يسيطر (×60)، الكمون المنخفض + الانزلاق المنخفض إضافة (×20 لكل منهما)، شارة verified-live إضافة مكافأة الثقة الصغيرة. حتمي + رتيب، لذا الطلب مستقر.

## API

- `POST /api/copy/profiles/{id}/publish` — نشر/تحديث قائمة ملف التعريف (`DisplayName`, `Description`, `PerformanceFeePercent`); verified-live مجموعة من حساب المصدر.
- `DELETE /api/copy/profiles/{id}/publish` — إلغاء النشر.
- `GET /api/copy/marketplace` — جميع القوائم المنشورة، مرتبة، كل واحدة مع ملخص الأداء (executions، معدل الملء، متوسط الكمون، متوسط الانزلاق، score) + شارة verified-live.

## الاختبارات

- **الوحدة** (`CopyProviderListingTests`) — ثابتات الإجمالي: display name مطلوب؛ نشر تعيين الطابع الزمني؛ إلغاء النشر يخفي؛ تحديث استبدال عرض الحقول + الرسم + الشارة.
- **التكامل** (`CopyMarketplaceTests`، Postgres حقيقي) — قائمة منشورة مثابت مع شارة؛ قائمة واحدة لكل ملف تعريف (فهرس فريد)؛ درجة الترتيب تفضل الموردين المتحققة/العالية الملء.

لا يتأثر مضيف النسخ (قوائم + نموذج قراءة فقط)، لذا حزمة الضغط DST النسخ لم تتأثر.
