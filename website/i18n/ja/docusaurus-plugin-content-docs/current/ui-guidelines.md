# UI デザインガイドライン — 必須

このアプリの**すべて**の新しいまたは変更された UI（Blazor ページ、ダイアログ、コンポーネント）のバインディング。これは `CLAUDE.md` で参照されるソースです。ルールがあなたをブロックしたら、停止して尋ねてください — UI をそれに違反するシップしないでください。`plans/ui-overhaul.md` にルーツ化されています。

## 1. モバイルファースト、常に

- **360–430px 電話用に最初にオーサリング**して、`min-width` メディアクエリ / MudBlazor ブレークポイント props で上向きに拡張します。`max-width` オーバーライドを持つデスクトップファーストは決してしません。
- **幅 320–1920px のいずれでも水平スクロールなし。** コンテンツがビューポートより広い場合、それはバグです。
- タッチターゲット ≥ **44px** (`var(--app-touch-target)`)。テキスト入力 ≥ 16px フォント（iOS フォーカスズーム停止）。
- ノッチを尊重する: `env(safe-area-inset-*)` を使用します。ビューポートはすでに `viewport-fit=cover` を設定しています。
- `prefers-reduced-motion` を尊重してください — 本質的な情報はアニメーションによってのみ伝達されることはありません。

## 2. デザイントークン — ハードコード値なし

- すべての色/半径/スペーシングは **デザイントークン**から来ます: MudBlazor テーマ（`Web/Components/Theme.cs`）+
  `Web/Branding/BrandingCss.cs` で発行された CSS カスタムプロパティ（`var(--app-primary)`、
  `--app-surface`、`--app-border`、`--app-text*`、`--app-radius`、...）。
- **コンポーネントまたは CSS ルールに 16 進色、半径、またはブランド文字列をハードコードしません。** トークンを読んでください。
  トークンはホワイトラベル `BrandingOptions` から流れます。再販業者のパレットは UI に無料で到達する必要があります。
- 新しいブランドに影響を与える値 → トークン + ブランディングフィールドを追加します。インラインにしません。

## 3. レスポンシブレイアウト & データ

- **テーブルは電話でカードに折りたたむ。** すべての `MudTable` は `Breakpoint="Breakpoint.Sm"` を設定し、すべての
  `MudTd` には `DataLabel` があります。モバイル上に生テーブルなし。（テンプレート: `Components/Pages/Nodes.razor`。）
- グリッド: `MudItem xs="12" sm="6" md="4"` — 電話で全幅、上向きに複数列。
- フォーム単列モバイル; 大きなタップターゲット; 入力に `inputmode`/`autocomplete`; 金銭/パーセント用の数値/小数 inputmode。
- リスト/詳細の**ロード、空、エラー**状態を提供します — モバイルサイズ。
- モバイル**ボトムナビゲーション**（`Components/Layout/BottomNav.razor`）がプライマリ電話ナビです。グループ化されたドローワーが完全なメニューです。高トラフィック先にそこに追加します。≤5 アイテムに保ちます。

## 4. ダイアログ（作成/編集）

- すべての追加/作成/編集/新規アクション は **MudBlazor ダイアログ**（`IDialogService.ShowAsync<TDialog>`）を使用します。インラインページフォームはしません。ダイアログは `Web/Components/Dialogs/` に存在します。`[Parameter]` を公開します。ネストされた `public sealed record …Result(...)` を返します。リスト行アクション（開始/停止/削除）はインラインのままアイコンボタンです。
- 電話では、ダイアログは**全画面 / 全幅**であり、キーボード対応である必要があります。

## 5. インラインヘルプ — すべてのコントロール

- 非自明なオプション、選択、スイッチ、またはアクションは、**`<HelpTip Text="…" />`** を取得します。
  （`Components/HelpTip.razor`）— デスクトップでホバー、**モバイルでタップ**。テキストを `docs/` から出力して、ガイダンスが動作と同期を保ち続けます。同じコミットの両方を更新します。

## 6. ホワイトラベル

- 製品名、ロゴ、説明、サポート/会社、色、ファビコンはすべて `BrandingOptions` から来ます。
  それらを参照してください（`IBrandingThemeProvider` / `IOptionsMonitor<AppOptions>`）。リテラル「cMind」またはブランド色は参照しません。PWA マニフェスト、アイコン、テーマカラー、ログインヒーローはすべてブランド化されています。

## 7. PWA

- アプリはインストール可能です。マニフェストエンドポイント（`/manifest.webmanifest`）をブランド化、アイコンを提示します。
  （192/512/maskable + apple-touch）、サービスワーカーアプリシェルのみ（Blazor 回路/`_framework`/hubs には触れません）、オフラインページが機能しています。新しい静的ルート → マニフェスト `scope` を保ちます。
- Blazor Server にはライブ SignalR 回路が必要です → **インストール可能 + アプリシェル**、完全オフラインではなく。オフラインインタラクティビティを約束しません。

## 8. アクセシビリティ

- 入力のラベル、カスタムコントロールの `aria-*`、表示フォーカス、論理フォーカス順序。テーマはホワイトラベル可能なため、固定パレットではなく、アクティブテーマに対して**コントラスト**を確認します。

## 9. E2E — UI はテストされていない出荷なし（ブロック）

すべてのユーザー向けの変更が `tests/E2ETests` で Playwright E2E を出荷します。実際のユーザーのように駆動されます。**モバイルデバイスエミュレーション**とデスクトップ:

- 新しいルート → `PageSmokeTests` **と** `MobileLayoutTests` に追加します（レンダー、ボトムナビ、エラー UI なし）。
- テーブル/ページを変換 → そのルートをモバイル**オーバーフロウなし**セットに追加します。
- 新しいフロー → 現実的なモバイルジャーニー（作成/編集/保存往復）**と**不幸なパス
  （無効な入力、空のリスト、ロール毎の許可拒否）。
- 新しいヘルプチップ → タップで開くことをアサート（`HelpTipTests` パターン）。
- `AppFixture.NewAuthedMobilePageAsync` / `NewAnonymousMobilePageAsync`（デバイスエミュレーション）を使用します。
- `dotnet test` は「完了」の前に緑。エミュレートされた WebKit ≠ モバイル Safari — リアルデバイスゲーティングは別のリリースステップです。
