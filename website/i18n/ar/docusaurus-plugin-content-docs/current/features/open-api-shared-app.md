# تطبيق Open API المشترك وحدود المعدل

افتراضيًا يسجل كل مستخدم **الخاص به** تطبيق cTrader Open API تحت
**الإعدادات → Open API**. يمكن لمشغل ملصق أبيض (عادة وسيط cTrader أو موزع) بدلاً من ذلك
شحن **تطبيق Open API واحد مشترك لجميع المستخدمين** — لا أحد يسجل خاصته؛ الجميع
تفويض حساباتهم من خلال تطبيق المشغل الفردي.

## طريقتان لتوفير التطبيق المشترك

يتم توفير التطبيق المشترك إما من تكوين النشر **أو** من واجهة المستخدم لإعدادات المالك
(القيمة المحددة من المالك تفوز). توفيره مرة واحدة والوضع المشترك يتحول لكل شخص.

### 1. تكوين النشر (البذور عند بدء التشغيل)

```jsonc
"App": {
  "OpenApi": {
    "PublicBaseUrl": "https://cmind.yourbroker.com",   // عنوان URL عام قانوني من هذا النشر
    "SharedApp": {
      "Enabled": true,
      "Name": "YourBroker Open API",
      "ClientId": "1234_abcd...",
      "ClientSecret": "…"                                // مشفرة في الراحة؛ أبدا السجلات
    }
  }
}
```

عند بدء التشغيل يبذر التطبيق تطبيق مشترك واحد يملكه حساب المالك (idempotent — إنه أبدًا
يكتب فوق قيمة edited-owner في وقت التشغيل والبذور إعادة no-op).

### 2. إعدادات المالك (وقت التشغيل بدون إعادة نشر)

**الإعدادات → Open API** (المالك فقط) يعرض شيئين: قسم **تطبيقك الخاص Open API** — حيث يسجل المالك ويحرر ويفوض **الخاص به** تطبيق per-user بالضبط مثل أي مستخدم (متاح بينما لا يوجد تطبيق مشترك مكون) — وبطاقة **تطبيق مشترك للنشر** لإضافة / تحرير / حذف التطبيق المشترك مع عنوان URL إعادة التوجيه المعروض لـ copy-paste. تأخذ التغييرات تأثير للتفويضات الجديدة على الفور. بمجرد تكوين تطبيق مشترك فإنه يحل محل تطبيق المالك الخاص به وقسم **تطبيقك الخاص Open API** ينتقل إلى إشعار بأن الحسابات تفوض الآن من خلال التطبيق المشترك.

## عنوان URL إعادة التوجيه (سجل هذا في cTrader)

كل تطبيق cTrader Open API يسجل **واحد** عنوان URL إعادة توجيه — **نفس القيمة الفردية** للـ
التطبيق المشترك وأي تطبيق per-user:

```
{your deployment URL}/openapi/callback
```

على سبيل المثال `https://cmind.yourbroker.com/openapi/callback`.

- التطبيق **يعرض القيمة الدقيقة** على صفحة إعدادات Open API (مع زر نسخ) — الصقها
  في بوابة شريك cTrader عند إنشاء تطبيق Open API.
- يتم تكوينه من `App:OpenApi:PublicBaseUrl` لذلك يبقى مستقرًا خلف بروكسي معكوس / CDN؛
  عندما يكون هذا غير مجموعة يعود إلى مضيف الطلب الوارد.
- تختلف تجربة الدعوة مقابل مستخدم عادي فقط في حيث يهبط المستخدم **بعد** إعادة الاتصال
  (قائمة حساباتهم مقابل تأكيد "حسابات مضافة") — عنوان URL إعادة التوجيه المسجل لم يتغير.

## ما يراه المستخدمون تحت وضع مشترك

عندما يوجد تطبيق مشترك:

- المستخدمون لا يحصلون على **خيار** لتسجيل تطبيق Open API الخاص بهم — صفحة الإعدادات تعرض
  **"يتم إدارة Open API من قبل موفرك"** و زر **تفويض الحسابات** الذي يستخدم المشترك
  التطبيق.
- أي تطبيقات شخصية موجودة مسبقًا **يتم إزالتها**؛ حساباتهم المصرح بها تشير إلى
  التطبيق المشترك و يجب **إعادة تفويض** (رموزهم القديمة تم إصدارها تحت عميل مختلف
  id). محاولة إنشاء تطبيق شخصي يعيد خطأ "يُدار من قبل موفرك".

## حدود معدل العميل (لكل نوع رسالة)

يقوم العميل بوتيرة رسائل Open API cTrader الصادرة لذا الانفجار لا يزعج أبدًا معدل من جانب الخادم
حد كتلة. الحدود **لكل نوع رسالة** تطابق وثائق cTrader Open API:

| الفئة | ما يغطيه | الافتراضي |
|---|---|---|
| `General` | رسائل التداول والقراءة (الأوامر والرموز واستعلامات الحساب) | 45 msg/s |
| `HistoricalData` | طلبات trendbar / tick-data (مخنوقة أكثر بواسطة cTrader) | 5 msg/s |

يتم حساب طلب البيانات التاريخية ضد **كل من** ودلو و عام الخاص به. Heartbeat و
رسائل المصادقة لم تكن أبدًا مسارية. الرسائل صف ودرين بالمعدل المتاح — لا شيء ينخفض و النظام محفوظ.

اضبطها إذا التفاوض وسيطك **أعلى** حدود cTrader أو اضبط فئة على **`0`** لتعطيل
المسار بالكامل (غير محدود):

- **التكوين:** `App:OpenApi:RateLimits:General` / `App:OpenApi:RateLimits:HistoricalData` (msgs/sec).
- **إعدادات المالك:** بطاقة **حدود معدل العميل** على **الإعدادات → Open API** (override المالك ينتصر
  ينطبق على الاتصالات الجديدة / عند إعادة الاتصال).
