# Υγεία στρατηγικής & φθορά alpha

Κάθε πλεονέκτημα φθείρεται — η έρευνα είναι σαφής ότι ο χρόνος ημιζωής μιας quant στρατηγικής έχει καταρρεύσει από χρόνια σε μήνες, οπότε *η προσαρμογή υπερτερεί της ανακάλυψης*. Η οθόνη Υγείας Στρατηγικής σας λέει, από το ίδιο το ιστορικό αποδόσεων μιας στρατηγικής, αν το πλεονέκτημα υπάρχει ακόμα.

Ανοίξτε **cBots → Strategy Health** (`/quant/health`).

## Τι κάνει

Δεδομένης μιας χρονικής σειράς αποδόσεων (ή καμπύλης ιδίων κεφαλαίων, από το παλαιότερο στο πιο πρόσφατο), κάνει:

- χωρίζει το ιστορικό σε ένα **προηγούμενο** και ένα **πρόσφατο** μισό και συγκρίνει τις Sharpe αναλογίες τους;
- εκτελεί σάρωση **CUSUM change-point** για να εντοπίσει την παρατήρηση όπου η μέση μετατοπίστηκε πιο ξεκάθαρα (αλλαγή καθεστώτος), που αναφέρεται μόνο όταν η απόκλιση είναι στατιστικά αξιοσημείωτη;
- επιστρέφει ένα verdict:

| Verdict | Σημασία |
|---|---|
| **Healthy** | Η πρόσφατη απόδοση είναι σύμφωνη με (ή καλύτερη από) το προηγούμενο ιστορικό. |
| **Degrading** | Η πρόσφατη Sharpe είναι σημαντικά ασθενέστερη από το προηγούμενο ιστορικό — παρακολουθήστε στενά. |
| **Decayed** | Το πλεονέκτημα έχει ουσιαστικά εξαφανιστεί στο πρόσφατο παράθυρο — εξετάστε το ενδεχόμενο παύσης. |
| **Unknown** | Δεν υπάρχει αρκετό ιστορικό για κρίση. |

```http
POST /api/quant/health
{ "returns": [...] }   // ή { "equity": [...] }
```

## Γιατί είναι αξιόπιστο

Είναι καθαρός, ντετερμινιστικός domain code (`Core.Health`) χωρίς εξάρτηση υποδομής και χωρίς εξωτερικές κλήσεις — unit-tested για τις περιπτώσεις decayed, degrading, healthy και too-short και για τον εντοπισμό change-point. Είναι το χειροκίνητο συνοδευτικό των always-on health checks που υποστηρίζουν τους αυτόνομους πράκτορες: οι ίδιες στατιστικές οδηγούν τον circuit breaker που μειώνει τον κίνδυνο μιας ζωντανής στρατηγικής της οποίας το πλεονέκτημα εξασθενεί.
