# cTrader 브로커를 위한 cMind 🏦

당신은 cTrader 중개소를 운영 중입니다. 당신의 클라이언트는 이미 거래할 수 있습니다 — 하지만 다른 모든 브로커의 클라이언트도 할 수 있습니다. **cMind는 당신에게 당신의 트레이더에게 당신의 브랜드로 풀 AI 기반 거래 운영 플랫폼을 제공하도록 합니다**, 그래서 그들은 당신의 에코시스템 안에서 전략을 빌드, 백테스트, 실행, 복사, 모니터링하고 제3자 도구로 표류하지 않습니다. 그것은 stickier 클라이언트, 더 많은 거래량, 그리고 터미널 외에는 아무것도 제공하지 않는 브로커에 대한 실제 우위입니다.

:::tip TL;DR
당신의 클라이언트를 위해 화이트라벨 cMind를 실행하세요. **당신의** 중개소로 계정을 제한하고, AI 및 복사 거래를 켜고, 당신의 브랜드 아래에 배포하세요. → [비즈니스를 위한 화이트라벨](./white-label-for-business.md)
:::

## 다른 브로커보다 당신이 얻는 우위

- **도구로 차별화하세요, 스프레드만이 아니라.** 클라이언트에게 AI cBot 생성, 관리되는 클러스터의 백테스팅, 복사 거래, prop-firm 도전을 주세요 — 대부분의 브로커는 단순히 제공하지 않는 기능.
- **클라이언트를 당신의 에코시스템 안에 유지하세요.** 트레이더가 당신의 브랜드 플랫폼 안에서 전략을 빌드하고 실행할 때, 그들은 유지됩니다. 유지는 전체 게임입니다.
- **당신의 브랜드, 당신의 도메인에서.** 이름, 로고, 색상, 파비콘, 심지어 설치 가능한 휴대폰 앱 — 모두 당신의 것. 아무도 "cMind"를 보지 않습니다. → [화이트라벨 기능](./features/white-label.md)

## 당신의 계정만 제공하세요 (브로커 허용 목록)

*당신의* 클라이언트를 위해 화이트라벨을 실행 중이세요? 사용자가 추가할 수 있는 브로커의 거래 계정을 제한하여 당신의 배포가 오직 당신의 장부를 제공하도록:

```json
{
  "App": {
    "Accounts": {
      "AllowedBrokers": ["Your Brokerage Name"]
    }
  }
}
```

허용 목록이 설정되면, cMind는 사용자가 추가하려는 모든 계정을 확인합니다 — cTrader Open API를 통해 및 수동 cID 로그인을 통해(계정의 실제 브로커 이름을 읽어 검증) — 그리고 당신의 목록에 없는 계정을 거절합니다. 비워두고 모든 브로커가 허용됩니다(기본). [화이트라벨 기능 문서](./features/white-label.md#broker-allowlist)를 참조하여 전체 메커니즘을 이해하세요.

## 모든 사용자를 위해 하나의 Open API 앱 배포하세요

사용자별 번거로움을 건너뛰세요: **하나의 cTrader Open API 애플리케이션**을 제공하고 모든 클라이언트가 그것을 통해 자신의 계정을 승인합니다 — 클라이언트는 절대 자신의 것을 등록하지 않습니다. 하나의 redirect URL을 등록하고, 자격 증명을 config 또는 owner 설정에 드롭하고, 공유 모드가 모두에 대해 켜집니다. cTrader 메시지 한계를 더 높게 협상했나요? **메시지 타입별 클라이언트 속도 한계**를 조정하세요(또는 페이싱을 비활성화). → [공유 Open API 애플리케이션 & 속도 한계](./features/open-api-shared-app.md)

## 수익화할 새로운 방법

- **AI, 클라이언트를 위한 zero friction.** 배포 레벨에서 기본 AI 제공자 키를 제공하고 모든 클라이언트가 AI 기능을 즉시 얻습니다 — 다른 곳에서 가입 없음. 마크업하거나, premium 계층에 번들하세요. 클라이언트는 여전히 자신의 키를 가져올 수 있습니다. → [AI 기능](./features/ai.md)
- **Prop-firm 도전.** 실시간 자산 추적 및 시행된 규칙으로 자금 조달 트레이더 도전을 실행하고, 항목에 대해 비용을 청구하세요. → [Prop-firm 규칙](./features/prop-firm.md)
- **복사 거래 비즈니스.** 성과 수수료 및 제공자 마켓플레이스는 복사 거래를 수익으로 전환합니다. → [성과 수수료](./features/copy-performance-fees.md) · [제공자 마켓플레이스](./features/copy-provider-marketplace.md)
- **기능 계층.** 각 클라이언트 세그먼트가 [기능 토글](./features/feature-toggles.md)로 어떤 기능을 볼지 결정합니다.

## 규제, 감사 가능, 멀티테넌트

- **[준수](./features/compliance.md)** 로그는 당신의 규제자가 물어볼 감사 증적을 제공합니다.
- **[2단계 인증](./features/two-factor-auth.md)**은 배포별로 필수로 만들어질 수 있습니다.
- **클라이언트별 브랜딩** — 당신의 자신의 제어 평면에서 구동되는 세그먼트별로 별도의 브랜드 인스턴스를 실행하세요. → [멀티테넌트 브랜딩](./white-label-for-business.md#multi-tenant-per-customer-branding)

## 어떻게 시작할지

1. [비즈니스를 위한 화이트라벨](./white-label-for-business.md)을 읽어서 60초 리브랜드를 위해.
2. `App:Accounts:AllowedBrokers`를 당신의 중개소로 설정하고 당신의 [기능 세트](./features/feature-toggles.md)를 선택하세요.
3. [배포](./deployment/cloud.md)하세요 — Docker, Kubernetes, Azure, 또는 AWS.

인프라를 직접 실행하지 않기를 선호하시나요? 호스팅 제공자가 관리형 cMind를 당신을 위해 운영할 수 있습니다 — [클라우드 & VPS 제공자를 위해](./for-cloud-providers.md)를 가리키세요.
