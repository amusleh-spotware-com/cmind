# نسخ التجارة

مرآة **رئيسي** حساب cTrader على واحد+ **عبد** حسابات — عبر الوسيط، عبر cID — مع التحكم لكل وجهة + المصالحة بدرجة أموال.

## المفاهيم

- **ملف تعريف النسخ** — واحد رئيسي (`SourceAccountId`) + واحد+ **وجهات**. دورة الحياة: `Draft → Running → Paused → Stopped` (`Error` على الفشل). جذر الإجمالي: `CopyProfile` (يملك `CopyDestination`).
- **الوجهة** — حساب عبد واحد + مجموعة قاعدة كاملة لكيفية نسخ الرئيسي عليها. جميع التكوين لكل وجهة، لذا تغذي الرئيس حساب الأرقام المحافظة + العدوانية في نفس الوقت.
- **مضيف محرك النسخ** — عامل يعمل لملف التعريف (`CopyEngineHost`). الاشتراك في تيار الكرم الرئيسي، تطبيق كل حدث على كل وجهة.
- **المشرف** — `CopyEngineSupervisor`، خدمة الخلفية على كل عقدة. استضافة ملفات التعريف المخصصة، الشفاء الذاتي عبر المجموعة (انظر [القياس](../deployment/scaling.md)).

## ما يتم عكسه

| حدث الرئيسي | إجراء العبد |
|--------------|--------------|
| فتح موضع السوق / market-range | فتح نسخة الحجم (مع عنوان معرف موضع المصدر) |
| أمر معلق Limit / stop / stop-limit | ضع أمر معلق مطابق |
| تعديل أمر معلق | تعديل أمر معلق مرآة في المكان |
| إلغاء أمر معلق / انتهاء الصلاحية | إلغاء أمر معلق مرآة |
| إغلاق جزئي | أغلق نفس نسبة موضع العبد |
| Scale-in (زيادة الحجم) | فتح حجم إضافي (اختياري) |
| تغيير stop-loss / trailing-stop | تعديل حماية موضع العبد |
| إغلاق كامل | أغلق النسخة العبد |

كل نسخة **مع عنوان معرف موضع/أمر المصدر**. بعد إعادة الاتصال، يعيد المضيف بناء الحالة من المصالحة: يفتح نسخ الرئيسي يحتفظ بها ولكن العبد مفقود، يغلق "الأيتام" العبد الرئيسي لم يعد يحتفظ بها — **بدون نسخ مكررة من الصفقات**.

## إنشاء ملف تعريف

يقوم حوار **ملف تعريف جديد** على صفحة Copy Trading بجمع كل شيء في وقت واحد: اسم ملف التعريف، المصدر (الرئيسي) حساب، الوجهة (العبد) حسابات (اختيار متعدد مع زر **Select all**؛ الرئيسي المختار مستبعد من قائمة العبد)، + مجموعة خيار كاملة لكل وجهة أدناه. جميع المدخلات **التحقق قبل الحفظ** — اسم مفقود/مصدر/وجهة، معامل تحجيم غير موجب، حدود لوت سالبة/غير متسقة، ٪ انسحاب خارج النطاق، نوع أمر غير ممكن، مرشح رمز فارغ، أو أزواج رموز مصفوفة مشوهة سطح كقائمة خطأ + حفظ الكتلة. عند التأكيد، يتم إنشاء ملف التعريف وإضافة كل عبد محدد مع الإعدادات المختارة.

تعامل الصفوف مع دورة الحياة: **البدء** ممكن فقط عند عدم التشغيل، **إيقاف** + **إيقاف مؤقت** فقط عند التشغيل، **حذف** معطّل أثناء التشغيل + يطلب تأكيد قبل إزالة ملف التعريف + الوجهات.

## خيارات لكل وجهة

تعيين في حوار New Profile، على لوحة لكل وجهة على صفحة Copy Trading، أو عبر `POST /api/copy/profiles/{id}/destinations`:

- **التحجيم** (`MoneyManagementMode` + معامل): لوت ثابت، مضروب notional/lot، رصيد متناسب/رصيد/هامش فارغ، ٪ مخاطرة ثابتة، رافعة ثابتة، متناسب تلقائي، **risk-%-from-stop** (M7). بالإضافة إلى حدود لوت min/max + force-min-lot. **Risk-from-stop** يحجم وجهة لذا تخاطر بنسبة مئوية مكونة من *رصيد الوجهة الخاص بها*، المشتق من **مسافة stop-loss الرئيسية** (`master risks 2% → slave auto-risks 2%`): `lots = balance×% ÷ (stopDistance × contractSize)`. فتح الرئيس **بدون** stop-loss ليس لديه مسافة لحجم ضده → استخدم **max-risk fallback lot** المجهز (M7) إذا تم تعيينه، وإلا تم تخطيه (`no_stop_loss`) لا يخمن. حجم متناسب-**رصيد**/**هامش فارغ** من **رصيد** حساب حقيقي (`balance + Σ floating P&L`، المشتق لكل cTrader Open API الذي لا يسلم رصيد)، وليس رصيد عادي — لذا الرئيسي الجالس على ربح/خسارة مفتوح يحجم النسخ بشكل صحيح. لا يتم فضح الهامش المستخدم بواسطة API المصالحة، لذا يتم التعامل مع الهامش الحر كرصيد (وكيل أموال متاح الصادقة)؛ أنماط أخرى قراءة الرصيد + تخطي جولة revaluation إضافية.
- **مرشح الاتجاه**: كليهما / طويل فقط / قصير فقط. **عكس**: فلي الجانب (+ swap SL↔ TP) لنسخ معاكس.
- **Manage-only** (Ignore-New-Trades / Close-Only): مرآة الإغلاقات، الإغلاقات الجزئية + حماية التغييرات على المراكز المنسوخة بالفعل، ولكن افتح **لا** مراكز/أوامر معلقة جديدة (تم تخطيه `manage_only`). استخدام لرياح وجهة بدون قطع النسخ الموجودة.
- **Sync-Open-on-start** / **Sync-Closed-on-start** (الافتراضي على): على **أول** إعادة مزامنة ملف التعريف، ما إذا فتح نسخ لمراكز الرئيسي الموجودة مسبقاً، + ما إذا أغلق نسخ الرئيسي أغلق بينما توقف ملف التعريف. كلاهما ينطبق فقط عند البدء — إعادة الاتصال الداخلية تصالح دائماً بالكامل حتى يتعافى desync بغض النظر.
- **رموز خريطة** + **مرشح رموز** (قائمة بيضاء / قائمة سوداء). تحمل كل إدخال رموز خريطة اختياري **مضروب الحجم لكل رمز** (cMAM override per-symbol) تحجيم نسخة الحجم لهذا الرمز على وجهة التحجيم (1 = لا تغيير). كل خريطة import/export كـ **CSV** (`GET …/symbol-map.csv`, `PUT …/symbol-map/csv`; الأعمدة `Source,Destination,VolumeMultiplier`) — يتم التحقق من صحة كل صف عبر كائنات قيمة النطاق، لذا يمكن لملف مشوه أن ينتج عن خريطة صالحة.
- **نافذة ساعات التداول** (C18) — نافذة UTC يومية لكل وجهة (`start`/`end` دقائق يوم، النهاية حصرية؛ `start == end` = طوال اليوم). فتح جديد خارج نافذة تم تخطيه (`trading_hours`); نافذة مع `start > end` يلتف بعد منتصف الليل (على سبيل المثال 22:00–06:00). المراكز الموجودة تبقى مدارة.
- **مرشح label مصدر** (C18، مكافئ cTrader من مرشح رقم سحر MT) — عند التعيين، انسخ فقط صفقات الرئيسي التي يطابق label **بالضبط** (على سبيل المثال صفقات bot واحد، أو label يدوي فقط)؛ وإلا تم تخطيه (`source_label`). فارغ = انسخ الجميع. تم نقله على `ExecutionEvent.SourceLabel` من `TradeData.Label` الموضع/أمر الرئيسي، محترم عند إعادة المزامنة أيضاً.
- **حماية الحساب** (ZuluGuard / Global Account Protection) — اعرض **رصيد حي** وجهة (`balance + Σ floating P&L`، استقصي كل `CopyDefaults.EquityGuardInterval`) ضد أرضية `StopEquity` و/أو سقف اختياري `TakeEquity`. على الانتهاك، ​​طبق الوضع: **CloseOnly** (إيقاف النسخ الجديدة، مواصلة إدارة الموجودة)، **Frozen** (إيقاف الفتح)، **SellOut** (أغلق **كل** نسخة على الوجهة فوراً). بمجرد تشغيلها، وجهة latched — لا توجد فتح جديد حتى إعادة تشغيل المضيف — + تم رفع تنبيه `CopyAccountProtectionTriggered`. `SellOut` يتطلب `StopEquity`; `TakeEquity` يجب أن يجلس فوق `StopEquity`. **لا-ضمان تحذير:** بيع الخروج يستخدم تنفيذ السوق — مثل كل مكافئ منافس، لا يمكن ضمان سعر الملء في السوق السريعة/المفتتة.
- **Flatten-All panic button** (C8) — `POST /api/copy/profiles/{id}/flatten` على الفور أغلق **كل** موضع منسوخ على كل وجهة + أقفال ضد فتح جديد. موجهة عبر العملية: تعيين API العلم، المشرف يسلم إلى المضيف التشغيل (إعادة استخدام قناة token-rotation)، والذي يسطح في المكان؛ تم حذف العلم لذا يطلق بالضبط مرة واحدة (تنبيه `CopyFlattenAll`). يقوم المستخدم بعد ذلك بإيقاف مؤقت/إيقاف ملف التعريف.
- **حراسة قاعدة الشركة الملكية** (C7) — إنفاذ قاعدة استنساخ الشركة الملكية التي يطلبها المستخدمون. لكل وجهة، **سقف الخسارة اليومية** (الخسارة من رصيد فتح اليوم) و/أو حد **trailing-drawdown** (الخسارة من ذروة عمل الرصيد)، كلاهما في عملة الإيداع. على انتهاك الوجهة **تلقائياً مسطحة** (كل نسخة مغلقة) + **مغلقة** بقية يوم UTC (فتح جديد تم تخطيه `prop_lockout`); تنبيه `CopyPropRuleBreached` ينطلق. يتم مسح الإغلاق عند لفة يوم UTC على (baseline/peak جديد مأخوذ). حصص نفس استقصاء الرصيد الحي مثل حماية الحساب.
- **Execution jitter** (C11، معطّل افتراضياً) — تأخير عشوائي `0..N` ms قبل وضع كل نسخة، لإلغاء الارتباط بطوابع زمنية متطابقة تقريباً عبر حسابات **الخاصة بك**. **تحذير الامتثال:** مساعدة للشركات الملكية التي *تسمح* بالنسخ — **ليس** أداة لتجنب الشركة التي تحظره؛ البقاء ضمن قواعد شركتك هو مسؤوليتك.
- **قفل التكوين** (C9) — تجميد إعدادات الوجهة لفترة (`POST …/destinations/{id}/lock` مع دقائق). بينما مقفل، لا يمكن إزالة الوجهة (الإجمالي يرفض مع `CopyDestinationConfigLocked`) — حراسة متعمدة ضد التغييرات المتهورة أثناء الانسحاب. ينتهي القفل تلقائياً عند الطابع الزمني الخاص به.
- **تنبيه ما قبل الاتساق** (C10) — تحذير (مرة واحدة لكل يوم UTC) عندما **الربح اليومي** وجهة يصل إلى نسبة مئوية مكونة من رصيد فتح اليوم (`CopyConsistencyThresholdApproaching`)، لذا تُحترم قاعدة اتساق الشركة الملكية *قبل* أن تفعل. جانب الربح، مستقل عن حفظ جانب الخسارة؛ يعمل الخط الأساسي اليومي نفسه كـ prop-rule guard.
- **مرشح نوع الأمر** — اختر بالضبط أي أنواع أوامر رئيسية لنسخ: السوق، market-range، limit، stop، stop-limit (`CopyOrderTypes` flags; الافتراضي الجميع). cMAM-style selectivity.
- **نسخ SL / نسخ TP** — مرآة stop-loss / take-profit الرئيسي، أو إدارة الحماية بشكل مستقل.
- **نسخ trailing stop**، **مرآة إغلاق جزئي**، **مرآة scale-in** — كل منها قابلة للتبديل بشكل مستقل.
- **نسخ انتهاء الصلاحية المعلقة** (الافتراضي على) — مرآة الطابع الزمني لصلاحية Good-Till-Date المعلقة الرئيسية.
- **نسخ الانزلاق الرئيسي** (الافتراضي على) — لأوامر market-range + stop-limit، ضع أمر العبد مع انزلاق الرئيسي في points (سعر أساسي مأخوذ من spot حي العبد).
- **الحراسات**: الحد الأقصى للانسحاب %، سقف الخسارة اليومية، الحد الأقصى لتأخير النسخ، مرشح الانزلاق (تخطي النسخ إذا تحرك سعر العبد بعيداً عن N pips من إدخال الرئيسي). **الحد الأقصى لتأخير النسخ** المقاس ضد الطابع الزمني للخادم الحقيقي للحدث الرئيسي (`ExecutionEvent.ServerTimestamp`) عبر `TimeProvider` المحقون: الإشارة الأقدم من تأخير أقصى مكون تم تخطيه، لذا لا يتم وضع نسخة قديمة متأخرة (تأخير سابق دائماً صفر + حراسة ميتة).
- **تطبيع دقة SL/TP** (M6) — تم تقريب أسعار stop-loss/take-profit المنسوخة إلى **دقة digit الرمز الوجهة** قبل التعديل، لذا لا يؤدي سعر الرئيسي بدقة أدق (أو عدم توافق digit عبر الوسيط) أبداً إلى تفعيل خادم `INVALID_STOPLOSS_TAKEPROFIT`.
- **قاطع دائرة الرفض / Follower Guard** (G8) — وجهة رفض `CopyDefaults.RejectionBudget` يفتح في صف **مرفوع**: لا توجد فتح جديد لنافذة فترة هادئة (تنبيه `CopyDestinationTripped` ينطلق)، إيقاف عاصفة الرفض من المطرقة (حساب الشركة الملكية). المراكز الموجودة لا تزال مدارة + مغلقة بينما مرفوع؛ قاطع إعادة تعيين تلقائي بعد فترة هادئة + نسخة ناجحة مسح العداد.
- **سقف sanity لوت** (C14) — الحد الأقصى المطلق لحجم النسخة و/أو cap متعدد الرئيسي. محسوب النسخة تجاوزت الحد الأقصى المطلق، أو تجاوزت `N×` حجم الرئيسي الخاص بهم، **صعب محصور** (مطبق كـ `lot_sanity` تخطي، يُحسب على `cmind.copy.skipped`) لم يتم وضعه — يدافع عن فئة catastrophic-oversize (لوت 0.23 الرئيسي يتحول إلى 3 لوت على كل متلقي عبر مضروب runaway أو خطأ تقريب). كلا البعدين افتراضي `0` (معطّل).

## الموثوقية والحالات الحدية

محرك مدمج للواقع الذي يمكن أن يفشل أي شيء في أي وقت:

- **timeout ربط ملء معلق العبد** (C13) — عبد معلق مرآة الذي تلاشى معلق الرئيسي (لا يستريح أو يملأ للتو) ملغى بعد انتهاء الصلاحية الارتباط، لذا لا يمكن لنسخة العبد ملء غير مرتبطة بموضع غير مُدار (`CopyPendingTimedOut`). إعادة المزامنة أيضاً تنظف اليتيم المعلق الممتلئ.
- **إغلاق/تسطيح قوي** (M8) — إغلاق يتيم على إعادة المزامنة، أو التسطيح على انتهاك الحماية، يتحمل الموضع الذي أغلقه الوسيط بالفعل (`POSITION_NOT_FOUND`): كل إغلاق يعمل بشكل مستقل، لذا معرف واحد قديم أبداً يحطم إعادة المزامنة أو يترك الحساب غير المسطح.

- **ابدأ مع الرئيسي بالفعل في الصفقات** — في بدء المضيف يصالح + يفتح نسخ لمراكز الرئيسي الموجودة.
- **انقطاع الاتصال / desync** — عند إعادة الاتصال، المضيف يصالح: يفتح نسخ مفقودة، يغلق الأيتام، re-labels المعلقة. لا توجد أوامر مكررة.
- **فشل وضع الأمر** — الفشل على وجهة واحدة سجل، أبداً يحظر وجهات أخرى.
- **رمز صالح واحد لكل cID** — cTrader يبطل token old الوصول الفوري الجديد الصادر. cMind مبادلة رمز المضيف التشغيل **في المكان** (إعادة auth على مقبس حي) لذا النسخ تستمر بدون إسقاط التيار. انظر [دورة حياة الرمز](token-lifecycle.md).

## إمكانية المراجعة

كل إجراء ينبعث حدث سجل منظم ومصدر المنتجة (`LogMessages`) مع معرف ملف التعريف، cID وجهة، معرفات الأمر/الموضع، + القيم — أمر مفروض/مخطي (مع السبب)، إغلاق جزئي، حماية مطبقة، trailing مطبقة، معلق مفروض/معدّل/ملغى، انتهاء صلاحية مرآة، market-range انزلاق مرآة، رمز مبادلة، ملخص إعادة المزامنة. هذا هو مسار التدقيق للامتثال + حل النزاعات.

جنباً إلى جنب مع السجلات، المحرك ينبعث **مقاييس OpenTelemetry** على `cMind.Copy` meter (مسجل في خط أنابيب OTel المشترك، تصدير على OTLP / إلى Azure Monitor مثل بقية الدول): `cmind.copy.latency` (حدث الرئيسي → dispatch، ms), `cmind.copy.dispatch.duration` (محبة لجميع الوجهات، ms), `cmind.copy.slippage.points`, `cmind.copy.placed` (tagged by destination), `cmind.copy.skipped` (tagged by reason), + `cmind.copy.failed`. هذه تجعل انسحاب الكمون/الانزلاق قابل للقياس، وليس مرئي فقط في سطر السجل — حزمة حية تؤكد عليهم ضد الميزانية.

## API

- `GET /api/copy/profiles` — قائمة.
- `POST /api/copy/profiles` — إنشاء (مع معرفات حساب وجهة اختيارية).
- `GET /api/copy/profiles/{id}` — تفاصيل كاملة incl. كل خيار وجهة.
- `POST /api/copy/profiles/{id}/destinations` — إضافة وجهة مع مجموعة الخيار الكاملة.
- `DELETE /api/copy/profiles/{id}/destinations/{destinationId}` — إزالة.
- `POST /api/copy/profiles/{id}/{start|pause|stop}` — دورة الحياة.
