# cMind для облачных и VPS провайдеров 🖥️

Вы уже сдаёте вычисления в аренду. cMind — это готовый открытый продукт, которым вы можете обернуть те вычисления: **предложите управляемый хостинг cMind** и приземлитесь на высокоценный, липкий, требующий много вычислений рабочий процесс — алготрейдеры, брокеры, проприетарные компании и торговые сообщества, которые хотят, чтобы платформа работала без того, чтобы стать командой ops.

:::tip[TL;DR]
Запустите Stateless уровень + Postgres + флот узлов; дайте клиентам брендированный URL. Монетизируйте подписку, вычисления, белый ярлык и ИИ. → [Развертайте в облаке](./deployment/cloud.md)
:::

## Почему предлагать управляемый cMind

- **Нет стоимости сборки.** Он открыт, распространяется под лицензией MIT и уже задокументирован, протестирован и контейнеризирован. Вы упаковываете и управляете — вы не создаёте.
- **Дифференцированный продукт для прибыльной нишевой рынка.** Алготрейдинг требует много вычислений: бэктесты и живые узлы сжигают CPU, что *биллируемое использование*, которое вы уже продаёте.
- **Липкие клиенты.** Трейдеры, которые создают и запускают стратегии внутри платформы, не отваливаются легко.
- **Превращает оговорку в апсел.** cMind самостоятельно разместившихся по замыслу — для клиентов, которые «не хотят быть командой ops», *вы* — ответ.

## Кто покупает управляемый cMind у вас

- **Одиночные квантти и трейдеры**, которые хотят его разместить. → [Для трейдеров](./for-traders.md)
- **Брокеры cTrader**, запускающие белый ярлык для своих клиентов. → [Для брокеров](./for-brokers.md)
- **Проприетарные компании и предприятия копирования торговли**, которые нуждаются в брендированной, проверяемой инфраструктуре.

## Что означает «управляемый cMind» для запуска

Вы управляете тремя уровнями; клиент получает брендированный веб-URL:

| Уровень | Что это | Где это работает |
|---|---|---|
| Stateless (Web + MCP) | Приложение + API + MCP-сервер | Любая платформа контейнеров, автомасштабируемая |
| База данных | PostgreSQL | Управляемый Postgres (RDS / Flexible Server / ваш собственный) |
| Флот узлов | Создает и запускает контейнеры cTrader | **ВМ или Kubernetes — требует привилегированного Docker** |

:::warning[Одна вещь, которую нужно определить заранее]
Агенты узлов создают и запускают контейнеры cTrader, поэтому им нужен **привилегированный Docker**. Это исключает среды выполнения контейнеров без сервера (Azure Container Apps, AWS Fargate) *для агентов* — запустите эти на [Kubernetes](./deployment/kubernetes.md), ВМ или EC2. Stateless уровень работает везде.
:::

Реальные руководства развертывания с copy-paste делают это конкретным: [обзор облака](./deployment/cloud.md) · [Azure](./deployment/cloud-azure.md) · [AWS](./deployment/cloud-aws.md) · [Kubernetes](./deployment/kubernetes.md) · [Масштабирование](./deployment/scaling.md).

## Как вы его монетизируете

- **Подписка на управляемый хостинг.** Месячные Starter / Team / Business планы, рассчитанные по флоту узлов и параллелизму бэктеста.
- **Использование и дозирование вычислений.** Выставляйте счета за часы бэктеста, часы живого узла и хранилище — естественно дозируется флотом контейнеров, который вы уже запускаете.
- **Уровни перепродажи белого ярлыка.** Взимайте больше за полный перебрендинг (логотип, цвета, PWA, `ShowSiteLink=false`) и за включение премиум-возможностей через [переключатели функций](./features/feature-toggles.md). → [Белый ярлык](./features/white-label.md)
- **Управляемый ИИ.** Свяжите ключ поставщика ИИ по умолчанию, чтобы каждый клиент пользователей получал ИИ без настройки, и наценка использование — или предложите принести свой ключ. → [Функция ИИ](./features/ai.md)
- **Доля доходов от проприетарной компании и копирования торговли.** Хостируйте компании, запускающие испытания и комиссии за производительность, и берите долю платформы. → [Проприетарная компания](./features/prop-firm.md) · [Комиссии за производительность](./features/copy-performance-fees.md) · [Рынок поставщиков](./features/copy-provider-marketplace.md)
- **Установка, адаптация и SLA.** Присоединяйте профессиональные услуги и премиум-поддержку.

## Мультитенантные паттерны

- **Развертывание для каждого клиента (рекомендуется).** Один брендированный экземпляр на клиента — сильная изоляция, брендирование для каждого клиента и база данных, отдельный маркер присоединения узла для каждого клиента. Брендирование читается из `IOptionsMonitor`, поэтому каждый экземпляр несёт свою собственную личность. → [Мультитенантное брендирование](./white-label-for-business.md#мультитенантное-брендирование-для-каждого-клиента) · [Обнаружение узлов](./operations/node-discovery.md)
- **Общая плоскость управления (продвинуто).** Управляйте многими экземплярами из своего собственного слоя подготовки, вводя брендирование и функции для каждого клиента программно.

## Дозирование использования для выставления счетов

Конечная точка, доступная только владельцу/администратору **`GET /api/usage`**, возвращает сводку доступа только для чтения, которую провайдер может опрашивать и выставлять счета — без любого нового домена или сохранения, он проецирует существующее состояние:

```json
{
  "users": { "total": 42 },
  "nodes": { "total": 6, "online": 5 },
  "instances": { "total": 1280, "backtestsRunning": 3, "runsRunning": 11 },
  "cbots": { "total": 210 },
  "tradingAccounts": { "total": 88 }
}
```

Опрашивайте за развертывание клиента, чтобы управлять ценообразованием на основе мест, на основе флота или на основе рабочей нагрузки. Комбинируйте с [логированием и наблюдаемостью](./operations/logging.md) для более тонкого дозирования вычислений.

## Начните работу

1. Поднимите эталонное развертывание из [руководств облака](./deployment/cloud.md).
2. Создайте его шаблон для каждого клиента (брендирование + маркер присоединения + БД) и свяжите вашу выставление счетов с использованием вычислений.
3. Список его — теперь у вас есть управляемая платформа алготрейдинга для продажи.

## Вносите обратно

Провайдеры, запускающие cMind в масштабе, первыми удаляют острые края. Внесение ваших операционных исправлений и улучшений IaC делает ваш флот дешевым в обслуживании — начните с [руководства для участников](./contributing.md).
