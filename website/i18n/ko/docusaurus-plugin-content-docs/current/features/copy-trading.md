# 카피 트레이딩

**마스터** cTrader 계정의 거래를 하나 이상의 **슬레이브** 계정에 미러링 — 브로커 간, cID 간 지원 — 대상별 제어 + 돈 등급 재무제표.

## 개념

- **카피 프로필** — 마스터 1개 (`SourceAccountId`) + 대상 1개 이상. 라이프사이클: `Draft → Running → Paused → Stopped` (실패 시 `Error`). 애그리게이트 루트: `CopyProfile` (`CopyDestination` 소유).
- **대상** — 슬레이브 계정 1개 + 마스터를 슬레이브에 복사하는 방식에 대한 전체 규칙 세트. 모든 설정이 대상별로 적용되므로, 하나의 마스터가 보수적 + 공격적 슬레이브에 동시에 공급될 수 있습니다.
- **카피 엔진 호스트** — 프로필용 실행 작업자 (`CopyEngineHost`). 마스터 실행 스트림을 구독하고 각 이벤트를 모든 대상에 적용합니다.
- **슈퍼바이저** — 각 노드의 백그라운드 서비스인 `CopyEngineSupervisor`. 할당된 프로필을 호스팅하고 클러스터에서 자체 복구합니다 ([scaling](../deployment/scaling.md) 참고).

## 미러링되는 항목

| 마스터 이벤트 | 슬레이브 동작 |
|--------------|--------------|
| 시장 / 시장 범위 포지션 개설 | 크기 조정된 복사본 개설 (소스 포지션 ID로 라벨 표시) |
| 대기 / 스탑 / 스탑 리밋 대기 주문 | 일치하는 대기 주문 배치 |
| 대기 주문 수정 | 미러링된 대기 주문을 제자리에서 수정 |
| 대기 주문 취소 / 만료 | 미러링된 대기 주문 취소 |
| 부분 청산 | 슬레이브 포지션의 같은 비율 청산 |
| 스케일 인 (볼륨 증가) | 추가 볼륨 개설 (옵트인) |
| 스탑 로스 / 트레일링 스탑 변경 | 슬레이브 포지션의 보호 수정 |
| 전체 청산 | 슬레이브 복사본 청산 |

모든 복사는 **소스 포지션/주문 ID로 라벨 표시**. 호스트 재연결 후 재조정에서 상태 재구축: 마스터가 보유하지만 슬레이브에 없는 복사본 개설, 마스터가 더 이상 보유하지 않은 슬레이브 "고아" 청산 — **거래 중복 없이**.

## 프로필 생성

카피 트레이딩 페이지의 **새 프로필** 대화상자에서 처음부터 모두 수집: 프로필 이름, 소스(마스터) 계정, 대상(슬레이브) 계정 (**모두 선택** 버튼이 있는 다중 선택; 선택된 마스터는 슬레이브 목록에서 제외), + 아래의 전체 대상별 옵션 세트. 모든 입력은 **저장 전 검증** — 이름/소스/대상 누락, 양수가 아닌 크기 조정 매개변수, 음수/일관성 없는ロット 범위, 범위 외 드로다운 %, 주문 유형 미활성화, 빈 심볼 필터, 잘못된 심볼 맵 쌍은 오류 목록으로 표시 + 저장 차단. 확인 시 프로필 생성 + 선택된 각 슬레이브가 선택된 설정으로 추가됩니다.

행 액션은 라이프사이클 존중: **시작**은 실행 중이 아닐 때만 활성화, **중지** + **일시정지**는 실행 중일 때만 활성화, **삭제**는 실행 중 비활성화 + 프로필 + 대상 제거 전 확인 요청.

## 대상별 옵션

새 프로필 대화상자, 카피 트레이딩 페이지의 대상별 패널, 또는 `POST /api/copy/profiles/{id}/destinations`에서 설정:

- **크기 조정** (`MoneyManagementMode` + 매개변수): 고정ロット,ロット/명목 승수, 비례 잔액/Equity/자유 마진, 고정 리스크 %, 고정 레버리지, 자동 비례, **리스크-%-프롬-스탑** (M7). 최소/최대ロット 범위 + 최소ロット 강제 적용. **리스크-프롬-스탑**은 *자신의* 잔액의 구성 비율을 리스크하는 대상 크기를 조정 — 마스터의 **스탑 로스 거리**에서 파생 (`마스터가 2% 리스크 → 슬레이브 자동 리스크 2%`): `lots = balance×% ÷ (stopDistance × contractSize)`. 마스터가 스탑 로스 **없이** 개설 시 크기 조정 거리가 없음 → 구성된 **최대 리스크 폴백ロット** 사용 (M7), 설정되지 않으면 건너뛰기 (`no_stop_loss`) — 추측 안 함. 비례 **Equity**/**자유 마진** 크기 조정은 실제 계정의 **Equity** (`balance + Σ floating P&L`, cTrader Open API가 전달하지 않는 Equity에서 파생)에 적용 — 마스터가 열린 수익/손실에 있으면 복사본 크기가 정확함. 사용 마진은 재조정 API에 의해 노출되지 않으므로, 자유 마진은 Equity로 처리 (정직한 사용 가능 자금 프록시); 다른 모드는 잔액 + 추가 재평가 라운드트립 건너뛰기.
- **방향 필터**: 양방 / 롱 전용 / 숏 전용. **리버스**: 역방향 (+ SL↔TP 스왑) 카피.
- **관리 전용** (신규 거래 무시 / 청산 전용): 미러가 청산, 기존 복사된 포지션의 부분 청산 + 보호 변경은 유지하되 신규 포지션/대기 주문 **개설 안 함** (건너뛰기 `manage_only`). 기존 복사본을 자르지 않고 대상을 축소하는 데 사용.
- **시작 시 동기화-오픈** / **시작 시 동기화-클로즈** (기본값: 켜짐): 프로필의 **첫 번째** 재동기화에서 마스터의 기존 포지션 복사본 개설 여부, 프로필 중지 중 마스터가 청산한 복사본 청산 여부. 둘 다 시작 시만 적용 — 중간 실행 재연결은 항상 완전히 재조정되어 동기화 해제 복구됨.
- **심볼 맵** + **심볼 필터** (화이트리스트 / 블랙리스트). 각 심볼 맵 항목은 선택적 **심볼별 볼륨 승수** (심볼별 cMAM 재정의)를携带하여 대상의 크기 조정 위에 해당 심볼의 복사 크기 조정 (1 = 변경 없음). 전체 맵은 **CSV**로 가져오기/내보내기 (`GET …/symbol-map.csv`, `PUT …/symbol-map/csv`; 열 `Source,Destination,VolumeMultiplier`) — 각 행은 도메인 값 객체를 통해 검증되므로 잘못된 파일은 잘못된 맵을 생성할 수 없음.
- **거래 시간 창** (C18) — 대상별 일일 UTC 창 (`start`/`end` 하루 중 분, end는 exclusive; `start == end` = 종일). 창 외의 신규 개설은 건너뛰기 (`trading_hours`); `start > end`인 창은 자정을 넘어 래핑 (예: 22:00–06:00). 기존 포지션은 관리 유지.
- **소스 라벨 필터** (C18, MT 매직 번호 필터의 cTrader 동등물) — 설정 시 마스터 거래 중 라벨이 **정확히** 일치하는 거래만 복사 (예: 하나의 봇 거래 또는 수동 전용 라벨); 그렇지 않으면 건너뛰기 (`source_label`). 빈 = 모두 복사. 마스터 포지션/주문의 `TradeData.Label`에서 온 `ExecutionEvent.SourceLabel`에 반영, 재동기화 시에도 적용.
- **계정 보호** (ZuluGuard / 글로벌 계정 보호) — 대상의 **실시간 Equity** (`balance + Σ floating P&L`, `CopyDefaults.EquityGuardInterval`마다 폴링)를 `StopEquity` 하한 및/또는 선택적 `TakeEquity` 상한과 대비. 위반 시 모드 적용: **CloseOnly** (신규 복사 중지, 기존 관리 유지), **Frozen** (개설 중지), **SellOut** (대상의 모든 복사본 즉시 청산). 발동 후 대상 래치됨 — 호스트 재시작 전까지 신규 개설 안 함 + `CopyAccountProtectionTriggered` 경고 발생. `SellOut`은 `StopEquity` 필요; `TakeEquity`은 `StopEquity` 위에 있어야 함. **보장 불가 고지:** 매도-아웃은 시장 실행 사용 — 모든 경쟁사의 동등물과 마찬가지로 빠르거나 갭 있는 시장에서 체결 가격 보장 불가.
- **전면 청산 패닉 버튼** (C8) — `POST /api/copy/profiles/{id}/flatten`은 모든 대상의 모든 복사된 포지션을 즉시 청산 + 신규 개설 방지. 크로스 프로세스 라우팅: API가 플래그 설정, 슈퍼바이저가 실행 중 호스트에 전달 (토큰 로테이션 채널 재사용), 제자리에서 전면 청산; 플래그 지워서 정확히 한 번만 발동 (`CopyFlattenAll` 경고). 사용자가 프로필을 일시정지/중지합니다.
- **프로펌 룰 가드** (C7) — 요청하는 프로펌 카피 사용자 시행. 대상별 **일일 손실 상한** (당일 개시 Equity로부터의 손실) 및/또는 **트레일링 드로다운** 제한 (실행 피크 Equity로부터의 손실), 모두 예치 통화. 위반 시 대상 **자동 전면 청산** (모든 복사본 청산) + UTC 일 중 나머지 시간 **잠금** (신규 개설 건너뛰기 `prop_lockout`); `CopyPropRuleBreached` 경고 발생. 잠금은 UTC 일 전환 시 지워짐 (새로운 기준/피크 적용). 계정 보호와 동일한 실시간 Equity 폴링 공유.
- **실행 지터** (C11, 기본값: 꺼짐) — 각 복사를 배치하기 전 무작위 `0..N`ms 지연, 사용자의 **자체** 계정에서 거의 동일한 주문 타임스탬프 간 비상관화. **컴플라이언스 고지:** 복사를 *허용하는* 프로펌 지원용 — 금지하는 프로펌 회피 도구 아님; firm's 규칙 준수 여부는 본인의 책임.
- **설정 잠금** (C9) — 기간 대상 설정 동결 (`POST …/destinations/{id}/lock`, 분単位). 잠금 동안 대상 제거 불가 (애그리게이트가 `CopyDestinationConfigLocked`로 거부) — 드로다운 중 즉각적 변경에 대한 의도적 가드. 잠금은 해당 타임스탬프에서 자동 만료.
- **일관성 사전 경고** (C10) — 대상의 **일일 수익**이 당일 개시 Equity의 구성 비율에 도달하면 (한 번 per UTC 일) 경고 (`CopyConsistencyThresholdApproaching`), 프로펌 일관성 규칙이 적용되기 *전에* 존중.
- **주문 유형 필터** — 복사할 마스터 주문 유형 선택: 시장, 시장 범위, 리밋, 스탑, 스탑 리밋 (`CopyOrderTypes` 플래그; 기본값: 모두). cMAM 스타일 선택성.
- **카피 SL / 카피 TP** — 마스터의 스탑 로스 / 테이크 프로핏 미러링 또는 보호 독립 관리.
- **카피 트레일링 스탑**, **미러 부분 청산**, **미러 스케일 인** — 각각 독립적으로 토글 가능.
- **카피 대기 만료** (기본값: 켜짐) — 마스터 대기 주문의 Good-Till-Date 만료 타임스탬프 미러링.
- **카피 마스터 슬리피지** (기본값: 켜짐) — 시장 범위 + 스탑 리밋 주문의 경우 슬레이브 주문을 마스터의 정확한 슬리피지(포인트 단위)로 배치 (기본 가격은 슬레이브의 실시간 스팟에서 가져옴).
- **가드**: 최대 드로다운 %, 일일 손실 상한, 최대 복사 지연, 슬리피지 필터 (슬레이브 가격이 마스터 진입가에서 N 피프 이상 이동 시 복사 건너뛰기). **최대 복사 지연**은 주입된 `TimeProvider`를 통해 마스터 이벤트의 실제 서버 타임스탬프 (`ExecutionEvent.ServerTimestamp`) 대비 측정: 구성된 최대 지연보다 오래된 시그널은 건너뛰기, 지연된 복사가 늦게 배치되지 않음 (이전에는 지연이 항상 0 + 가드 미작동).
- **SL/TP 정밀도 정규화** (M6) — 복사된 스탑 로스/테이크 프로핏 가격이 수정 전 **대상** 심볼의 숫자 정밀도로 반올림되어 마스터 가격이 더 세밀한 정밀도(또는 브로커 간 숫자 불일치)더라도 서버의 `INVALID_STOPLOSS_TAKEPROFIT`를トリガー하지 않음.
- **거부 서킷 브레이커 / 팔로워 가드** (G8) — `CopyDefaults.RejectionBudget`의 열린 복사를 연속으로 거부하는 대상은 **트리핑**: 쿨다운 창 동안 신규 개설 안 함 (`CopyDestinationTripped` 경고 발생), (프로펌) 계정을hammering하는 거부 폭풍 중지. 기존 포지션은 계속 관리 + 트리핑 중 청산; 서킷 브레이커는 쿨다운 + 성공적 복사 후 자동 재설정.
- **ロット 서니 시일링** (C14) — 최대 복사 크기 및/또는 마스터ロット 배수. 절대 상한을 초과하거나 마스터 자신의ロット 크기의 `N×`를 초과하는 계산된 복사본은 **하드 차단** (`lot_sanity` 건너뛰기로 표시, `cmind.copy.skipped`에 카운트) — 배치되지 않음 — 카테고포틱 오버사이즈(class) 방어 (0.23로트 마스터가失控 승수 또는 반올림 버그로 각 수신자에게 3로트가 되는 경우). 두 차원 모두 기본값 `0` (꺼짐).

## 신뢰성 및 엣지 케이스

엔진은 모든 것이 항상 실패할 수 있다는 현실을 위해 구축됨:

- **슬레이브-대기 체결-상관관계 타임아웃** (C13) — 마스터 대기가 사라진 미러링된 슬레이브 대기(휴식 중도, 새로 체결도 아닌)가 상관관계 타임아웃 후 취소되어 슬레이브 복사본이 관리되지 않는 포지션에 상관 없이 체결되지 않음 (`CopyPendingTimedOut`). 재동기화는 또한 채워진 대기의 고아 주문 ID 라벨 청소를 수행.
- **강력한 청산/전면 청산** (M8) — 재동기화에서 고아 청산 또는 가드 위반 시 전면 청산 시 브로커가 이미 청산한 포지션 허용 (`POSITION_NOT_FOUND`): 각 청산은 독립적으로 실행되므로 하나의 오래된 ID가 재동기화를 중단시키거나 나머지 계정 미청산을 남기지 않음.

- **트레이드 중 마스터로 시작** — 시작 시 호스트가 마스터의 기존 포지션 복사본을 재조정 + 개설.
- **연결 끊김 / 동기화 해제** — 재연결 시 호스트 재조정: 누락된 복사본 개설, 고아 청산, 대기 재라벨. 주문 중복 없음.
- **주문 배치 실패** — 한 대상의 실패는 로그되지만 다른 대상 차단 안 함.
- **cID당 단일 유효 토큰** — cTrader는 새 토큰이 발급되는 순간 cID의 이전 액세스 토큰을 무효화. cMind는 실행 중인 호스트의 토큰을 **제자리에서** 스왑 (실시간 소켓에서 재인증)하여 복사를 계속하면서 스트림을 드롭하지 않음. [토큰 수명주기](token-lifecycle.md) 참고.

## 감사 가능성

모든 액션은 프로필 ID, 대상 cID, 주문/포지션 ID + 값을 포함한 구조화된 소스 생성 로그 이벤트 (`LogMessages`)를 방출 — 배치된/건너뛰기된 주문 (이유 포함), 부분 청산, 보호 적용, 트레일링 적용, 대기 배치/수정/취소, 만료 미러링, 시장 범위 슬리피지 미러링, 토큰 스왑, 재동기화 요약. 이것은 컴플라이언스 + 분쟁 해결을 위한 감사 트레일입니다.

로그와 함께 엔진은 `cMind.Copy` 미터 (`cMind.Copy` 미터, 공유 OTel 파이프라인에 등록, OTLP / Azure Monitor로 내보내기)에서 **OpenTelemetry 메트릭** 방출: `cmind.copy.latency` (마스터 이벤트 → 디스패치, ms), `cmind.copy.dispatch.duration` (모든 대상への 팬아웃, ms), `cmind.copy.slippage.points`, `cmind.copy.placed` (대상별 태그), `cmind.copy.skipped` (이유별 태그), + `cmind.copy.failed`. 이를 통해 지연/슬리피지 회귀가 로그 라인이 아닌 측정 가능 — 라이브 스위트는 예산 대비 주장.

## API

- `GET /api/copy/profiles` — 목록.
- `POST /api/copy/profiles` — 생성 (선택적 대상 계정 ID 포함).
- `GET /api/copy/profiles/{id}` — 전체 세부정보 (모든 대상 옵션 포함).
- `POST /api/copy/profiles/{id}/destinations` — 전체 옵션 세트로 대상 추가.
- `DELETE /api/copy/profiles/{id}/destinations/{destinationId}` — 제거.
- `POST /api/copy/profiles/{id}/{start|pause|stop}` — 라이프사이클.

- **E2E** (`tests/E2ETests`) — API + UI를 통한 대상 옵션 라운드트립, 전체 라이프사이클.
- **스트레스 / DST** (`tests/StressTests`) — 결정론적 시뮬레이션 테스트: 시드된 난수화 작업 + 폴트 인젝션 (소켓 플랩, 주문 거부, 시장 범위 거부, 토큰 로테이션, 노드 죽음)이 `CopyEngineHost`를 quiescence로駆動 + 수렴 불변량 주장. [testing/stress-testing.md](../testing/stress-testing.md) 참고. 이 스위트는 실제 시작 경주를 발견 + 수정: `OnReconnected`가 초기 참조 로드 + 재동기화 전에 연결되어 있어 시작 중 소켓 플랩이 두 번째 재동기화를 동시에 실행 + 호스트의 비동시 상태 사전 손상 가능 — 시작 로드 + 첫 번째 재동기화가 이제 `_stateGate`에서 실행됨.
- **라이브** — 실제 cTrader 데모 계정; [testing/live-copy-trading.md](../testing/live-copy-trading.md) 참고.

실시간 + E2E 티어에 사용할 단일 자격 증명 파일은 [dev-credentials.md](../testing/dev-credentials.md) 참고.
