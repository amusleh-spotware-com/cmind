# Agent Studio

يتيح لك Agent Studio إنشاء **وكيل تجاري لديه شخصية** — بدون كود — ومنحه إدارة حساباتك نحو أهداف قابلة للقياس. الوكيل يشبه cBot موجه بالشخصية: تختار نمط بدائي وموقفاً، وتحدد الحدود الوقائية، وينفذ تحت **Autonomy & Safety Kernel**.

افتح **AI → Agent Studio** (`/agent-studio`).

## إنشاء وكيل

تجمع حوار **New agent** بدون كود:

- **الاسم والنمط البدائي** — Scalper (مضارب قصير الأجل)، Day Trader، Swing Trader، Position Trader، News Trader (مضارب الأخبار)، Contrarian (معاكس)، Mean Reversion أو Breakout/Momentum. يحدد كل إعداد مسبق إيقاعاً وموقفاً معقولاً.
- **الموقف** — أشرطة التهور والصبر والمتابعة الاتجاهية.
- **مستوى الاستقلالية** — **Advisory** (يقترح فقط) أو **Approval-gated** (يعمل فقط بعد موافقتك على كل إجراء). **Full Auto** (بدون موافقة لكل صفقة) يتطلب أيضاً **محيط مخاطرة** وقبول إخلاء المسؤولية عن المخاطر قبل تفعيله.

تترجم الشخصية **بشكل حتمي** إلى موجه النظام الخاص بالوكيل (لا يقوم نموذج لغة بتأليفه)، لذا يعطي نفس التكوين دائماً نفس التعليمات — قابل للتكرار والمراجعة.

## السجل

يظهر كل وكيل في جدول غرفة التحكم: **أي وكيل وما نوعه، وكم عدد الحسابات التي يديرها، وأهدافه، وحالة التشغيل، والإجراء الأخير**، مع عناصر تحكم **بدء / إيقاف / إيقاف طارئ**. يوقف مفتاح الإيقاف الطارئ وكيلاً يعمل بشكل فوري.

## الأمان هو ثابت النطاق، وليس إعداداً

كل شيء يمس الأموال يسير عبر **Autonomy & Safety Kernel**:

- **محيط المخاطرة** — حدود صعبة لكل أمر (الخسارة اليومية القصوى، التعرض المفتوح، حجم المركز، الرافعة المالية، الخسائر المتتالية، الأوامر/ساعة، الرموز المسموح بها). يتم التحقق من صحة كل أمر ضده قبل الإرسال؛ يتم رفض الانتهاك، وليس تقليله.
- **قاطع الدائرة** — يوقف بشكل حتمي المخاطرة الجديدة على تسلسل خسارة أو انتهاك خسارة يومية أو **انتهاك هدف الأداء الصعب** أو **عدم توفر مزود الذكاء الاصطناعي** (لا يفتح نموذج معطل أو هلوسة أبداً مراكز جديدة).
- **موافقة الإخلاء الموثوقة** — يلزم قبول موثوق ومُصَدَّر مرة واحدة لتفعيل Full Auto (موافقة مطلوبة قانوناً، وليس موافقة لكل صفقة)؛ يفرض تحديث الإخلاء إعادة موافقة.
- **مفتاح الإيقاف الطارئ** — إيقاف طارئ خاملة على كل وكيل يعمل.

## الأهداف

أعط وكيلاً **أهداف قابلة للقياس** — على سبيل المثال *اجعل الحد الأقصى للانسحاب أقل من 4%*، *عامل الربح على الأقل 1.5*، *معدل الفوز ≥ 55%*. كل هدف هو **صعب** (حد وقائي — يؤدي الانتهاك إلى تشغيل قاطع الدائرة) أو **ناعم** (يوجه التفكير فقط)، يتم تقييمه على أنه On-track / At-risk / Breached.

## خط أنابيب القرار

بمجرد البدء، يعمل الوكيل **حلقة مراقبة 24/7** (`AgentRuntimeService`). في كل شريحة، لكل حساب مُدار، يقرأ **حالة الحساب الحتمية** (الحقيقة الأساسية، ليست أبداً ذاكرة النموذج)؛ يسأل محرك القرار عن خطوة؛ يمررها عبر **بوابة الأمان** (`AgentDecisionProcessor`) — مستوى الاستقلالية → قاطع الدائرة → محيط المخاطرة؛ يكتب `AgentDecisionRecord` **إضافي فقط**؛ ويتوقف أو ينفذ حسب ما تملي البوابة. الحلقة **معزولة عن الأخطاء** (فشل وكيل واحد لا يلمس أبداً آخر أو المضيف) و**آمن افتراضياً**: إنها خاملة ما لم يتم تكوين الذكاء الاصطناعي *و* تم تعيين `App:Ai:AgentRuntimeEnabled`، وأبداً لا تفتح مخاطرة جديدة بينما مزود الذكاء الاصطناعي غير متوفر.

- **بوابة الموافقة** — يتم تسجيل الأمر المقترح لوكيل **Approval-gated** بصيغة **Pending** ولا يفعل شيئاً حتى توافق المالك عليه (`POST /api/agent-studio/{id}/decisions/{seq}/approve` أو `/reject`); يمسح **Full Auto** الحد الأدنى بدون موافقة لكل صفقة؛ يقترح **Advisory** فقط.
- **دفتر التدقيق** — كل قرار قابل للتكرار: التفكير (XAI)، الدليل الذي استشهد به، حكم البوابة، نية الأمر وما إذا كان نفذ، في `GET /api/agent-studio/{id}/decisions`.
- **Research desk** — نقاش متعدد الوكلاء عند الطلب: يعطي كل محلل Alpha/Sentiment/Technical/Risk وجهة نظر وا synthesises Reviewer اقتراحاً (`POST /api/agent-studio/{id}/debate`).
- **الذاكرة** — يتذكر الوكيل كل قرار ويستدعي الذاكرة الأخيرة في موجهه التالي للاستمرارية (`GET /api/agent-studio/{id}/memory`).

يفتح **تفاصيل** كل صف السجل تغذية قرار الوكيل (مع Approve/Reject على الأوامر المعلقة)، وذاكرته، وتبويب Run-debate.

## الاختصاصات

مُرسَّلة: دورة حياة الوكيل الكاملة، بوابة الأمان الحتمية، وقت التشغيل 24/7، بوابة الموافقة البشرية في الحلقة، دفتر التدقيق، و **تكامل Open API cTrader الحي** — متجر حالة الحساب (يقرأ الرصيد الفعلي والمراكز والتعرض المفتوح بالكثير) والموفر الأمر (يضع أوامر السوق الفعلية، كثير → حجم عبر حجم اللوت للرمز)، كلاهما يحل بيانات اعتماد OAuth لكل حساب مُدار ويتدهور بأمان عندما لا يكون الحساب مرتبطاً. **يتطلب مفتاح Anthropic API** ليقوم النموذج بإنشاء أوامر (حتى ذلك الحين يحتفظ المحرك)؛ ما زالت آتية هي أدوار النقاش متعدد الوكلاء والذاكرة/الانعكاس المطبقة. وقت التشغيل معطّل ما لم يتم تعيين `App:Ai:AgentRuntimeEnabled`، لذا لا يحدث التداول الحي إلا على اختيار صريح وموافق عليه بالكامل.
