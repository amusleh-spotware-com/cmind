# cMind لوسطاء cTrader

أنت تدير شركة وساطة cTrader. يمكن لعملائك بالفعل التداول - لكن يمكن لعملاء كل وسيط آخر أيضاً. **cMind يسمح لك بيد متاجريك منصة عمليات تداول كاملة مدعومة بالذكاء الاصطناعي، مصنفة باسمك**، لذلك يبنون واختبارات وتشغيل وسخ والمراقبة استراتيجيات داخل *النظام البيئي الخاص بك* بدلاً من الانجراف إلى أداة طرف ثالث. وهذا أكثر لزوجة للعملاء، أكثر الحجم، وحافة حقيقية على الوسطاء الذين لا يقدمون سوى محطة طرفية.

:::tip TL;DR
قم بتشغيل white-label cMind لعملائك. قيد الحسابات إلى **شركتك**، تفعيل AI ونسخ التداول، وشحنه تحت علامتك التجارية. → [White-label للأعمال](./white-label-for-business.md)
:::

## الميزة التي تحصل عليها على الوسطاء الآخرين

- **التمييز على الأدوات وليس فقط على الفروق.** امنح العملاء توليد AI cBot والاختبار على مجموعة مُدارة ونسخ التداول وتحديات prop-firm - قدرات معظم الوسطاء ببساطة لا يقدمونها.
- **احتفظ بالعملاء في النظام البيئي الخاص بك.** عندما يبني المتاجرون ويشغلون استراتيجياتهم داخل منصتك المتعلمة الخاصة بك، يبقون. الاحتفاظ هو كل اللعبة.
- **تحت علامتك التجارية، في مجالك.** الاسم والشعار والألوان والفافيكون، حتى التطبيق الهاتفي القابل للتثبيت - كل شيء لك. لا أحد يرى "cMind". → [ميزة White-label](./features/white-label.md)

## خدم حساباتك فقط (قائمة بيضاء للوسيط)

تشغيل white-label لـ *عملائك الخاصين*؟ تقيد حسابات التداول من الوسطاء التي قد يضيفها المستخدمون حتى ينشر نشرك فقط في كتابك:

```json
{
  "App": {
    "Accounts": {
      "AllowedBrokers": ["Your Brokerage Name"]
    }
  }
}
```

عندما تكون القائمة البيضاء محددة، cMind يتحقق من كل حساب يحاول المستخدم إضافته - سواء عبر cTrader Open API أو عبر تسجيل دخول cID يدوي (التحقق من خلال قراءة اسم الوسيط الحقيقي للحساب) - ويرفض أي حساب ليس في قائمتك. اتركه فارغاً وكل وسيط مسموح (الافتراضي). انظر [وثيقة ميزة White-label](./features/white-label.md#broker-allowlist) للميكانيكا الكاملة.

## شحن تطبيق Open API واحد لجميع المستخدمين

تخطي المتاعب لكل مستخدم: توفير **تطبيق cTrader Open API واحد** وكل عميل يصرح حساباتهم من خلاله - لا عميل أبداً يسجل حسابهم الخاص. تسجيل عنوان URL إعادة توجيه واحد، أسقط بيانات الاعتماد في الإعدادات أو إعدادات المالك، والوضع المشترك يتحول لا للجميع. التفاوض على حد رسالة cTrader أعلى؟ ضبط **حدود معدل عميل لكل نوع رسالة** (أو تعطيل خطي). → [تطبيق Open API المشترك وحدود المعدل](./features/open-api-shared-app.md)

## طرق جديدة للربح

- **AI، بدون احتكاك على العملاء.** توفير مفتاح موفر AI افتراضي على مستوى النشر وكل عميل يحصل على ميزات AI على الفور - لا التسجيل في مكان آخر. حده، أو استشرقه في طبقات premium. يمكن للعملاء أيضاً أن يأتوا بمفتاحهم. → [ميزة AI](./features/ai.md)
- **تحديات Prop-firm.** تشغيل تحديات المتاجرين الممولين مع تتبع الإنصاف الحي والقواعس المفروضة، والشحن للمدخلات. → [قواعد Prop-firm](./features/prop-firm.md)
- **نسخ تداول الأعمال.** تحويل الرسوم والسوق موفر تحويل نسخ التداول إلى الإيرادات. → [رسوم الأداء](./features/copy-performance-fees.md) · [سوق الموفر](./features/copy-provider-marketplace.md)
- **طبقات الميزات.** قرر أي الإمكانيات كل قطاع العميل يرى مع [مفاتيح الميزات](./features/feature-toggles.md).

## منظم وقابل للتدقيق وتعدد المستأجرين

- **[الامتثال](./features/compliance.md)** السجلات تعطيك مسار تدقيق منظم الخاص بك سيطلب.
- **[المصادقة ثنائية العامل](./features/two-factor-auth.md)** يمكن جعلها إلزامية لكل نشر.
- **العلامة التجارية لكل عميل** - قم بتشغيل مثيل مصنف منفصل لكل قطاع، مدفوع من لوحة التحكم الخاصة بك. → [العلامة التجارية متعددة المستأجرين](./white-label-for-business.md#multi-tenant-per-customer-branding)

## كيفية البدء

1. اقرأ [White-label للأعمال](./white-label-for-business.md) للإعادة الخاصة بـ 60 ثانية.
2. ضع `App:Accounts:AllowedBrokers` إلى شركتك الخاصين واختر [مجموعة الميزات](./features/feature-toggles.md).
3. [نشر](./deployment/cloud.md) - Docker و Kubernetes و Azure أو AWS.

لا تريد تشغيل البنية التحتية بنفسك؟ يمكن لمزود استضافة تشغيل cMind مُدار لك - اشير إليهم [للسحابة ومزودي VPS](./for-cloud-providers.md).
