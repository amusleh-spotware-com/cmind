# Копирование торговли

Отражайте **мастер**-счет cTrader на один+ **рабских** счетов — кросс-брокер, кросс-cID — с контролем для каждого направления + согласование денежного класса.

## Понятия

- **Профиль копирования** — один мастер (`SourceAccountId`) + один+ **направления**. Жизненный цикл: `Draft → Running → Paused → Stopped` (`Error` при отказе). Корневой агрегат: `CopyProfile` (владеет `CopyDestination`).
- **Направление** — один рабский счет + полный набор правил того, как мастер скопирован на него. Всё config для каждого направления, поэтому один мастер кормит консервативных и агрессивных рабских одновременно.
- **Хост механизма копирования** — работающий рабочий процесс для профиля (`CopyEngineHost`). Подписывается на поток выполнения мастера, применяет каждое событие для каждого направления.
- **Супервизор** — `CopyEngineSupervisor`, фоновый сервис на каждом узле. Хосты назначенных профилей, самостоятельное исцеление по кластеру (смотрите [масштабирование](../deployment/scaling.md)).

## Что отражается

| Событие мастера | Действие раба |
|--------------|--------------|
| Открытие позиции рынка / диапазон рынка | Откройте копию размера (обозначенную исходным id позиции) |
| Отложенный заказ лимита / остановка / стоп-лимит | Разместите соответствующий отложенный заказ |
| Поправка отложенного заказа | Поправьте отражённый отложенный заказ на месте |
| Отмена отложенного заказа / истечение | Отмените отражённый отложенный заказ |
| Частичное закрытие | Закройте ту же долю позиции раба |
| Масштабирование (увеличение объема) | Откройте добавленный объем (выбор) |
| Изменение стоп-лосса / трейлинга | Поправьте защиту позиции раба |
| Полное закрытие | Закройте копию раба |

Каждая копия **обозначена исходным id позиции/заказа**. После повторного подключения хост перестраивает состояние из согласования: открывает копии, которые мастер держит, но раб отсутствует, закрывает рабские "сироты", которых мастер больше не держит — **без дублирования сделок**.

## Создание профиля

Диалог **New Profile** на странице Copy Trading собирает всё заранее: имя профиля, исходный (мастер) счет, направления (рабские) счета (мультивыбор с кнопкой **Select all**; выбранный мастер исключён из списка рабов) + полный набор параметров для каждого направления ниже. Все вводы **проверены перед сохранением** — отсутствие имени/источника/направления, не положительные параметры размера, отрицательные/несогласованные границы лота, вне диапазона проценты прорисовки, нет включённого типа заказа, пустой фильтр символа или неправильные пары символов карты поверхности как список ошибок + блок сохранения. При подтверждении профиль создан + каждый выбранный раб добавлен с выбранными настройками.

Действия строк уважают жизненный цикл: **Start** включен только когда не работает, **Stop** + **Pause** только когда работает, **Delete** отключен при работе + запрашивает подтверждение перед удалением профиля + направлений.

## Параметры для каждого направления

Установите в диалоге New Profile, в панели для каждого направления на странице Copy Trading или через `POST /api/copy/profiles/{id}/destinations`:

- **Размер** (`MoneyManagementMode` + параметр): фиксированный лот, лот/условное множество, пропорциональный баланс/капитал/свободная маржа, фиксированный риск %, фиксированный леверидж, авто-пропорциональный, **риск-%-из-стопа** (M7). Плюс мин/макс границы лота + сила-мин-лот. **Риск-из-стопа** размеры направления так, чтобы он рисковал сконфигурированным процентом *его собственного* баланса, полученным из **расстояния стоп-лосса мастера** (`мастер рискует 2% → раб авто-рискует 2%`): `лоты = баланс×% ÷ (stopDistance × contractSize)`. Мастер открыт **без** стоп-лосса нет расстояния для размера против → использует сконфигурированный **max-risk fallback лот** (M7), если установлен, иначе пропущен (`no_stop_loss`) не угадан. Пропорциональный-**капитал**/**свободная маржа** размер вне реального счета **капитал** (`баланс + Σ плавающий P&L`, полученный в cTrader Open API, который не доставляет капитал), не простой баланс — так что мастер, сидящий на открытой прибыли/потере, размер копии прямо. Использованная маржа не открыта по согласованию API, поэтому свободная маржа рассматривается как капитал (честное доступное прокси средств); другие режимы читают баланс + пропускают дополнительный раунд переоценки.
- **Фильтр направления**: оба / только длинный / только короткий. **Развернуть**: флип сторона (+ подкачка SL↔TP) для контрарной копии.
- **Только управление** (Игнорировать-новые-сделки / Закрыть только): отражайте закрытия, частичные закрытия + изменения защиты на уже скопированных позициях, но откройте **никакие** новые позиции/отложенные заказы (пропущены `manage_only`). Используйте для ветра направления без обрезания существующих копий.
- **Sync-Open-on-start** / **Sync-Closed-on-start** (по умолчанию включены): при **первом** повторном синхронизации профиля, открывать ли копии для существующих до профиля позиций мастера, + закрывать ли копии, которые мастер закрыл, пока профиль остановлен. Оба применяются только в начале — повторное подключение в реальном времени всегда полностью согласовывается, так что desync восстанавливается независимо.
- **Карта символов** + **фильтр символов** (белый список / чёрный список). Каждая запись карты символов содержит необязательный **множитель объема для каждого символа** (переопределение для каждого символа cMAM), масштабирующий размер копии для этого символа поверх размера направления (1 = без изменения). Вся карта импортов/экспортов как **CSV** (`GET …/symbol-map.csv`, `PUT …/symbol-map/csv`; колонки `Source,Destination,VolumeMultiplier`) — каждая строка проверяется через объекты значений домена, поэтому неправильный файл не может произвести невалидную карту.
- **Окно торговых часов** (C18) — окно UTC для каждого направления в день (`start`/`end` минут дня, конец исключительный; `start == end` = весь день). Новые открытия вне окна пропущены (`trading_hours`); окно с `start > end` переносится после полуночи (например 22:00–06:00). Существующие позиции остаются управляемыми.
- **Фильтр исходного ярлыка** (C18, эквивалент cTrader фильтра магического числа MT) — когда установлено, копируйте только мастер сделки, чей ярлык **точно** совпадает (например, сделки одного бота или ярлык только вручную); иначе пропущен (`source_label`). Пусто = копируйте всё. Переносимо на `ExecutionEvent.SourceLabel` из `TradeData.Label` позиции/заказа мастера, чтимо при повторной синхронизации.
- **Защита счета** (ZuluGuard / Глобальная защита счета) — смотрите **живой капитал** направления (`баланс + Σ плавающий P&L`, опрашивается каждый `CopyDefaults.EquityGuardInterval`) против пола `StopEquity` и/или необязательный потолок `TakeEquity`. При нарушении применяйте режим: **CloseOnly** (остановить новые копии, продолжить управление существующими), **Frozen** (остановить открытие), **SellOut** (закройте **каждую** копию на направлении немедленно). После срабатывания направление запёрто — нет новых открытий до перезагрузки хоста — + поднято предупреждение `CopyAccountProtectionTriggered`. `SellOut` требует `StopEquity`; `TakeEquity` должны сидеть выше `StopEquity`. **Нет гарантии caveat:** распродажа использует рыночное выполнение — как эквивалент каждого конкурента, не может гарантировать цену заполнения на быстром/разрывающемся рынке.
- **Flatten-All кнопка паники** (C8) — `POST /api/copy/profiles/{id}/flatten` немедленно закрывает **каждую** скопированную позицию на каждом направлении + блокирует против новых открытий. Маршрутизировано кросс-процесс: API устанавливает флаг, супервизор доставляет к работающему хосту (повторно использующему канал ротации маркера), который выравнивает на месте; флаг очищен, чтобы срабатывать ровно один раз (`CopyFlattenAll` предупреждение). Пользователь затем приостанавливает/останавливает профиль.
- **Защита правил проприетарной компании** (C7) — применение проприетарной копии пользователей просят. За направление, **дневной лимит убытков** (потеря от открытого капитала дня) и/или **трейлинг-прорисовка** лимит (потеря из работающего пикового капитала), оба в валюте депозита. При нарушении направление **авто-выравнено** (каждая копия закрыта) + **заблокировано** остаток UTC дня (новые открытия пропущены `prop_lockout`); `CopyPropRuleBreached` предупреждение срабатывает. Блокировка очищается когда UTC день переходит (свежий базовый/пик взят). Акции то же же полутекст капитала, что и защита счета.
- **Jitter выполнения** (C11, выключен по умолчанию) — случайная задержка `0..N` мс перед размещением каждой копии, чтобы рассоединить почти идентичные временные метки заказов на **собственных** счета пользователя. **Compliance caveat:** помощь для проприетарных компаний, которые *разрешают* копирование — **не** инструмент обхода компании, которая это запрещает; остаться в правилах компании — ваша ответственность.
- **Блокировка config** (C9) — заморозить параметры направления на период (`POST …/destinations/{id}/lock` с минутами). Пока заблокировано, направление не может быть удалено (агрегат отклоняет с `CopyDestinationConfigLocked`) — намеренная защита против импульсивных изменений при прорисовке. Блокировка истекает автоматически в его временной метке.
- **Предупреждение перед согласованностью** (C10) — предупредите (один раз за UTC день), когда **дневная прибыль** направления достигает сконфигурированного процента дневного открытого капитала (`CopyConsistencyThresholdApproaching`), поэтому правило согласованности проприетарной компании уважается *перед* триггерством. Сторона прибыли, независимая от блокировки на убытки; работает вне того же базового дня, что и защита правил проприетарной компании.
- **Фильтр типа заказа** — выберите ровно какие типы мастер заказов копировать: рынок, диапазон рынка, лимит, остановка, стоп-лимит (флаги `CopyOrderTypes`; по умолчанию все). Селективность в стиле cMAM.
- **Копируйте SL / Копируйте TP** — отражайте стоп-лосс/взятие прибыли мастера или управляйте защитой независимо.
- **Копируйте трейлинг стоп**, **отражайте частичное закрытие**, **отражайте масштабирование** — каждое независимо переключаемо.
- **Копировать истечение отложено** (по умолчанию включено) — отражайте временную метку истечения Good-Till-Date отложенного заказа мастера.
- **Копировать проскальзывание мастера** (по умолчанию включено) — для диапазона рынка + стоп-лимит заказов, разместите рабский заказ с точным проскальзыванием-в-пипсах мастера (базовая цена взята из живого спота раба).
- **Охраны**: макс прорисовка %, дневной лимит потерь, макс задержка копии, фильтр проскальзывания (пропустить копию, если цена раба переместилась за N пипс от входа мастера). **Макс задержка копии** измеряется против реальной временной метки сервера события мастера (`ExecutionEvent.ServerTimestamp`) через инъектированный `TimeProvider`: сигнал старше сконфигурированного макс-лага пропущен, поэтому стареющая копия никогда не размещена поздно (ранее задержка всегда ноль + охрана мёртва).
- **SL/TP точность нормализация** (M6) — скопированный стоп-лосс/взятие прибыли цены округлены до **направления** точность цифр символа перед поправкой, поэтому цена мастера при более тонкой точности (или кросс-брокер несоответствие цифр) никогда не срабатывает сервер `INVALID_STOPLOSS_TAKEPROFIT`.
- **Отказ выключатель / Фолловер Охрана** (G8) — направление отклоняет `CopyDefaults.RejectionBudget` открытия подряд **срабатывают**: нет новых открытий для окна кулдауна (`CopyDestinationTripped` предупреждение срабатывает), останавливающего отказ шторм от молотка (проприетарной компании) счета. Существующие позиции всё ещё управляются + закрываются при срабатывании; выключатель авто-сбрасывается после кулдауна + успешная копия очищает счетчик.
- **Лот здравомыслие потолок** (C14) — абсолютный размер копии макса и/или множество-мастера макса. Вычисленная копия, превышающая абсолютный макс, или превышающая `N×` собственный размер лота мастера, **жёстко-блокирована** (выверхнута как `lot_sanity` пропуск, подсчитана на `cmind.copy.skipped`) не размещена — защищает от катастрофического класса переразмера (0.23-лот мастер в 3 лота на каждого приёмника через зашедший множитель или ошибку округления). Обе размерности по умолчанию `0` (отключено).

## Надёжность и граничные случаи

Механизм построен для реальности, что что-нибудь может отказать в любое время:

- **Рабский отложенный таймаут корреляции заполнения** (C13) — отражённый рабский отложенный, чьё отложенное мастера исчезло (ни покоящееся, ни свежего заполнения) отменены после таймаута корреляции, поэтому рабская копия не может заполниться без корреляции в неуправляемую позицию (`CopyPendingTimedOut`). Повторная синхронизация также очищает заполненный отложенный сирот по id заказа.
- **Надёжное закрытие/выравнивание** (M8) — закрытие сироты при повторной синхронизации или выравнивание при нарушении охраны, терпит позицию брокер уже закрыл (`POSITION_NOT_FOUND`): каждое закрытие запускается независимо, поэтому один стареющий id никогда не прерывает повторную синхронизацию или не оставляет остаток счета не выравненным.

- **Начните с мастером уже в сделках** — на старте хост согласовывает + открывает копии для существующих позиций мастера.
- **Подключение падает / desync** — при повторном подключении хост согласовывает: открывает недостающие копии, закрывает сирот, повторно обозначает отложенные. Нет дублирующихся заказов.
- **Отказ размещения заказа** — отказ на одном направлении регистрируется, никогда не блокирует другие направления.
- **Один действительный маркер на cID** — cTrader аннулирует маркер доступа cID старый момент новый издан. cMind подкачивает работающий маркер хоста **на месте** (повторная аутентификация на живом сокете), поэтому копирование продолжается без отправки потока. Смотрите [жизненный цикл маркера](token-lifecycle.md).

## Проверяемость

Каждое действие излучает структурированное событие логирования, генерируемое источником (`LogMessages`) с id профиля, cID направления, id заказов/позиций + значения — заказ размещен/пропущен (с причиной), частичное закрытие, применена защита, применено отслеживание, отложено размещено/поправлено/отменено, истечение отражено, проскальзывание диапазона рынка отражено, маркер подкачан, резюме повторной синхронизации. Это аудиторский след для соответствия + разрешение споров.
