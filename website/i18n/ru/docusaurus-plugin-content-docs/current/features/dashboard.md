# Приборная панель 📊

Первое, что вы видите при входе, и честно говоря, страница, которую вы оставляете открытой весь день. Целевая страница (`/`, `Components/Pages/Index.razor`) — это **центр управления в реальном времени с приоритетом мобильности** для активности подписанного пользователя во всех запусках cBot, бэктестов, ресурсов и (для администраторов) кластера узлов. Это обновляет себя, отлично выглядит на телефоне и никогда не заставляет вас нажимать F5.

## Что это показывает

Сверху вниз, приоритет упорядочен для телефона (каждый блок — полнота элемента стека на мобильном, адаптивная сетка на планшете/рабочем столе):

1. **Заголовок** — название, живой индикатор (реальная пульсирующая точка; статическая при `prefers-reduced-motion`), время последнего обновления и **переключатель периода** (`1H · 24H · 7D · 30D`), который управляет KPI и графиком.
2. **Hero KPI** — четыре карты беглого просмотра, каждая большое число + встроенная диаграмма SVG, и (где имеет смысл) **дельта vs предыдущий период**:
   - **Активно сейчас** — запуски + бэктесты в настоящий момент запускаются/выполняются.
   - **Процент успеха** — завершены ÷ (завершены + не удалось) за период; дельта в процентных пункты.
   - **Завершено** — готовые запуски/бэктесты в этот период; дельта vs предыдущий период.
   - **Не удалось** — отказы в этот период; дельта (меньше лучше, поэтому падение показывает зеленый).
3. **График активности** — диаграмма ApexCharts временная шкала начало / завершено / не удалось за период времени.
4. **Кольцо статуса экземпляра** — пончик работающих / бэктестов / ожидающих / завершенных / не удалось, итого в центре.
5. **Бэктесты** — снимок трёх плиток (работающие / завершённые / не удалось), нажмите для просмотра к `/backtest`.
6. **Копирование торговли** — ваши профили копирования торговли с точкой живого статуса, количеством направлений и **Live** значком на работающих профилях; нажмите для просмотра к `/copy-trading`.
7. **Агенты ИИ** — ваши агенты торговли, управляемые персональностью, с состоянием запуска (архетип · статус) и временем последнего действия; нажмите для просмотра к `/agent-studio`.
8. **Лента живой активности** — 20 самых последних событий (самое новое первым) с точкой раскраски статуса и относительной временной меткой.
9. **Здоровье кластера** (только администраторы) — активные vs всего узлов и калибр использованной ёмкости.
10. **Плитки ресурсов** — cBots, торговые счета, cTrader ID, ключи MCP (нажмите для просмотра к своим страницам).

## Настройте вашу приборную панель

Каждый блок выше — это **виджет, который вы контролируете**. Нажмите **Customize** (верхний правый угол заголовка) чтобы открыть диалог, где вы **показываете/скрываете** любой виджет и **переупорядочиваете** их со стрелками вверх/вниз. **Reset to default** восстанавливает каталог заказов. Ваш выбор **сохранен на сервере для каждого пользователя**, поэтому он следует вам по браузерам и устройствам — не только эта вкладка.

- Виджеты с функцией ворот и только администратора (Копирование торговли, Агенты ИИ, Здоровье кластера) отображаются в диалоге только, когда ваше развертывание/роль может их использовать.
- Каталог виджетов — единственный источник истины в `Core/Dashboard/DashboardWidgets.cs`; представление (ярлык + значок + доступность) живет в `Components/Dashboard/DashboardWidgetMeta.cs`.

## Как это остаётся живым

Страница опрашивает `GET /api/dashboard/overview?period=<1h|24h|7d|30d>` каждые 10 секунд и повторно отображает виджеты на месте — нет ручной перезагрузки. Кратковременный отказ выборки проглатывается и повторяется на следующем тике; цикл чисто останавливается при утилизации. Первая нагрузка показывает скелет; постоянный отказ показывает карту ошибок с **Retry**; пользователь без данных видит нулевые KPI и пустые копии статуса.
