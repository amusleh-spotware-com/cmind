# cMind за облак и VPS добављаче 🖥️

Већ изнајмљујете рачунање. cMind је готов, отворени извор производ коју можете завити око тог рачуна: **нуди управљан cMind хостинг** и постави високо вредну, траженију, рачунmarijsku тешку оптерећење — алгоритамске трговце, берзе, проп фирме и трговачке заједнице коју желе платформу да се покрене без да саме постану ops тим.

:::tip[РЕЗИМЕ]
Покрените без статуса слоја + Postgres + флоте чвора; дајте клијентима белу ознаку URL. Монетизирајте претплату, рачунање, белабежену и AI. → [Развоја на облак](./deployment/cloud.md)
:::

## Зашто нудити управљан cMind

- **Без трошка грађења.** То је отворени извор, MIT лиценциран, и већ документован, тестиран и контејнериван. Паковање и управљање — не градите то.
- **Разликована производ за лукративну нишу.** Алго трговање је рачунarily тежко: тестови и живи чворови пале CPU, што је *наплатива употреба* коју већ продајете.
- **Траженији клијенти.** Трговци коју граде и покрећу стратегије унутар платформе не теку лако.
- **Претвара упозорење у надстежење.** cMind је сам-хостиран по дизајну — за клијенте коју „не желе бити ops тим," *ви* сте одговор.

## Ко купује управљан cMind од вас

- **Индивидуални квантови & трговци** коју га желе хостиран. → [За трговце](./for-traders.md)
- **cTrader берзе** водећи белу ознаку за своје клијенте. → [За берзе](./for-brokers.md)
- **Проп фирме & посали копирања трговања** коју требају белу ознаку, проверљиву инфраструктуру.

## Шта „управљан cMind" значи за покретање

Водите три слоја; клијент добиј белу ознаку веб URL:

| Слој | Шта је то | Где се покреће |
|---|---|---|
| Без статуса (Web + MCP) | Апликација + API + MCP сервер | Било која контејнер платформа, аутоматски скалирана |
| База подака | PostgreSQL | Управљан Postgres (RDS / Flexible Server / ваш сопствени) |
| Флота чвора | Граде и покрећу cTrader контејнере | **VMs или Kubernetes — требајуприиларни Docker** |

:::warning[Једна ствар за опширење напред]
Чворови агенти граде и покрећу cTrader контејнере, па требају **привилегирани Docker**. То искључује веб без сервера контејнер рунтајме (Azure Container Apps, AWS Fargate) *за агенте* — покрените те на
[Kubernetes](./deployment/kubernetes.md), VM или EC2. Без статуса слој се покреће било где.
:::

Праве, копирај-залепи водиче за развој чине ово конкретним: [облак преглед](./deployment/cloud.md) ·
[Azure](./deployment/cloud-azure.md) · [AWS](./deployment/cloud-aws.md) ·
[Kubernetes](./deployment/kubernetes.md) · [Скалирање](./deployment/scaling.md).

## Како то монетизирате

- **Управљана претплата за хостинг.** Месечни Starter / Team / Business планови величине флоте чвора и парелноје тестова.
- **Употреба & рачунање мерења.** Наплаћујте тест-часе, живи-чвор-часе и складиште — природно измерено од флоте контејнера коју већ водите.
- **Беларабежена препродајач слојева.** Наплаћујте више за потпуну ребранд (логотип, боје, PWA,
  `ShowSiteLink=false`) и за омогућавање премијумских способности преко
  [прекидачи функција](./features/feature-toggles.md). → [Беларабежена](./features/white-label.md)
- **Управљан AI.** Комбинујте подразумевану AI клавијатуру тако да сваки корисник клијента добиј AI без постављања, и обележите употребу — или нудите донесите-вашу-сопствену-клавијатуру. → [AI функција](./features/ai.md)
- **Проп-фирме & прилог копирања трговања.** Хост фирме покреће изазове и комисије за перформансе и узимају платформу. → [Проп-фирме](./features/prop-firm.md) ·
  [Комисије за перформансе](./features/copy-performance-fees.md) ·
  [Тржиште добављача](./features/copy-provider-marketplace.md)
- **Постављање, онбордирање & SLA.** Приложите професионалне услуге и премијумску подршку.

## Мултинајмерски обрасци

- **Развој-по-станару (препоручено).** Једна означена инстанца по клијенту — јака изолација,
  по-станар брендирање и база подака, различит чвор придружи токен по станару. Брендирање се чита из
  `IOptionsMonitor`, тако да свака инстанца нози своју идентичност.
  → [Брендирање мултинајмерског](./white-label-for-business.md#multi-tenant-per-customer-branding) ·
  [Откривање чвора](./operations/node-discovery.md)
- **Дељена контрола раван (напредена).** Вожење многих инстанци из ваше сопствене слоја за фабрику, сејање брендирања и функција по станару програмски.

## Мерење употреба за наплату

Власник/администратор-само **`GET /api/usage`** крајња тачка враћа само-читање резиме добављача може да анкета и наплату на — без икакве нове домене или трајност, пројектује постојеће стање:

```json
{
  "users": { "total": 42 },
  "nodes": { "total": 6, "online": 5 },
  "instances": { "total": 1280, "backtestsRunning": 3, "runsRunning": 11 },
  "cbots": { "total": 210 },
  "tradingAccounts": { "total": 88 }
}
```

Анкету по станару развој за вожење седишта базирано, флота-базирана или оптерећење базирана цена. Парови са
[логирање & посматрање](./operations/logging.md) за финерно рачунање мерења.

## Почните

1. Подигните референцу развој из [облак водичи](./deployment/cloud.md).
2. Шаблонирајте по станару (брендирање + придружи токен + DB) и жица наплату за рачунање употреба.
3. Листајте — сада имате управљану алго-трговачку платформу за продају.
