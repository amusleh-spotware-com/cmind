# Agent Studio

Agent Studioでは、**キャラクターを持つ取引エージェント**をコード不要で作成し、
測定可能な目標に向かってアカウントの管理を任せることができます。
エージェントはペルソナ駆動のcBotのようなものです：アーキタイプと態度を選択し、ガードレールを設定し、
**自律性と安全カーネル**の下で実行します。

**AI → Agent Studio**（`/agent-studio`）を開きます。

## エージェントの作成

**新規エージェント**ダイアログでは、コード不要で以下を収集します：

- **名前**と**アーキタイプ** — Scalper、Day Trader、Swing Trader、Position Trader、News Trader、
  Contrarian、Mean Reversion、Breakout/Momentum。各プリセットは合理的なケイデンスと姿勢を決定します。
- **Attitude** — 攻撃性、忍耐力、トレンドフォローシングスライダー。
- **管理対象アカウント** — **エージェントを作成するには最低1つは必須です**（アカウントのないエージェントは開始できないため、*作成*は1つ選択するまで無効のままです）。まだ取引アカウントをリンクしていない場合、ダイアログはそう述べ、まずリンクするように指示します。
- **自律レベル** — **Advisory**（提案のみ）または**Approval-gated**（アクション前に毎アクション承認）。
  **Full Auto**（毎取引承認なし）は、さらに**リスクエンベロープ**とリスク免責事項の同意の受諾が必要
 ，才能武装解除。

ペルソナはdeterministicallyにエージェントのシステムプロンプトにコンパイルされます
（LLMが作成するわけではなく）、同じ設定が常に同じ指示を生成 — 再現可能、監査可能。

## ロスター

すべてのエージェントがコントロールルームテーブルに表示されます：どのエージェント、そのタイプ、
いくつアカウントを管理しているか、目標、実行ステータス、最後のアクション +
**Start / Stop / Kill**コントロール。Killスイッチは実行中のエージェントを直ちに停止します。

## 安全は設定ではなくドメイン不変量

お金を扱うすべては**自律性と安全カーネル**を通じてルーティングされます：

- **リスクエンベロープ** — ハード кажд注文制限（最大日次損失、オープンエクスポージャー、
  ポジションサイズ、レバレッジ、連続損失、時間/hour許容シンボル）。各注文はディスパッチ前に
  エンベロープに対して検証されます; 違反はクランプされずに拒否されます。
  Full Autoになる前に必要です。
- **サーキットブレーカー** — 損失streak、 最大日次損失違反、**ハードパフォーマンス目標違反**、
  または**AIプロバイダ利用不可**发生时、deterministicallyに新しいリスク停止
  （停止またはハルシネーションしているモデルは新鮮なポジションを開かない）。
- **バージョン付き免責事項同意** — Full Autoを武装解除するために1回限りのバージョン付き同意が必要
  （法的に義務化された同意、毎取引承認ではない）; 免责事項がバンプされると再同意が強制されます。
- **Killスイッチ** — すべての実行中のエージェントに対する幂等緊急停止。

## 目標

エージェントに**測定可能な目的**を与えます — 例：*最大ドローダウンを4%未満に維持*、
*プロフィットファクター少なくとも1.5*、*勝率≥ 55%*。各ターゲットは**Hard**
（ガードレール — 違反はサーキットブレーカーをトリガー）または**Soft**（推論のみを誘導）、
評価：On-track / At-risk / Breached。

## 決定パイプライン

開始すると、エージェントは**24/7監督ループ**（`AgentRuntimeService`）を実行します。
各ティックで、 管理アカウントごとに： deterministic account stateを読み取ります
（モデルのメモリではなく ground truth）; 決定エンジンにmoveを要求; **安全ゲート**
（`AgentDecisionProcessor`）にパス — 自律レベル → サーキットブレーカー → リスクエンベロープ;
 append-onlyな`**AgentDecisionRecord`を作成; ゲートが指示するように停止または実行。
 ループは**障害分離**（1つのエージェントの失敗が他力やホストに触れることはない）、
 **デフォルトで安全**： AIが設定되어 있지 않으면不活性、
 `App:Ai:AgentRuntimeEnabled`が設定されていない限り、アクティブな間不安全、
 AIプロバイダーが利用できない間、新しいリスクを開くことはない。

- **承認ゲート** — **Approval-gated**エージェントの提案された注文は**Pending**として記録され、
  所有者（`POST /api/agent-studio/{id}/decisions/{seq}/approve`または`/reject`）が承認するまでは
  何もしない; **Full Auto**は每取引承認なしでエンベロープをクリア; **Advisory**は提案のみ。
- **監査台帳** — すべての決定は再生可能：推論（XAI）、それが引用した証拠、ゲートの評決、
  注文意図と実行されたかどうか、`GET /api/agent-studio/{id}/decisions`。
- **リサーチデスク** — オンデマンドマルチエージェントデbate：
  Alpha/Sentiment/Technical/Riskアナリストがそれぞれviewを示し、Reviewerがproposalを総合
  （`POST /api/agent-studio/{id}/debate`）。
- **メモリ** — エージェントは各決定を記憶し、次のプロンプトに最近のメモリをrecallして継続性を維持
  （`GET /api/agent-studio/{id}/memory`）。

各roster行の**Details**を開くと、エージェントの決定フィード（保留中の注文への承認/拒否付き）、
そのメモリ、Run-debateタブが開きます。

## スコープ

出荷済み：完全なエージェントライフサイクル、決定論的安全ゲート、24/7ランタイム、
human-in-the-loop承認ゲート、監査台帳、**ライブcTrader Open API統合** —
 account-state store（real balance、positions、オープンlotsでのエクスポージャーを読み取る）
 と注文executor（実際の 成行注文を配置、lots→symbol lot sizeで 量に変換）、
 管理アカウントごとにOAuth認証情報を解決し、アカウントがリンクされていない場合は安全に低下。
 **Anthropic APIキーが必要**（モデルが注文を生成するまで — その間エンジンは保持）；
 マルチエージェントデbate役とレイヤー付きメモリ/リフレクションはまだ登場していません。
 ランタイムは`App:Ai:AgentRuntimeEnabled`が設定されていない限りオフのため、
 ライブ取引は明示的な完全に同意されたオプトインでのみ発生します。

## 管理アカウントと編集

エージェント作成時に管理する取引アカウント（作成時に最低1つは必須）を選択します — 
***作成*ボタンは1つが選択されるまで無効され、作成エンドポイントは空の選択を拒否します**。
すべてのエージェントは後で**編集可能**です（名前、気質、自律レベル、管理アカウント）
roster行の鉛筆アイコンから。ライサイクルコントロール（details、edit、start、stop、kill）
はアイコン buttonで、アクションが適用されない状態では無効になります。
