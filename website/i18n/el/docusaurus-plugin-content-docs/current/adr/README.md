# Αρχιτεκτονικές Αποφάσεις

Αυτές καταγράφουν τις αποφάσεις σχεδιασμού που **δεν μπορείτε να συμπεράνετε από τον κώδικα** — τις ανταλλαγές, τους δρόμους που δεν λαμβάνονται, και γιατί. Κάθε είναι σύντομη: *Πλαίσιο → Απόφαση → Συνέπειες*. Νέα δομική απόφαση → προσθέστε ένα ADR εδώ (επόμενος αριθμός) ώστε ο επόμενος μηχανικός (ανθρώπινος ή AI) να κληρονομήσει τη συλλογιστική, όχι μόνο το
αποτέλεσμα.

| # | Απόφαση |
|---|---|
| [0001](./0001-strict-ddd-pure-core.md) | Αυστηρό DDD με καθαρό `Core` |
| [0002](./0002-tph-instance-replaces-entity.md) | Η κατάσταση της παρουσίας είναι TPH. μια μετάβαση αντικαθιστά την οντότητα |
| [0003](./0003-external-nodes-http-jwt.md) | Οι κόμβοι cTrader CLI είναι HTTP + JWT, όχι SSH/shell |
| [0004](./0004-cbotbuilder-on-web-host.md) | Ο `CBotBuilder` εκτελείται στον διακομιστή web σε κοντέινερ sandbox |
| [0005](./0005-anthropic-raw-http.md) | Ο πελάτης AI χρησιμοποιεί ακατέργαστο HTTP, όχι το SDK της Anthropic |
| [0006](./0006-copy-profile-db-lease.md) | Η φιλοξενία αντιγράφων συντονίζεται από ένα ατομικό μίσθωμα DB |
