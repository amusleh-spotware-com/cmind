# 복사 거래

**마스터** cTrader 계정을 하나 이상의 **슬레이브** 계정으로 미러링 — 크로스 브로커, 크로스 cID — 목적지별 제어 + 금융 등급 조정과 함께.

## 개념

- **복사 프로필** — 하나의 마스터 (`SourceAccountId`) + 하나 이상의 **목적지**. 생명 주기: `Draft → Running → Paused → Stopped` (실패 시 `Error`). 집계 루트: `CopyProfile` (`CopyDestination` 소유).
- **목적지** — 하나의 슬레이브 계정 + 마스터가 어떻게 복사되는지에 대한 전체 규칙 세트. 모든 구성은 목적지별이므로 하나의 마스터가 동시에 보수적 및 공격적 슬레이브를 제공합니다.
- **복사 엔진 호스트** — 프로필에 대한 실행 워커 (`CopyEngineHost`). 마스터 실행 스트림을 구독하고 각 이벤트를 모든 목적지에 적용합니다.
- **감독자** — `CopyEngineSupervisor`, 각 노드의 백그라운드 서비스. 할당된 프로필을 호스팅하고, 클러스터 전체에 자체 복구합니다 ([확장](../deployment/scaling.md) 참조).

## 미러링되는 항목

| 마스터 이벤트 | 슬레이브 액션 |
|--------------|--------------|
| 마켓 / 마켓 범위 포지션 열기 | 크기 조정된 복사 열기 (소스 포지션 ID로 레이블됨) |
| 한정 / 손절 / 손절 한정 대기 주문 | 일치하는 대기 주문 배치 |
| 대기 주문 수정 | 미러된 대기 주문을 제자리에서 수정 |
| 대기 주문 취소 / 만료 | 미러된 대기 주문 취소 |
| 부분 매도 | 슬레이브 포지션과 동일한 비율 매도 |
| 스케일인 (거래량 증가) | 추가된 거래량 열기 (옵트인) |
| 손절 / 후행 손절 변경 | 슬레이브 포지션의 보호 수정 |
| 완전 매도 | 슬레이브 복사 매도 |

모든 복사는 **소스 포지션/주문 ID로 레이블됨**. 재연결 후 호스트는 조정에서 상태를 재구축합니다: 마스터가 보유하지만 슬레이브가 없는 복사를 열고, 슬레이브 "고아"를 마스터가 더 이상 보유하지 않도록 닫습니다 — **거래를 복제하지 않고**.

## 프로필 생성

복사 거래 페이지의 **새 프로필** 대화는 모두 앞서 수집합니다: 프로필 이름, 소스 (마스터) 계정, 목적지 (슬레이브) 계정 (다중 선택 **모두 선택** 버튼 포함; 선택된 마스터는 슬레이브 목록에서 제외), + 아래의 전체 목적지별 옵션 세트. 모든 입력은 **저장하기 전에 검증됨** — 누락된 이름/소스/목적지, 음수 크기 조정 매개변수, 음수/불일치 로트 경계, 범위 벗어난 드로다운 %, 활성화된 주문 유형 없음, 빈 기호 필터 또는 잘못된 기호 지도 쌍이 오류 목록으로 표면화되고 저장을 차단합니다. 확인 시 프로필이 생성되고 모든 선택된 슬레이브가 선택된 설정으로 추가됩니다.

행 작업은 생명 주기를 존중합니다: **시작**은 실행 중이 아닐 때만 활성화됨, **중지** + **일시 중지**는 실행 중일 때만, **삭제**는 실행 중에 비활성화되고 프로필 + 목적지 제거 전에 확인을 요청합니다.

## 목적지별 옵션

새 프로필 대화, 복사 거래 페이지의 목적지별 패널 또는 `POST /api/copy/profiles/{id}/destinations`를 통해 설정:

- **크기 조정** (`MoneyManagementMode` + 매개변수): 고정 로트, 로트/명목 승수, 비례 잔액/지분/여유 마진, 고정 위험 %, 고정 레버리지, 자동 비례, **위험 %-손절 거리에서** (M7). 또한 최소/최대 로트 경계 + 강제 최소 로트. **손절 거리에서 위험**은 **마스터의 손절 거리**에서 파생된 자체 잔액의 구성된 백분율을 위험하도록 목적지 크기를 조정합니다 (`마스터 위험 2% → 슬레이브 자동 위험 2%`): `로트 = 잔액×% ÷ (손절거리 × 계약 크기)`. 마스터 열기 **손절 없이**는 크기를 조정할 거리가 없습니다 → 설정된 **최대 위험 폴백 로트** (M7)를 사용합니다 (설정된 경우), 그렇지 않으면 건너뜀 (`no_stop_loss`). 비례 **지분**/**여유 마진**은 실제 계정 **지분** (`잔액 + Σ 부동 손익`, cTrader Open API에서 파생)을 읽으므로 순수 잔액이 아니라 마스터가 열린 이익/손실에서 복사를 올바르게 크기 조정합니다. 사용된 마진은 조정 API에 의해 노출되지 않으므로 여유 마진은 지분으로 처리됩니다 (정직한 사용 가능 자금 프록시); 다른 모드는 잔액을 읽고 추가 재평가 왕복을 건너뜁니다.
- **방향 필터**: 양쪽 / 롱만 / 숏만. **역**: 측면 뒤집기 (+ SL↔TP 스왑) 대조 복사용.
- **관리만** (무시 새 거래 / 매도만): 이미 복사된 포지션에 대한 매도, 부분 매도 + 보호 변경을 미러링하지만 **새** 포지션/대기 주문을 열지 않습니다 (건너뜀 `manage_only`). 기존 복사를 자르지 않고 목적지를 정리하는 데 사용합니다.
- **시작 시 동기화 열기** / **시작 시 동기화 닫기** (기본값 켜짐): 프로필의 **첫** 재동기화에서 마스터의 사전 존재하는 포지션에 대해 복사를 열지 여부, + 프로필이 중지된 동안 마스터가 닫은 복사를 닫을 여부. 둘 다 시작 시에만 적용됩니다 — 중간 실행 재연결은 항상 완전히 조정되므로 재동기화는 관계없이 복구됩니다.
- **기호 지도** + **기호 필터** (화이트리스트 / 블랙리스트). 각 기호 지도 항목은 선택적 **기호당 거래량 승수** (cMAM 기호별 재정의)를 수행하여 목적지의 크기 조정 위에 해당 기호에 대한 복사 크기를 확장합니다 (1 = 변경 없음). 전체 지도는 **CSV**로 가져오기/내보내기 (`GET …/symbol-map.csv`, `PUT …/symbol-map/csv`; 열 `Source,Destination,VolumeMultiplier`) — 각 행은 도메인 값 객체를 통해 검증되므로 잘못된 파일은 유효하지 않은 지도를 생성할 수 없습니다.
- **거래 시간 윈도우** (C18) — 목적지별 일일 UTC 윈도우 (`시작`/`종료` 일의 분, 종료 배타적; `시작 == 종료` = 종일). 윈도우 밖의 새 열기는 건너뜀 (`trading_hours`); `시작 > 종료`를 포함하는 윈도우는 자정을 지나갑니다 (예: 22:00–06:00). 기존 포지션은 관리됨.
- **소스 레이블 필터** (C18, MT 매직 수 필터의 cTrader 동등물) — 설정된 경우 레이블이 **정확히** 일치하는 마스터 거래만 복사 (예: 한 봇의 거래 또는 수동만 레이블); 그렇지 않으면 건너뜀 (`source_label`). 빈 = 모두 복사. 마스터 포지션/주문의 `TradeData.Label`에서 `ExecutionEvent.SourceLabel`에 수행되고 재동기화에서도 존중됩니다.
- **계정 보호** (ZuluGuard / 글로벌 계정 보호) — 목적지의 **라이브 지분** (`잔액 + Σ 부동 손익`, `CopyDefaults.EquityGuardInterval`마다 폴링)을 `StopEquity` 바닥과 선택적 `TakeEquity` 천장에 대해 지켜봅니다. 위반 시 모드 적용: **CloseOnly** (새 복사 중지, 기존 관리 유지), **Frozen** (열기 중지), **SellOut** (목적지에서 **모든** 복사를 즉시 매도). 한번 실행되면 목적지가 래칭됨 — 호스트가 재시작될 때까지 새 열기 없음 — + `CopyAccountProtectionTriggered` 경고 발생. `SellOut`은 `StopEquity`를 필요로 함; `TakeEquity`는 `StopEquity` 위에 앉아야 합니다. **무보증 주의:** 판매 아웃은 시장 실행을 사용합니다 — 모든 경쟁사와 마찬가지로 빠르거나 갭이 있는 시장에서 체결 가격을 보장할 수 없습니다.
- **Flatten-All 패닉 버튼** (C8) — `POST /api/copy/profiles/{id}/flatten`은 모든 목적지에서 **모든** 복사된 포지션을 즉시 매도하고 새 열기를 잠급니다. 크로스 프로세스 라우팅: API가 플래그를 설정하고, 감독자는 실행 호스트 (토큰 로테이션 채널을 재사용)에 제공하고, 제자리에서 평탄화하고; 플래그는 지워져 정확히 한 번 발생합니다 (`CopyFlattenAll` 경고). 사용자는 프로필을 일시 중지/중지합니다.
- **프로퍼 펌 규칙 가드** (C7) — 집행 프로퍼 펌 복사기 사용자가 요청합니다. 목적지별로 **일일 손실 상한선** (개장 지분에서의 손실)과/또는 **후행 드로다운** 한정 (실행 피크 지분에서의 손실), 모두 예금 통화. 위반 시 목적지 **자동 평탄화** (모든 복사 매도) + **UTC 하루의 나머지 잠금** (새 열기 건너뜀 `prop_lockout`); `CopyPropRuleBreached` 경고 발생. 잠금은 UTC 하루가 지나갈 때 지워집니다 (신선한 기준선/피크). 계정 보호와 동일한 라이브 지분 폴을 공유합니다.
- **실행 지터** (C11, 기본값 꺼짐) — 각 복사를 배치하기 전에 무작위 `0..N` ms 지연으로 사용자의 **자신의** 계정에 걸쳐 거의 동일한 주문 타임스탠프를 상관 해제합니다. **준수 주의:** 복사를 **허용하는** 프로퍼 펌 지원 — **금지하는** 펌을 회피하는 도구가 아님; 펌의 규칙을 따르는 것은 당신의 책임입니다.
- **구성 잠금** (C9) — 기간 동안 목적지의 설정을 동결 (`POST …/destinations/{id}/lock` 분 포함). 잠금 중인 동안 목적지를 제거할 수 없습니다 (집계가 `CopyDestinationConfigLocked`로 거부) — 드로다운 중 충동적 변경에 대한 의도적 가드. 잠금은 타임스탠프에서 자동으로 만료됩니다.
- **일관성 사전 경고** (C10) — 목적지의 **일일 이익**이 개장 지분의 구성된 백분율에 도달할 때 경고 (UTC 하루당 한 번) (`CopyConsistencyThresholdApproaching`)하여 프로퍼 펌 일관성 규칙이 **트립 전에** 존중됩니다. 이익 측 손실 측 잠금과 독립적; 프로퍼 펌 규칙 가드와 동일한 하루 기준선에서 실행됩니다.
- **주문 유형 필터** — 복사할 정확히 어떤 마스터 주문 유형을 선택: 마켓, 마켓 범위, 한정, 손절, 손절 한정 (`CopyOrderTypes` 플래그; 기본값 모두). cMAM 스타일 선택성.
- **복사 SL / 복사 TP** — 마스터의 손절 / 익절을 미러링하거나 보호를 독립적으로 관리합니다.
- **후행 손절 복사**, **부분 매도 미러링**, **스케일인 미러링** — 각각 독립적으로 토글 가능.
- **대기 주문 만료 복사** (기본값 켜짐) — 마스터 대기 주문의 Good-Till-Date 만료 타임스탠프를 미러링합니다.
- **마스터 슬리피지 복사** (기본값 켜짐) — 마켓 범위 + 손절 한정 주문의 경우, 마스터의 정확한 슬리피지 인포인트로 슬레이브 주문을 배치합니다 (기본 가격은 슬레이브의 라이브 스팟에서 가져옴).
- **가드**: 최대 드로다운 %, 일일 손실 상한선, 최대 복사 지연, 슬리피지 필터 (슬레이브 가격이 마스터 진입에서 N 핍 이상 움직인 경우 복사 건너뜀). **최대 복사 지연**은 주입된 `TimeProvider`를 통해 마스터 이벤트의 실제 서버 타임스탠프 (`ExecutionEvent.ServerTimestamp`)에 대해 측정됨: 구성된 최대 지연보다 오래된 신호는 건너뜀, 따라서 부실 복사는 지연되게 배치되지 않습니다 (이전 지연은 항상 0 + 가드 불명).
- **SL/TP 정밀도 정규화** (M6) — 복사된 손절/익절 가격이 수정 전에 **목적지** 기호의 자릿수 정밀도로 반올림됨, 따라서 더 정교한 정밀도의 마스터 가격 (또는 크로스 브로커 자릿수 불일치)은 서버의 `INVALID_STOPLOSS_TAKEPROFIT`을 트립하지 않습니다.
- **거부 회로 차단기 / 팔로워 가드** (G8) — 목적지가 행의 `CopyDefaults.RejectionBudget` 열기를 거부하는 것은 **트립됨**: 쿨다운 윈도우 동안 새 열기 없음 (`CopyDestinationTripped` 경고 발생), 거부 폭주가 (프로퍼 펌) 계정을 해머링하지 못함. 기존 포지션은 트립되는 동안 계속 관리 및 매도됨; 브레이커는 쿨다운 후 자동 재설정 및 성공적 복사 카운터 클리어.
- **로트 적합성 천장** (C14) — 절대 최대 복사 크기 및/또는 다중 마스터 상한선. 계산된 복사가 절대 상한선을 초과하거나 `N×` 마스터의 자체 로트 크기를 초과하면 **하드 블록됨** (`lot_sanity` 건너뜀으로 표면화, `cmind.copy.skipped`에서 계산됨) 배치되지 않음 — 재앙적 초과 크기 등급 (0.23 로트 마스터가 런어웨이 승수 또는 반올림 버그를 통해 각 수신자에서 3 로트로 변함)을 방어합니다. 둘 다 기본값 `0` (끔).

## 신뢰성 및 엣지 케이스

엔진은 언제든 무엇이든 실패할 수 있다는 현실을 위해 구축됨:

- **슬레이브 대기 체결 상관 타임아웃** (C13) — 마스터 대기가 사라진 미러된 슬레이브 대기 (정지하지 않거나 새로 체결되지 않음) 상관 타임아웃 후 취소됨, 따라서 슬레이브 복사는 상관 해제되지 않은 관리되지 않는 포지션으로 체결될 수 없습니다 (`CopyPendingTimedOut`). 재동기화는 또한 주문 ID 레이블 체결 대기 고아를 정리합니다.
- **견고한 매도/평탄화** (M8) — 재동기화에서 고아 매도 또는 가드 위반에서 평탄화, 브로커가 이미 매도한 포지션 허용 (`POSITION_NOT_FOUND`): 각 매도는 독립적으로 실행되므로 하나의 부실 ID는 재동기화를 중단하거나 나머지 계정을 평탄화되지 않은 상태로 두지 않습니다.

- **마스터가 이미 거래 중인 상태에서 시작** — 시작 시 호스트는 조정하고 마스터의 기존 포지션에 대한 복사를 엽니다.
- **연결 드롭 / 재동기화** — 재연결 시 호스트는 조정: 누락된 복사 열기, 고아 매도, 대기 재라벨. 이중 주문 없음.
- **주문 배치 실패** — 하나의 목적지에서의 실패는 기록되고 다른 목적지를 절대 차단하지 않습니다.
- **cID당 단일 유효 토큰** — cTrader는 새 토큰이 발급되는 순간 cID의 오래된 액세스 토큰을 무효화합니다. cMind는 실행 호스트의 토큰을 **제자리에서** 스왑합니다 (라이브 소켓에 재인증) 따라서 복사는 스트림을 드롭하지 않고 계속됩니다. [토큰 생명 주기](token-lifecycle.md)를 참조하세요.

## 감사 가능성

모든 액션은 구조화된 소스 생성 로그 이벤트 (`LogMessages`)를 프로필 ID, 목적지 cID, 주문/포지션 ID + 값 — 주문 배치/건너뜀 (이유 포함), 부분 매도, 보호 적용, 후행 적용, 대기 배치/수정/취소, 만료 미러, 마켓 범위 슬리피지 미러, 토큰 스왑, 재동기화 요약으로 방출합니다. 이것은 준수 + 분쟁 해결을 위한 감사 추적입니다.

로그 옆에 엔진은 공유 OTel 파이프라인에 등록되고 OTLP / Azure Monitor로 내보내지는 `cMind.Copy` 미터에 **OpenTelemetry 메트릭**을 방출합니다 (나머지처럼): `cmind.copy.latency` (마스터 이벤트 → 디스패치, ms), `cmind.copy.dispatch.duration` (모든 목적지로 팬아웃, ms), `cmind.copy.slippage.points`, `cmind.copy.placed` (목적지로 태그됨), `cmind.copy.skipped` (이유로 태그됨), + `cmind.copy.failed`. 이를 통해 지연/슬리피지 회귀는 로그 라인에서 가시적인 것뿐 아니라 측정 가능합니다 — 라이브 스위트는 예산에 대해 그것들을 어설션합니다.

## API

- `GET /api/copy/profiles` — 나열.
- `POST /api/copy/profiles` — 생성 (선택적 목적지 계정 ID 포함).
- `GET /api/copy/profiles/{id}` — 모든 목적지 옵션을 포함한 전체 세부 정보.
- `POST /api/copy/profiles/{id}/destinations` — 전체 옵션 세트로 목적지 추가.
- `DELETE /api/copy/profiles/{id}/destinations/{destinationId}` — 제거.
- `POST /api/copy/profiles/{id}/{start|pause|stop}` — 생명 주기.
