# クラウド & VPS プロバイダー向け cMind 🖥️

あなたはすでにコンピュートをレンタルしています。cMind は、そのコンピュートの周りにラップできる既製のオープンソース製品です: **管理 cMind ホスティングを提供**して、高価値で粘り気のあるコンピュート集約的なワークロードをランディングしてください — アルゴリズムトレーダー、ブローカー、プロップファーム、プラットフォームを実行することなく自分たちが運用チームになることを望まないトレーディングコミュニティ。

:::tip TL;DR
ステートレスティア + Postgres + ノード群を実行します。顧客にブランド化された URL を渡してください。サブスクリプション、コンピュート、ホワイトラベル、AI を現金化してください。→ [クラウドにデプロイ](./deployment/cloud.md)
:::

## 管理 cMind を提供する理由

- **ビルドコストなし。** オープンソース、MIT ライセンス、および既にドキュメント化、テスト、コンテナ化されています。パッケージと運用を行います — ビルドしません。
- **儲かるニッチの差別化された製品。** アルゴトレードはコンピュート集約的です: バックテストとライブノードは CPU を消費します。これは*課金可能な使用量*であり、あなたはすでに販売しています。
- **粘り気のあるお客様。** プラットフォーム内で戦略をビルドして実行するトレーダーは気軽に離れません。
- **警告を上売り機会に変える。** cMind は設計上自己ホストされています — 「運用チームになりたくない」顧客の場合、*あなたが*答えです。

## 誰が管理 cMind を購入するか

- **個人クワント & トレーダー**がホストしたいと考えています。→ [トレーダー向け](./for-traders.md)
- **cTrader ブローカー**がクライアント向けのホワイトラベルを実行しています。→ [ブローカー向け](./for-brokers.md)
- **プロップファーム & コピートレードビジネス**がブランド化、監査可能なインフラストラクチャが必要です。

## 「管理 cMind」の実行の意味

3 つのティアを運用します。顧客はブランド化された Web URL を取得します:

| ティア | それは何か | どこで実行されますか |
|---|---|---|
| ステートレス（Web + MCP） | アプリ + API + MCP サーバー | 任意のコンテナプラットフォーム、自動スケール |
| データベース | PostgreSQL | 管理 Postgres（RDS / Flexible Server / あなた自身） |
| ノード群 | cTrader コンテナをビルドして実行 | **VM または Kubernetes — 特権 Docker が必要** |

:::warning 前もってスコープすること
ノードエージェントは cTrader コンテナをビルドして実行するため、**特権 Docker** が必要です。これはサーバーレスコンテナランタイム（Azure Container Apps、AWS Fargate）を*エージェント*に対して除外します — [Kubernetes](./deployment/kubernetes.md)、VM、または EC2 で実行してください。ステートレスティアはどこでも実行されます。
:::

実際のコピペデプロイメントガイドはこれを具体的にします: [クラウド概要](./deployment/cloud.md) ·
[Azure](./deployment/cloud-azure.md) · [AWS](./deployment/cloud-aws.md) ·
[Kubernetes](./deployment/kubernetes.md) · [スケーリング](./deployment/scaling.md)。

## どのように現金化するか

- **管理ホスティングサブスクリプション。** ノード群とバックテスト並行性のサイズ設定された月次 Starter / Team / Business プラン。
- **使用量 & コンピュートメーリング。** バックテストアワー、ライブノードアワー、ストレージを請求してください — あなたがすでに実行するコンテナ群によって自然に測定されます。
- **ホワイトラベル再販ティア。** 完全な再ブランド化（ロゴ、色、PWA、`ShowSiteLink=false`）およびプレミアム機能の有効化に対してより多く請求してください。[機能トグル](./features/feature-toggles.md)経由。→ [ホワイトラベル](./features/white-label.md)
- **管理 AI。** デフォルト AI プロバイダーキーをバンドルして、すべての顧客のユーザーがセットアップなしで AI を取得し、使用量をマークアップするか、持参キーを提供してください。→ [AI 機能](./features/ai.md)
- **Prop-firm & コピートレード収益シェア。** チャレンジとパフォーマンス手数料を実行するファームをホストし、プラットフォームカットを取得してください。→ [Prop-firm](./features/prop-firm.md) ·
  [パフォーマンス手数料](./features/copy-performance-fees.md) ·
  [プロバイダーマーケットプレイス](./features/copy-provider-marketplace.md)
- **セットアップ、オンボード、SLA。** プロフェッショナルサービスとプレミアムサポートを添付してください。

## マルチテナントパターン

- **デプロイメント-per-テナント（推奨）。** 顧客ごとの 1 つのブランド化されたインスタンス — 強い分離、テナントごとのブランディングとデータベース、テナントごとの異なるノード結合トークン。ブランディングは `IOptionsMonitor` から読み取られるため、各インスタンスは独自のアイデンティティを運びます。
  → [マルチテナントブランディング](./white-label-for-business.md#multi-tenant-per-customer-branding) ·
  [ノード検出](./operations/node-discovery.md)
- **共有コントロールプレーン（高度）。** あなた自身のプロビジョニングレイヤーから多数のインスタンスを駆動し、プログラム的にテナントごとのブランディングと機能をシードします。

## 請求用の使用量測定

所有者/管理者のみ **`GET /api/usage`** エンドポイントは、プロバイダーがポーリングして請求できる読み取り専用サマリーを返します — 新しいドメインまたはパーシスタンスなしで、既存の状態を投影します:

```json
{
  "users": { "total": 42 },
  "nodes": { "total": 6, "online": 5 },
  "instances": { "total": 1280, "backtestsRunning": 3, "runsRunning": 11 },
  "cbots": { "total": 210 },
  "tradingAccounts": { "total": 88 }
}
```

テナントデプロイメント毎にポーリングして、シートベース、フリートベース、またはワークロードベースの価格設定を駆動してください。[ログ & 可観測性](./operations/logging.md)とペアリングして、より細かいコンピュート測定を行います。

需要にノードをスケールし、Postgres ティアを共有し、ステートレスティアを自動スケールしてください。必要な運用サーフェスはすでにそこにあります:

- [スケーリング & 自己回復](./deployment/scaling.md)
- [ログ & 可観測性](./operations/logging.md)

## 始め方

1. [クラウドガイド](./deployment/cloud.md)から参照デプロイメントを立ち上げます。
2. テナント毎にテンプレート化（ブランディング + 結合トークン + DB）し、コンピュート使用量への請求を配線します。
3. リストアップしてください — あなたは現在、販売する管理アルゴトレーディングプラットフォームを持っています。

## バックアップを貢献する
