# コピートレーディング

**マスター**cTraderアカウントを1つ以上の**スレーブ**アカウントにミラー — クロスブローカー、クロスcID — 宛先ごとの制御＋マネーグレード調整付き。

## コンセプト

- **コピープロファイル** — 1つのマスター（`SourceAccountId`）＋1つ以上の**宛先**。ライフサイクル：`Draft → Running → Paused → Stopped`（失敗時`Error`）。アグリゲートルート：`CopyProfile`（`CopyDestination`を所有）。
- **宛先** — 1つのスレーブアカウント＋マスターがコピーされる方法の完全なルールセット。すべてのコンフィグ宛先ごとため、1つのマスターは保守的と積極的なスレーブを同時に供給。
- **コピーエンジンホスト** — プロファイル用の実行中ワーカー（`CopyEngineHost`）。マスター実行ストリームを購読し、各イベントをすべての宛先に適用。
- **スーパーバイザー** — `CopyEngineSupervisor`、各ノード上のバックグラウンドサービス。割り当てられたプロファイルをホストし、クラスタ全体で自己修復（[スケーリング](../deployment/scaling.md)を参照）。

## ミラーされるもの

| マスターイベント | スレーブアクション |
|--------------|--------------|
| マーケット/マーケットレンジポジションオープン | サイズ付きコピーを開く（ソースポジションidでラベル付け） |
| リミット/ストップ/ストップリミット未決済注文 | マッチング未決済注文を配置 |
| 未決済注文修正 | ミラーされた未決済注文をプレースで修正 |
| 未決済注文キャンセル/有効期限 | ミラーされた未決済注文をキャンセル |
| 部分的クローズ | スレーブポジションの同じ割合をクローズ |
| スケールイン（ボリューム増加） | 追加ボリュームをオープン（オプトイン） |
| ストップロス/トレーリングストップ変更 | スレーブポジションの保護を修正 |
| 完全クローズ | スレーブコピーをクローズ |

すべてのコピーはソースポジション/注文idで**ラベル付け**。再接続後、ホストは調整から状態を再構築：マスターが保持しているがスレーブが欠落しているコピーをオープンし、マスターがもはや保持していないスレーブ「孤立」をクローズ — **取引を複製することなく**。

## プロファイル作成

コピートレーディングページ上の**新規プロファイル**ダイアログはすべてを前面で収集：プロファイル名、ソース（マスター）アカウント、宛先（スレーブ）アカウント（**すべてを選択**ボタン付きマルチセレクト；選択されたマスターはスレーブリストから除外）、以下の完全な宛先ごとのオプションセット。すべての入力は保存前に**検証済み** — 欠落名/ソース/宛先、非正ジオサイジングパラメータ、負/矛盾したロット境界、範囲外引き出し%、注文タイプ有効なし、空のシンボルフィルタ、または形成不良なシンボルマップペアは、エラーリスト＋ブロック保存を浮上。確認時にプロファイルが作成され、選択されたスレーブすべてが選択された設定で追加されます。

行アクション尊重ライフサイクル：実行されていない場合のみ**開始**有効、実行中場合のみ**停止** + **一時停止**、実行中＋プロファイル＋宛先削除前の確認要求 **削除**無効。

## 宛先ごとのオプション

新規プロファイルダイアログ、コピートレーディングページの宛先ごとのパネル、または`POST /api/copy/profiles/{id}/destinations`で設定：

- **サイジング**（`MoneyManagementMode` + パラメータ）：固定ロット、ロット/名目乗数、比例残高/エクイティ/フリーマージン、固定リスク%、固定レバレッジ、自動比例、**リスク％からストップ**（M7）。加えて最小/最大ロット境界＋強制最小ロット。**リスクからストップ**サイズ宛先が設定パーセント*その独自*残高をリスク、**マスターのストップロス距離**から導出（`マスターリスク2% → スレーブ自動リスク2%`）：`ロット = 残高×% ÷ (ストップ距離 × 契約サイズ)`。マスターオープン**ストップロスなし**距離ない → サイズに対して使用`MaxRiskFallbackLot`（M7）設定されている場合、そうでなければスキップ（`no_stop_loss`）推測しない。比例**エクイティ**/**フリーマージン** サイズリアルアカウント**エクイティ**（`残高 + Σ浮くP&L`、cTrader Open APIあたり導出はエクイティを提供しません）、プレーン残高ではなく — マスターオープン利益/損失に座ってコピーサイズ右。使用マージンは調整APIによって公開されないため、フリーマージンはエクイティとして扱われます（正直な利用可能資金プロキシ）。その他のモードは残高を読み取り、追加再評価ラウンドトリップをスキップします。
- **方向フィルタ**：両方/ロングのみ/ショートのみ。**反転**：側面を反転（＋SL↔ TP交換）反対のコピー。
- **管理のみ**（新規取引を無視/クローズのみ）：ミラークローズ、部分的クローズ既にコピーされたポジションに対する保護変更、しかし**新しい**ポジション/未決済注文をオープン（スキップ`manage_only`）。宛先を既存のコピーをカットすることなく風下に使用。
- **開始時同期オープン** / **開始時同期クローズ**（デフォルト有効）：プロファイルの**初回**再同期時、マスターの既存ポジションのコピーをオープンするか、プロファイルが停止中にマスターがクローズしたコピーをクローズするか。両方開始時のみ適用 — 中実行再接続は常に完全に調整し、デシンク関係なく回復。
- **シンボルマップ** + **シンボルフィルタ**（ホワイトリスト/ブラックリスト）。各シンボルマップエントリはオプションの**シンボルごともボリューム乗数**（cMAMシンボルごとオーバーライド）スケーリングのコピーサイズ宛先のサイジングの上（1 = 変更なし）。マップ全体はインポート/エクスポート**CSV**（`GET …/symbol-map.csv`、`PUT …/symbol-map/csv`；列`Source,Destination,VolumeMultiplier`） — 各行はドメイン値オブジェクトを通じて検証されるため、形成不良ファイルは無効マップを生成できません。
- **取引時間ウィンドウ**（C18）— 宛先ごとの日次UTC ウィンドウ（`開始`/`終了`分/日、終了排他；`開始 == 終了` = 終日）。ウィンドウ外の新しいオープンスキップ（`trading_hours`）。`開始 > 終了`ウィンドウ過去真夜中をラップ（例22:00–06:00）。既存ポジションは管理されたまま。
- **ソースラベルフィルタ**（C18、cTrader同等MT魔法番号フィルタ）— 設定される場合、**正確に**マッチするマスター取引コピーのみ（例1つのボット取引、またはマニュアルのみラベル）。そうでなければスキップ（`source_label`）。空 = すべてコピー。マスターポジション/注文の`TradeData.Label`の`ExecutionEvent.SourceLabel`で運ばれる、再同期も尊重。
- **アカウント保護**（ZuluGuard/グローバルアカウント保護）— 宛先の**ライブエクイティ**（`残高 + Σ浮くP&L`、すべての`CopyDefaults.EquityGuardInterval`投票）をウォッチ`StopEquity`床および/またはオプションの`TakeEquity`天井に対して。侵害時、モード適用：**CloseOnly**（新しいコピー停止、既存管理保持）、**Frozen**（オープン停止）、**SellOut**（宛先上の**すべての**コピーを即座にクローズ）。発火後、宛先ラッチ — ホスト再起動まで新しいオープンなし — `CopyAccountProtectionTriggered`アラート発生。`SellOut`は`StopEquity`必須；`TakeEquity`は`StopEquity`上に座る必須。**保証なし警告：** 売却出しはマーケット実行を使用 — 各競争相手の同等、高速/ギャップ市場で満たし価格を保証できません。
- **フラットンすべてパニックボタン**（C8）— `POST /api/copy/profiles/{id}/flatten`即座にすべての宛先上の**すべての**コピー位置をクローズし、新しいオープンに対してロック。クロスプロセス処理：APIは旗を設定、スーパーバイザーは実行中のホストに配信（トークンローテーションチャネルの再利用）。プラットフォーム。フラグがクリアされるため正確に1回（`CopyFlattenAll`アラート）発火します。ユーザーはプロファイルを一時停止/停止。
- **プロップ会社ルールガード**（C7）— 施行プロップ会社コピアーユーザーリクエスト。宛先ごと、**日次損失キャップ**（日の開始エクイティからの損失）および/または**トレーリング引き出し**制限（実行中のピークエクイティからの損失）、両方が預金通貨。侵害時宛先**自動フラット**（すべてのコピークローズ）+ **ロック**UTC日の残り（新しいオープンスキップ`prop_lockout`）；`CopyPropRuleBreached`アラート発火。ロックはUTC日ロールオーバーときクリア（新しいベースライン/ピーク取得）。ライブエクイティと同じポーリングをアカウント保護として共有。
- **実行ジッター**（C11、デフォルト無効）— ランダム`0..N` ms 各コピー配置前の遅延、ユーザーの**独自**アカウント全体の近い等しい注文タイムスタンプを相関解除。**コンプライアンス警告：** コピーを許可するプロップ会社のサポート — **それを禁止する会社を回避するツール**ではなく；会社のルール内に留まることはあなたの責任。
- **設定ロック**（C9）— 期間凍結宛先のセッティング（`POST …/destinations/{id}/lock` 分で）。ロック中、宛先は削除できません（アグリゲート拒否`CopyDestinationConfigLocked`） — 引き下げ中の衝動的な変化に対する意図的なガード。ロックはそのタイムスタンプで自動的に期限切れ。
- **コンシステンシープレアラート**（C10）— 警告（UTC日あたり1回）宛先の**日次利益**が設定パーセント到達日開始エクイティ（`CopyConsistencyThresholdApproaching`）に達したとき、プロップ会社のコンシステンシールール尊重前に**トリップ**。利益側、損失側ロックアウト独立；プロップルールガードと同じ日ベースラインから実行。
- **注文タイプフィルタ** — マスター注文タイプをコピーするを正確に選択：マーケット、マーケットレンジ、リミット、ストップ、ストップリミット（`CopyOrderTypes`フラグ；デフォルトすべて）。cMAMスタイル選択性。
- **SLコピー / TPコピー** — ミラーマスターのストップロス/テイクプロフィット、または独立して保護を管理。
- **コピートレーリングストップ**、**ミラー部分的クローズ**、**ミラースケールイン** — 各は独立でトグル可能。
- **コピー未決済有効期限**（デフォルト有効）— ミラーマスター未決済注文の満期タイムスタンプ。
- **マスタースリップをコピー**（デフォルト有効）— マーケットレンジ＋ストップリミット注文、スレーブ注文をマスターの正確なスリップインポイント（ベース価格はスレーブのライブスポットから取得）で配置。
- **ガード**：最大引き下げ%、日次損失キャップ、最大コピー遅延、スリップフィルタ（マスター進入からスレーブ価格がN pips超えて移動されている場合はコピーをスキップ）。**最大コピー遅延**マスターイベントのリアルサーバータイムスタンプに対して測定（`ExecutionEvent.ServerTimestamp`）注入`TimeProvider`を通じて：設定された最大ラグより古いシグナルはスキップされ、遅いコピーは後で配置されません（以前の遅延は常にゼロ＋ガード死）。
- **SL/TP精度正規化**（M6）— コピーストップロス/テイクプロフィット価格は**宛先**シンボルの数字精度に前の修正に四捨五入されるため、マスター価格ファイナー精度（またはクロスブローカー数字不一致）サーバーの`INVALID_STOPLOSS_TAKEPROFIT`トリップしません。
- **拒否回路ブレーカー/フォロワーガード**（G8）— 宛先で`CopyDefaults.RejectionBudget`オープンが行で拒否するのは**トリップ**：クールダウンウィンドウ新しいオープンなし（`CopyDestinationTripped`アラート発火）、拒否嵐をハンマリングから停止（プロップ会社）アカウント。既存ポジションはトリップ時管理＋クローズされます。ブレーカーはクールダウン後およびが成功したコピーカウンタの自動リセットをクリア。
- **ロット正気の天井**（C14）— 絶対最大コピーサイズ および/または マスターキャップの倍数。計算コピーが絶対キャップを超過、またはマスターの独自ロットサイズの`N×`超過、**ハードブロック**（`lot_sanity`スキップサーフェス、`cmind.copy.skipped`でカウント）配置されません — 大災害規模クラスに対する防御（0.23-ロット マスター各レシーバに3ロットに変わる暴走乗数または四捨五入バグを通じて）。両方の次元デフォルト`0`（無効）。

## 信頼性＆エッジケース

エンジンは何でもいつでも失敗する可能性がある現実の構築：

- **スレーブ未決済満たし相関タイムアウト**（C13）— ミラーされたスレーブ未決済マスター未決済消えた（休止中もしくは新しく満たされた）相関タイムアウト後にキャンセルされた、スレーブコピーは未相関未管理ポジションに満たし(&`CopyPendingTimedOut`）できません。再同期も注文idラベル満たした孤立クリーンアップ。
- **ロバストクローズ/フラット**（M8）— 孤立クローズ再同期、またはガード侵害時フラット、ブローカーがクローズ位置容認（`POSITION_NOT_FOUND`）：各クローズが独立して実行され、1つの古いidは再同期を中止したり、アカウント未フラットの残りを離すことはできません。

- **すでにマスター取引で開始** — 開始時ホストは調整＋マスターの既存ポジションのコピーをオープン。
- **接続がドロップ/デシンク** — 再接続時ホスト調整：欠落コピーオープン、孤立クローズ、ペンディングの再ラベル。重複注文なし。
- **注文配置失敗** — 1つの宛先での失敗はログ、決してその他宛先をブロック。
- **cID あたり単一有効トークン** — cTrader cID の古いアクセストークン失効瞬間新しい1つ発行。cMind は実行中ホストのトークン**プレースで交換**（ライブソケットで再認証）コピーは ストリームドロップなしで続行。[トークンライフサイクル](token-lifecycle.md)を参照。

## 監査能力

すべてのアクション構造化を発行、ソース生成ログイベント（`LogMessages`）プロファイルid、宛先cID、注文/ポジションid、＋値 — 注文配置/スキップ（理由付き）、部分的クローズ、保護適用、トレーリング適用、未決済配置/修正/キャンセル、有効期限ミラー、マーケットレンジスリップミラー、トークンスワップ、再同期サマリー。これはコンプライアンス＋紛争解決の監査証跡です。

ログに加えて、エンジンは**OpenTelemetryメトリクス**を`cMind.Copy`メーター上で発行します（共有OTelパイプラインに登録、OTLP/Azure Monitorなどにエクスポート）：`cmind.copy.latency`（マスターイベント → ディスパッチ、ms）、`cmind.copy.dispatch.duration`（すべての宛先へのファンアウト、ms）、`cmind.copy.slippage.points`、`cmind.copy.placed`（宛先でタグ付け）、`cmind.copy.skipped`（理由でタグ付け）、+ `cmind.copy.failed`。これはレイテンシ/スリップ回帰を測定可能にします。ログラインに見える日 — ライブスイートはそれらの予算に対して主張。

## API

- `GET /api/copy/profiles` — リスト。
- `POST /api/copy/profiles` — 作成（オプション宛先アカウントidを付き）。
- `GET /api/copy/profiles/{id}` — 完全な詳細宛先オプションすべてを含む。
- `POST /api/copy/profiles/{id}/destinations` — 完全なオプションセットを持つ宛先を追加。
- `DELETE /api/copy/profiles/{id}/destinations/{destinationId}` — 削除。
- `POST /api/copy/profiles/{id}/{start|pause|stop}` — ライフサイクル。
