# 0004 — `CBotBuilder` ウェブホスト でサンドボックスコンテナ で実行

## コンテキスト

ビルド ユーザー cBot 意味 実行 **不信 MSBuild** — 任意コード ビルド時 (ターゲット、ソースジェネレータ、スクリプト復元)。必要 Docker ソケット SDKコンテナ スピンアップ。ノード 実行 トレーディングコンテナ + ビルド権限 キープしない べき。

## 決定

`CBotBuilder` 実行 **ウェブホスト で** (既にDocker ソケット 持つ)、中 **使い捨てSDK コンテナ**:

- バインドマウント `/work` ディレクトリ (のみビルド入出力、ホストファイルシステム ではなく);
- 共有 `app-nuget-cache` ボリューム リストア パフォーマンス;
- ホスト ネットワークアクセス なし 復元 必要 超えて。

不信 MSBuild ホスト ファイルシステム または ネットワーク到達不可。実行/バックテスト コンテナ、対比、実行 ノード 選択 `NodeScheduler`。

## 結果

- ビルド権限 (Docker ソケット) ウェブホスト 限定; ノード のみ実行 許可 トレーディング イメージ。
- 各ビルド 隔離 使い捨てコンテナ — 悪質ビルド 持続不可 または エスケープ。
- ウェブホスト Docker ソケット available必須; これ デプロイメント要件、オプション ではなく。
