# Agent Studio

Agent Studio를 사용하면 **캐릭터를 갖춘 거래 에이전트**를 코딩 없이 생성하고, 측정 가능한 목표를 향해 계정 관리를 맡길 수 있습니다. 에이전트는 شخص성 중심의 cBot과 같습니다: 아키타입과 태도를 선택하고 가드레일을 설정하면, **Autonomy & Safety Kernel** 하에서 실행됩니다.

**AI → Agent Studio** (`/agent-studio`)에서 열립니다.

## 에이전트 생성

**새 에이전트** 대화상자에서 코딩 없이 수집:

- **이름** 및 **아키타입** — Scalper, Day Trader, Swing Trader, Position Trader, News Trader, Contrarian, Mean Reversion, Breakout/Momentum. 각 프리셋은 합리적인 주기와 포즈를 고정합니다.
- **태도** — 공격성, 인내심, 추세 추종 슬라이더.
- **관리 계정** — **최소 하나는 에이전트 생성에 필수**(계정이 없는 에이전트는 절대 시작할 수 없으므로 *Create* 버튼은 하나를 선택할 때까지 비활성 상태). 아직 거래 계정을 연결하지 않았다면, 대화상자에서 이를 알려주고 먼저 하나를 연결하도록 안내합니다.
- **자율성 수준** — **Advisory**(제안만 함) 또는 **Approval-gated**(행동 전 승인 필요). **Full Auto**(거래별 승인 불필요)는 추가로 **리스크Envelope** 필요 및 활성화 전 위험 면책 조항에 동의해야 합니다.

페르소나는 에이전트의 시스템 프롬프트로 **결정론적으로** 컴파일됩니다(LLM이 작성하지 않음). 따라서 동일한 설정은 항상 동일한 지시를 생성합니다 — 재현 가능하고 감사 가능합니다.

## 명단

모든 에이전트가 제어실 테이블에 표시됩니다: **에이전트 이름, 유형, 관리 계정 수, 목표, 실행 상태, 마지막 행동**과 함께 **Start / Stop / Kill** 컨트롤. Kill 스위치는 실행 중인 에이전트를 즉시 중지합니다.

## 안전은 설정이 아닌 도메인 불변량입니다

돈이 오가는 모든 것은 **Autonomy & Safety Kernel**을 통과합니다:

- **리스크Envelope** — 주문별 하드 제한(일일 최대 손실, 공개 노출, 포지션 크기, 레버리지, 연속 손실, 시간당 주문 수, 허용된 심볼). 모든 주문은 발송 전 검증되며, 위반 시 거부(클램프 아님). Full Auto에 도달하려면 필수.
- **서킷 브레이커** — 손실 스트릭, 일일 손실 위반, **하드 성능 목표 위반**, 또는 **AI 제공자 불가용** 시 결정론적으로 새 리스크를 중지합니다(다운되거나 환각하는 모델은 절대 새 포지션을 열지 않음).
- **버전화된 면책 조항 동의** — Full Auto 활성화를 위해 일회성 버전화된 동의를 받아야 합니다(법적 필수 동의, 거래별 승인 아님); 면책 조항이 업데이트되면 재동의가 강제됩니다.
- **Kill 스위치** — 모든 실행 중인 에이전트에 대한 멱등 비상 정지.

## 목표

에이전트에 **측정 가능한 목표**를 부여합니다 — 예: *최대 드로우다운 4% 이하 유지*, *수익률 1.5 이상*, *승률 ≥ 55%*. 각 목표는 **Hard**(가드레일 — 위반 시 서킷 브레이커 작동) 또는 **Soft**(추론에만 영향)이며, On-track / At-risk / Breached로 평가됩니다.

## 의사결정 파이프라인

시작되면 에이전트는 **24/7 감독 루프**(`AgentRuntimeService`)를 실행합니다. 각 틱마다 관리하는 계정마다: **결정론적 계정 상태**를 읽고(그라운드 트루스, 모델 메모리 아님), 의사결정 엔진에 행동 요청, **안전 게이트**(`AgentDecisionProcessor`)에 통과 — 자율성 수준 → 서킷 브레이커 → 리스크Envelope, 추가 전용 **`AgentDecisionRecord`** 작성, 게이트가 지시한 대로 중지 또는 실행. 루프는 **결함 격리**(한 에이전트의 실패가 다른 에이전트나 호스트에 영향 없음) 및 **기본 안전**: AI가 구성되고 `App:Ai:AgentRuntimeEnabled`가 설정되지 않으면 비활성 상태, AI 제공자를 사용할 수 없는 동안 절대 새 리스크를 열지 않음.

- **승인 게이트** — **Approval-gated** 에이전트의 제안 주문은 **Pending**으로 기록되며 소유자가 승인할 때까지(`POST /api/agent-studio/{id}/decisions/{seq}/approve` 또는 `/reject`) 아무 동작 안 함; **Full Auto**는 거래별 승인 없이Envelope 통과; **Advisory**는 제안만 함.
- **감사 원장** — 모든 결정은 재생 가능: 추론(XAI), 인용한 증거, 게이트 verdict, 주문 의도 및 실행 여부, `GET /api/agent-studio/{id}/decisions`에서 확인.
- **리서치 데스크** — 주문형 멀티 에이전트 토론: Alpha/Sentiment/Technical/Risk 분석가가 각각 견해를 제시하고 Reviewer가 제안서를 종합합니다(`POST /api/agent-studio/{id}/debate`).
- **메모리** — 에이전트는 각 결정을 기억하고 다음 프롬프트에 최근 메모리를 재호출하여 연속성을 유지합니다(`GET /api/agent-studio/{id}/memory`).

명단 행의 **Details**는 에이전트의 결정 피드(대기 주문에 대한 Approve/Reject 포함), 메모리, Run-debate 탭을 엽니다.

## 범위

출시됨: 전체 에이전트 수명주기, 결정론적 안전 게이트, 24/7 런타임, 인간-인-루프 승인 게이트, 감사 원장, **라이브 cTrader Open API 통합** — 계정 상태 저장소(실제 잔고, 포지션, 공개 노출 수량 읽기) 및 주문 실행기(실제 시장 주문 배치, 수량→볼륨은 심볼 수량 크기로), 두 가지 모두 각 관리 계정의 OAuth 자격 증명 확인 및 계정이 연결되지 않은 경우 안전하게 저하. **Anthropic API 키 필요** — 모델이 주문 생성까지는 엔진이 유지; 멀티 에이전트 토론 역할 및 계층화 메모리/반성은 아직 제공 예정. 런타임은 `App:Ai:AgentRuntimeEnabled`가 설정되지 않으면 비활성 상태이므로 라이브 거래는 명시적이고 완전히 동의된 옵인에서만 발생합니다.

## 관리 계정 및 편집

에이전트 생성 시 관리할 거래 계정을 선택합니다 — **생성 시 최소 하나는 필수**(선택할 때까지 *Create* 버튼이 비활성화되며, 생성 엔드포인트는 빈 선택을 거부합니다). 모든 에이전트는 이후 **편집** 가능(명단 행의 연필 아이콘에서 이름, 성격, 자율성, 관리 계정). 수명주기 컨트롤(세부정보, 편집, 시작, 중지, 삭제)은 아이콘 버튼이며, 해당 동작이 적용되지 않는 상태에서는 비활성화됩니다.
