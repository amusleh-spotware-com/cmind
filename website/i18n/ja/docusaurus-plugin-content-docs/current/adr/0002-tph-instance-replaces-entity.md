# 0002 — インスタンス状態 TPH; 遷移 エンティティ置換

## コンテキスト

実行/バックテスト インスタンス 移動 状態 (保留 → スケジュール → 開始中 → 実行中 → ターミナル)。モデル状態 EF Core **テーブルあたり階層** (TPH): 各状態 サブタイプ (`StartingRunInstance`, `RunningRunInstance`, ...)。EF の TPH 判別列 **変更不可** 既存行。

## 決定

状態遷移 **エンティティ置換** 新規サブタイプ インスタンス で 代わり ステータスフィールド ミューテート。列置換 のため、 **インスタンス id 変更** 横 開始中 → 実行中 → ターミナル。**コンテナ id 安定** + 実行 横 実行; HTTP ノード エージェント キー コンテナ id でステータス/レポート/停止/ログ。

## 結果

- 各状態 異なる型 のみフィールド + メソッド有効 その状態 — 違法遷移 + ナンセンス フィールドアクセス コンパイルエラー、ランタイムチェック ではなく。
- 呼び出し元 **キャッシュしてはいけない** インスタンス id 遷移横; 使用 コンテナ id 安定 ハンドル ステート横スパン 任意のもの。
- 遷移ロジック ライブ `InstanceTransitions`; id 変更 意図的、バグ ではなく。
