# بناء واختبار cBots

بناء واختبار cBots خاص بـ cTrader (C# **و** Python، كليهما .NET) من محرر Monaco المدمج في المتصفح، يتم التشغيل على صورة `ghcr.io/spotware/ctrader-console` الرسمية.

## البناء

- تستضيف صفحة **Builder** محرر Monaco؛ يقوم `CBotBuilder` بترجمة المشروع باستخدام `dotnet build` **في حاوية مؤقتة** (`AppOptions.BuildImage`، مع ربط مجلد العمل في `/work`)، بحيث لا تصل أهداف MSBuild غير الموثوقة إلى المضيف. يتم تخزين استعادة NuGet مؤقتًا عبر عمليات البناء من خلال مجلد مشترك. يحتاج مضيف الويب إلى وصول مقبس Docker.
- تقع قوالب البدء الخاصة بـ C# و Python في `src/Nodes/Builder/Templates/`.

## التشغيل والاختبار

- **Instances** = هرمية حالة TPH (`Run`/`Backtest` × `Pending`/`Scheduled`/`Starting`/`Running`/`Stopping`/`Stopped`/`Failed`). تحل الانتقال محل الكيان (تغيير المعرف)، مع الاحتفاظ برقم معرف الحاوية.
- يختار `NodeScheduler` أقل عقدة محملة مؤهلة؛ يقوم `ContainerDispatcherFactory` بتوجيه الطلب إلى عامل عقدة HTTP بعيد أو موزع Docker محلي.
- يقوم منسقو الإكمال بمطابقة الحاويات التي خرجت (تخرج حاويات الاختبار تلقائيًا عبر `--exit-on-stop`)؛ الملف موجود → مكتمل (تخزين `ReportJson`)، مفقود → فشل.
- تتدفق السجلات الحية للحاوية إلى المتصفح عبر SignalR؛ يتم تحليل منحنيات الأسهم من التقرير ورسمها على الرسم البياني.

## يتم تخزين بيانات سوق الاختبار مؤقتاً لكل حساب

يقوم cTrader Console بتنزيل بيانات القراد/الشريط التاريخية إلى `--data-dir`. هذا المجلد هو **ذاكرة تخزين مؤقت مستقرة وثابتة مفتاح على حساب التداول** (رقم الحساب الخاص به) — محمول من قرص العقدة في مسار الحاوية الخاص بها (`/mnt/data`)، وهو **جبل منفصل وغير متداخل** من مجلد العمل لكل مثيل. لذلك كل اختبار في نفس الحساب **يعيد استخدام** البيانات المحملة بالفعل بدلاً من إعادة تحميلها في كل تشغيل. (في الماضي، كان مجلد البيانات يعيش تحت مجلد العمل لكل مثيل، الذي يتغير معرفه في كل تشغيل، مما أجبر على تحميل جديد في كل اختبار.) لا يزال مجلد العمل المؤقت لكل مثيل يحتفظ بالخوارزمية والمعاملات وكلمة المرور والتقرير؛ يتم احتساب ذاكرة البيانات المشتركة في استخدام بيانات الاختبار للعقدة ويتم مسحها من خلال إجراء التنظيف للعقدة.

## إعدادات الاختبار

يعرض الحوار **Backtest** كل إعدادات التي يقبلها CLI لـ cTrader Console، لذلك لا تضطر أبداً إلى لمس سطر أوامر:

- **From / To** — نافذة الاختبار (`--start` / `--end`).
- **Data mode** — أحد الأوضاع الثلاثة الخاصة بـ cTrader (`--data-mode`): **Tick data** (`tick`، دقيق)، **m1 bars** (`m1`، سريع)، أو **Open prices only** (`open`، الأسرع).
- **Starting balance** — افتراضي إلى `10000` (`--balance`). **الرصيد 0 لا يضع أي صفقات ويجعل cTrader ينبعث تقرير فارغ ثم يتعطل عليه** ("Message expected")، لذا يتم إرسال رصيد غير صفر دائمًا.
- **Commission** و **Spread** — `--commission` / `--spread` (الانتشار بالنقاط).
- **Data file** (اختياري) — مسار من جانب العقدة إلى ملف بيانات تاريخية (`--data-file`)؛ اتركه فارغًا لاستخدام البيانات المحملة/المخزنة مؤقتًا.
- **Expose environment variables** — تبديل يمرر متغيرات بيئة المضيف إلى cBot (`--environment-variables` flag).

## صفحة تفاصيل المثيل

يوضح فتح مثيل (`/instance/{id}`) حالته الحية والسجلات و — في حالة الاختبار — منحنى الأسهم. **يعكس عنوان علامة المتصفح** مثيل محدد (**اسم cBot · الوضع · الرمز**، على سبيل المثال `TrendBot · Backtest · EURUSD`) بحيث يكون من الممكن تمييز علامة تشغيل مباشر وعلامة اختبار بسهولة. يتم تتبع تشغيل واختبار نفس cBot كـ **lineages** مختلفة (معرف lineage مستقر محمول عبر انتقالات الحالة)، لذلك تتبع الصفحة مثيل واحد بالضبط ولا تخلط بين بيانات التشغيل واختبار.

## عناصر تحكم دورة حياة المثيل

لكل صف مثيل (وصفحة التفاصيل الخاصة به) تحكم صحيح الحالة. يظهر مثيل **active** **Stop**؛ يعرض واحد **terminal** (Stopped / Completed / Failed) **Start (▶)** لإعادة إطلاقه بنفس cBot والحساب والرمز والإطار الزمني وحزمة المعاملات والصورة (يعاد تشغيل التشغيل كتشغيل، واختبار كاختبار). سيؤدي النقر على Stop إلى عرض إشعار "Stopping…" وتعطيل الرمز حتى يتم حله، وسيظهر تشغيل تم إنشاؤه حديثًا في القائمة فورًا — بدون إعادة تحميل الصفحة.

يتم **الاحتفاظ بسجلات وحدة التحكم عند انتهاء مثيل** — للتشغيل (على Stop) والاختبار (عند الإكمال) على حد سواء — بحيث تبقى سجلات التشغيل الأخيرة قابلة للعرض على صفحة التفاصيل و، من خلال شريط أدوات السجل، **نسخ إلى الحافظة** (رمز Copy logs) أو **تنزيل** (رمز Download logs) حتى بعد زوال الحاوية. يعمل كلاهما على السجل الكامل للمثيل، وليس فقط ذيل الشاشة.

لم يتم بناء **uploaded** `.algo` هنا أبدًا، لذا يتم ترك **Last Build** column على صفحة cBots فارغ (يُظهر وقت البناء فقط لـ cBots التي تبنيها في المتصفح).

## تحرير وإعادة تشغيل مثيل متوقف

يحتوي **stopped** instance (تشغيل أو اختبار) على **Edit** control — أيقونة على صفه في القائمة **و** بجانب Start/Stop على صفحة تفاصيله — تفتح حوار **مملوء مسبقًا** بتكوينه الحالي. يمكنك تغيير **حساب التداول والرمز والإطار الزمني وحزمة المعاملات وعلامة الصورة** (و، للاختبار ، **النافذة وجميع إعدادات الاختبار** أعلاه)، ثم **Save & start** تعيد تشغيله بالإعدادات الجديدة (استبدال المثيل المتوقف). يتم **تعطيل التحكم أثناء نشاط المثيل** — يمكن تحرير مثيل متوقف فقط.

## التشغيل من محرر الأكواد

سيؤدي النقر على **Run** في محرر الأكواد إلى فتح حوار بدلاً من فتح تشغيل أعمى مرمز بشدة:

- **Trading account** (مطلوب) — حساب cTrader الذي يتصل به cBot.
- **Parameter set** (اختياري) — اختر مجموعة موجودة، أو اتركها فارغة للتشغيل مع **قيم المعاملات الافتراضية** الخاصة بـ cBot. يقوم زر **+** بجانب المحدد بإنشاء مجموعة معاملات جديدة مضمنة (انظر أدناه) واختيارها.
- **Symbol / Timeframe** افتراضي إلى `EURUSD` / `h1` ويمكن تغييره؛ **Cancel** أو **Run**.

عند **Run** يحفظ المحرر + ينشئ المصدر الحالي، يبدأ المثيل على الحساب المختار بالمعاملات المختارة، ثم يتابع سجلات الحاوية الحية. (يقوم تدفق السجل بإعادة توجيه ملف تعريف الارتباط للمصادقة للمستخدم الذي تم تسجيل دخوله إلى `/hubs/logs` SignalR hub، بحيث يتصل بدلاً من الفشل مع `Invalid negotiation response received`.)

## مجموعات المعاملات

**parameter set** هي مجموعة مسماة وقابلة لإعادة الاستخدام من تجاوزات معاملات cBot المخزنة كـ JSON مسطح يعيّن اسم كل معامل إلى قيمة عددية، على سبيل المثال `{"Period": 14, "Label": "trend"}`. وقت التشغيل/الاختبار، يتم تحويله إلى ملف cTrader `params.cbotset` (`{ "Parameters": { … } }`). يمكنك إنشاء/تحرير مجموعة كـ JSON خام من حوار **Parameter sets** الخاص بـ cBot أو مضمن من حوار Run.

كل مجموعة معاملات **تنتمي إلى cBot**: يسرد حوار New Parameter Set جميع cBots الخاصة بك ويجب عليك **اختيار واحد** — يتم حجب الإنشاء حتى يتم اختيار cBot. **اسم مجموعة فريد لكل cBot**: إنشاء أو إعادة تسمية مجموعة إلى اسم مجموعة أخرى من نفس cBot بالفعل يستخدم تم رفضه (خطأ واضح في الحوار، `409 Conflict` في API). قد يتم إعادة استخدام نفس الاسم على **مختلف** cBot.

يتم **التحقق من** JSON على الحفظ: يجب أن يكون كائنًا مسطحًا واحدًا تكون قيمه عددية (سلسلة/رقم/منطقي). جذر non-object، array، كائن متداخل، قيمة `null`، أو JSON غير صحيح يتم رفضه (خطأ واضح في الحوار، `400 Bad Request` في API). كائن فارغ `{}` مسموح به ويعني "لا تجاوزات".

## ملاحظات CLI خاص بـ cTrader Console

تحتاج الاختبارات إلى `--data-mode` (افتراضي `m1`)، التواريخ كـ `dd/MM/yyyy HH:mm`، و `params.cbotset` JSON حجة موضعية؛ `run` ترفض `--data-dir` (اختبار فقط). انظر `ContainerCommandHelpers`.

## العقد والتوسع

تتسع السعة التنفيذية بإضافة وكلاء عقدة (التسجيل الذاتي + نبضة القلب). انظر [node discovery](../operations/node-discovery.md) و [scaling](../deployment/scaling.md).

## حساب تداول مطلوب

يحتاج تشغيل أو اختبار cBot إلى حساب تداول cTrader للاتصال به. حتى تضيف واحدًا ضمن **Trading accounts**، يتم تعطيل أزرار **Run New cBot** / **Backtest New cBot** (مع تلميح) وتعرض الصفحة موجه يرتبط بإعداد الحساب — لم تعد تصطدم بخطأ `stream connect failed` خام من bot بدون حساب.
