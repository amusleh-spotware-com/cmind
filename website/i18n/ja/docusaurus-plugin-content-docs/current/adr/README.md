# アーキテクチャ決定記録

これらの記録は、コードからは**推論できない**設計判断を文書化したものです — トレードオフ、採用されなかった道筋、そしてその理由です。すべて短くまとめられています：*コンテキスト → 決定 → 結果*。新しい構造的決定ができた場合は、ここにADRを追加してください（次の番号）。そうすることで、次のエンジニア（人間またはAI）は結果だけでなく、その根拠を継承できます。
