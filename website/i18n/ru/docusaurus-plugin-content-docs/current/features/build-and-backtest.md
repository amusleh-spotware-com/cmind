# Build & backtest cBots

Создавайте, запускайте, тестируйте на истории cBots cTrader (C# **и** Python, оба .NET) из встроенного редактора Monaco, запускайте на официальном образе `ghcr.io/spotware/ctrader-console`.

## Build

- Страница **Builder** содержит редактор Monaco; `CBotBuilder` компилирует проект с помощью `dotnet build` **в одноразовом контейнере** (`AppOptions.BuildImage`, рабочая директория bind-mount в `/work`), поэтому untrusted user MSBuild не может достичь хоста. NuGet restore кэшируется между сборками через общий volume. Web хост должен иметь доступ к Docker socket.
- Начальные шаблоны C# + Python находятся в `src/Nodes/Builder/Templates/`.

## Run & backtest

- **Instances** = иерархия состояния TPH (`Run`/`Backtest` × `Pending`/`Scheduled`/`Starting`/`Running`/`Stopping`/`Stopped`/`Failed`). Переход заменяет сущность (изменение id), container id переносится.
- `NodeScheduler` выбирает наименее загруженный подходящий node; `ContainerDispatcherFactory` маршрутизирует на удаленный HTTP агент node или локальный Docker dispatcher.
- Completion pollers согласуют завершенные контейнеры (backtest контейнеры самостоятельно завершаются через `--exit-on-stop`); отчет присутствует → завершено (сохранить `ReportJson`), отсутствует → failed.
- Логи живого контейнера потоком передаются в браузер через SignalR; кривые капитала backtest парсятся из отчета + отображаются на графике.

## Backtest market data is cached per account

cTrader Console загружает исторические tick/bar данные в свой `--data-dir`. Эта директория — **стабильный, постоянный кэш, привязанный к торговому счету** (его номер счета) — bind-mounted с диска node в его собственный путь контейнера (`/mnt/data`), **отдельное, не вложенное монтирование** от рабочей директории для каждого instance. Так что каждый backtest на одном счете **повторно использует** уже загруженные данные вместо повторной загрузки при каждом запуске. (Ранее data dir находился под рабочей директорией для каждого instance, чей id меняется при каждом запуске, что вынуждало свежую загрузку для каждого backtest.) Временная рабочая директория для каждого instance по-прежнему содержит algo, params, password и report; общий кэш данных учитывается в использовании backtest-data на node и очищается действием node-clean.

## Backtest settings

Диалог **Backtest** предоставляет каждый параметр, который принимает cTrader Console backtest CLI, поэтому вам никогда не нужно трогать командную строку:

- **From / To** — окно backtest (`--start` / `--end`).
- **Data mode** — один из трех режимов cTrader (`--data-mode`): **Tick data** (`tick`, точный), **m1 bars** (`m1`, быстрый) или **Open prices only** (`open`, самый быстрый).
- **Starting balance** — по умолчанию `10000` (`--balance`). **Нулевой баланс не совершает никаких сделок и заставляет cTrader выдавать пустой отчет, который затем падает** ("Message expected"), поэтому отправляется ненулевой баланс.
- **Commission** и **Spread** — `--commission` / `--spread` (spread в pips).
- **Data file** (опционально) — путь на node для файла исторических данных (`--data-file`); оставьте пустым, чтобы использовать загруженные/кэшированные данные.
- **Expose environment variables** — переключатель, который передает переменные окружения хоста в cBot (флаг `--environment-variables`).

## Instance detail page

Открытие instance (`/instance/{id}`) показывает его live статус, логи и — для backtest — кривую капитала. **Заголовок вкладки браузера** отражает конкретный instance (**имя cBot · kind · symbol**, например `TrendBot · Backtest · EURUSD`) поэтому вкладка live-run и вкладка backtest различаются с первого взгляда. Run и backtest одного cBot отслеживаются как отдельные **lineages** (стабильный lineage id переносится через переходы состояния), поэтому страница следует ровно одному instance и никогда не смешивает данные run с данными backtest.

## Instance lifecycle controls

Каждая строка instance (и его страница деталей) имеет управление, соответствующее состоянию. **Активный** instance показывает **Stop**; **терминальный** (Stopped / Completed / Failed) показывает **Start (▶)** для перезапуска с тем же cBot, счетом, symbol, timeframe, parameter set и image (run перезапускается как run, backtest как backtest). Нажатие Stop показывает уведомление "Stopping…" и отключает значок до разрешения, и новый запущенный run немедленно появляется в списке — без перезагрузки страницы.

Логи консоли **сохраняются при завершении instance** — для run (при Stop) и для **backtest** (при завершении) — поэтому логи последнего запуска остаются просматриваемыми на странице деталей и, через панель инструментов логов, **скопированы в буфер обмена** (значок Copy logs) или **загружены** (значок Download logs) даже после исчезновения контейнера. Оба действуют на полный логи консоли instance, а не только видимую часть.

Загруженный `.algo` никогда не был построен здесь, поэтому его колонка **Last Build** на странице cBots остается пустой (она показывает время сборки только для cBots, которые вы создаете в браузере).

## Edit & re-run a stopped instance

**Остановленный** instance (run или backtest) имеет управление **Edit** — значок на его строке в списке **и** рядом с Start/Stop на его странице деталей — который открывает диалог **предзаполненный** его текущей конфигурацией. Вы можете изменить **торговый счет, symbol, timeframe, parameter set и image tag** (и, для backtest, **окно и все параметры backtest** выше), затем **Save & start** перезапускает его с новыми параметрами (заменяя остановленный instance). Управление **отключено, пока instance активен** — только остановленный instance может быть отредактирован.

## Run from the code editor

Нажатие **Run** в редакторе кода открывает диалог вместо слепого запуска с жестко закодированными параметрами:

- **Trading account** (требуется) — счет cTrader, к которому подключается cBot.
- **Parameter set** (опционально) — выберите существующий набор или оставьте пустым для запуска с **значениями параметров cBot по умолчанию**. Кнопка **+** рядом с селектором создает новый parameter set встроенно (см. ниже) и выбирает его.
- **Symbol / Timeframe** по умолчанию `EURUSD` / `h1` и могут быть изменены; **Cancel** или **Run**.

На **Run** редактор сохраняет + собирает текущий исходный код, запускает instance на выбранном счете с выбранными параметрами, затем читает логи живого контейнера. (Поток логов передает cookie аутентификации вошедшего пользователя в SignalR hub `/hubs/logs`, поэтому он подключается вместо отказа с `Invalid negotiation response received`.)

## Parameter sets

**Parameter set** — это именованный, переиспользуемый набор переопределений параметров cBot, сохраненный как плоский JSON объект, отображающий каждое имя параметра на скалярное значение, например `{"Period": 14, "Label": "trend"}`. При запуске/backtest время он преобразуется в cTrader файл `params.cbotset` (`{ "Parameters": { … } }`). Вы можете создать/отредактировать набор как raw JSON из диалога **Parameter sets** cBot или встроенно из диалога Run.

Каждый parameter set **принадлежит cBot**: диалог New Parameter Set перечисляет всех ваших cBots и вы **должны выбрать один** — создание блокируется, пока cBot не выбран. **Имя** набора уникально на cBot: создание или переименование набора в имя, которое уже использует другой набор того же cBot, отклоняется (четкая ошибка в диалоге, `409 Conflict` в API). Это же имя может быть повторно использовано на **другом** cBot.

JSON **проверяется** при сохранении: это должен быть один плоский объект, значения которого все скалярные (string / number / bool). Не-объект в корне, массив, вложенный объект, значение `null` или malformed JSON отклоняется (четкая ошибка в диалоге, `400 Bad Request` в API). Пустой объект `{}` разрешен и означает "без переопределений".

## cTrader Console CLI notes

Backtests требуют `--data-mode` (по умолчанию `m1`), даты как `dd/MM/yyyy HH:mm` и JSON позиционный аргумент `params.cbotset`; `run` отклоняет `--data-dir` (только для backtest). См. `ContainerCommandHelpers`.

## Nodes & scale

Емкость выполнения масштабируется путем добавления node agents (самостоятельная регистрация + heartbeat). См. [node discovery](../operations/node-discovery.md) и [scaling](../deployment/scaling.md).

## A trading account is required

Запуск или backtest cBot требует торгового счета cTrader для подключения. Пока вы не добавите его под **Trading accounts**, кнопки **Run New cBot** / **Backtest New cBot** отключены (с подсказкой) и страница показывает подсказку со ссылкой на настройку счета — вы больше не получаете raw ошибку `stream connect failed` от bot без счета.
