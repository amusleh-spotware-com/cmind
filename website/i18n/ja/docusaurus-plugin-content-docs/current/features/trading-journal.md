# Trading Journal & Coach

トレーディングのためのAIの newest 真有价值なカテゴリは市場を予測すること而非 — *自身の*行動を分析することです。Trading Journalは実行とバックテストの履歴を正直なフィードバックに変換因此 既に持っている戦略を改善できます。

**AI → Trading Journal**（`/journal`）で開きます。

## 何を表面化するか

インスタンス（実行とバックテスト）から、次のように決定論的に計算します：

- バックテスト全体の勝ち/負け/失敗カウントと勝率；
- **行動的洞察** — 静かに零售トレーダーにコストをかける漏れ：
  - **過集中** — 活動の大部分が1つのシンボルにある；
  - **反復失敗** — 高率の実行がビルドまたは設定に失敗；
  - **損失バイアス** — 負け多于りバックテスト（Integrity Labを実行してedgeがrealかどうか確認する nudging付き）；
  - 上記のいずれもない場合は清爽な bill of health。

```http
GET /api/journal
```

## これが信頼できる理由

行動分析は純粋で決定論的なドメインベコード（`Core.Journal`）インフラ依存なし — 過集中、反復失敗、損失バイアス、均衡ケース、空のアカウントのユニットテスト済み。事実は第一です；AI coach（Portfolio Digest）はオプショナルな narrative layer上で、Anthropic APIキーでゲートのため、日記はAI構成なしで完全に動作します。
