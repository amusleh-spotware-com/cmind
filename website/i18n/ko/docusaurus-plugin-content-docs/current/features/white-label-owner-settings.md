# 소유자 설정의 화이트라벨 옵션

배포가 구성 (`appsettings`/env)을 통해 설정할 수 있는 모든 화이트라벨 옵션은 **재배포 없이** **설정 → 배포**에서 앱 소유자가 런타임에 **설정 가능**합니다. 소유자 재정의는 **구성을 이기고**; 클리어하면 옵션이 배포의 구성 (또는 기본 제공 기본값)으로 되돌립니다.

이는 화이트라벨 *배포*가 제품을 구성하는 방식을 미러링합니다 — 동일한 노브, 동일한 효과 — 따라서 운영자는 브랜딩, 게이트 및 정책을 실시간으로 조정하고 결과를 즉시 볼 수 있습니다.

## 위치

- **UI:** 설정 대화의 소유자 전용 **배포** 섹션 및 깊은 링크 가능 페이지 **`/settings/deployment`**. 옵션은 **카테고리별 탭** (브랜딩, 테마, 기능, 등록, 계정, 이메일, AI, Open API, Prop 펌)으로 그룹화되며, 모바일 우선, 데스크톱에서 창이 있는 대화 및 휴대폰에서 전체 화면 표면.
- **API:** `/api/whitelabel` (소유자 전용, 기능 게이트되지 않음):
  - `GET /api/whitelabel` — 모든 옵션과 효과적 값, 제공처 (`Config` / `Owner` / `Default`) 및 재정의 설정 여부. **비밀은 마스킹됨** (값은 절대 반환되지 않음).
  - `PUT /api/whitelabel/{key}` `{ "value": "…" }` — 재정의 설정 (옵션 종류별로 검증). **비밀**에 대한 공백 값은 기존 비밀을 유지합니다.
  - `DELETE /api/whitelabel/{key}` — 하나의 재정의 클리어 (구성으로 되돌림).
  - `POST /api/whitelabel/reset` — **모든** 재정의 클리어 (배포를 순수 구성으로 되돌림).

## 재정의가 적용되는 방식

소유자 재정의는 암호화된-필요한 곳 `AppSetting` 행으로 저장되고 데코레이트된 `IOptionsMonitor<AppOptions>`로 바인딩된 `AppOptions` 위에 계층화됩니다. 모든 컨슈머가 이미 해당 모니터를 통해 옵션을 읽으므로 재정의는 **라이브**로 전체 앱에 적용됩니다 — 테마, 페이지 제목, MFA 게이트, AI 공급자 게이트, 브로커 허용 목록, 등록 정책, 이메일 전송 설정 등이 다음 읽기에서 업데이트됩니다 (테마/브랜딩은 즉시 다시 렌더링). 데이터베이스가 잠시 사용 불가능하면 계층은 구성된 기준선으로 **실패 개방**되므로 재정의 읽기는 앱을 깨뜨릴 수 없습니다.

**기능 플래그**는 동일한 표면의 일부이지만 기존 기능 재정의 저장소 (`IFeatureGate`)를 통해 유지되므로 기능 탭과 독립 실행형 기능 토글은 절대 갈라집니다.

**비밀** (SMTP 암호, CAPTCHA 비밀, 프로비저닝 비밀)은 저장 시 암호화됩니다 (`ISecretProtector`, 목적 `whitelabel.secret`), UI에서 쓰기 전용, API에서 절대 반환되지 않습니다.

## 위임된 옵션

**공유 Open API 응용 프로그램** 자격 증명 및 **메시지 유형별 속도 제한**은 **Open API** 설정 섹션에서 관리됩니다 (복사 거래 / Open API 문서 참조). 배포 카탈로그에서 *위임된* 항목 (읽기 전용, 링크 포함)으로 표시되므로 중복되지 않으며 동기 보장은 여전히 이를 포함된 것으로 계산합니다.

## 항상 동기화됨 (적용됨)

새로운 화이트라벨 옵션을 구성에 추가하면 동일한 커밋에서 소유자 설정에 표면화되어야 합니다. `WhiteLabelCatalogParityTests`에 의해 적용됩니다: 모든 화이트라벨 옵션 레코드 속성에 대해 반영되고 속성이 `Core/WhiteLabel/WhiteLabelCatalog`에 등록되지 않으면 빌드를 실패합니다 (또는 `IntentionallyExcluded`에 이유와 함께 명시적으로 나열). `CLAUDE.md`의 mandate 10을 참조하세요.

## 노트

- 처음에 **메일을 구성하지 않은** 배포에서 SMTP를 활성화하려면 재시작이 필요합니다 (발송자 유형은 시작 시 선택됨); 이미 구성된 발송자의 호스트/자격 증명은 실시간으로 업데이트됩니다.
- 옵션 **레이블/설명**은 데이터로 표시되는 기술 구성 노브 식별자입니다; 탭 레이블 및 모든 대화형 크롬은 완전히 로컬라이즈됩니다.
