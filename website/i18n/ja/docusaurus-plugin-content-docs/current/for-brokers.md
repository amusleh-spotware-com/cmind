# cTrader ブローカー向け cMind 🏦

cTrader ブローカレッジを実行しています。クライアントは既にトレードできます — しかし他のすべてのブローカーのクライアントもトレードできます。**cMind はトレーダーに完全な AI 搭載トレーディング運用プラットフォームをあなた自身として**ブランド化したものを渡します。彼らはあなたのエコシステムの内部で戦略をビルド、バックテスト、実行、コピー、監視します。第三者ツールへ漂流する代わりに。それはより粘り気のあるクライアント、より多くのボリューム、そしてターミナル以外に何も提供していないブローカーより優位に立つ実際のエッジです。

:::tip TL;DR
クライアント向けのホワイトラベル cMind を実行してください。アカウントを**あなたの**ブローカレッジに制限し、AI とコピートレードをオンにし、あなたのブランド下で出荷してください。→ [ビジネス向けホワイトラベル](./white-label-for-business.md)
:::

## 他のブローカーより得られるエッジ

- **スプレッドだけでなく、工具で差別化してください。** クライアントに AI cBot 生成、管理クラスタ上のバックテスト、コピートレード、prop-firm チャレンジを提供してください — ほとんどのブローカーが単に提供していない機能。
- **クライアントをあなたのエコシステムに保ってください。** トレーダーがあなたのブランド化されたプラットフォーム内で戦略をビルドして実行するとき、彼らは滞在します。保有率がゲーム全体です。
- **あなたのブランドの下で、あなたのドメイン上で。** 名前、ロゴ、色、ファビコン、さらには インストール可能な電話アプリ — すべてあなたのもの。誰も「cMind」を見ません。→ [ホワイトラベル機能](./features/white-label.md)

## あなたのアカウントのみを提供する（ブローカー許可リスト）

*あなたの*クライアント向けのホワイトラベルを実行していますか？ユーザーが追加できるブローカーのトレーディングアカウントを制限して、デプロイメントが あなたの本だけを提供するようにしてください:

```json
{
  "App": {
    "Accounts": {
      "AllowedBrokers": ["Your Brokerage Name"]
    }
  }
}
```

許可リストが設定されている場合、cMind はユーザーが追加しようとするすべてのアカウントをチェックします — cTrader Open API 経由とマニュアル cID ログイン経由の両方（アカウントの実際のブローカー名を読むことで検証）— リストにない任意のアカウントを拒否します。空のままにするとすべてのブローカーが許可されます（デフォルト）。完全なメカニクスについては[ホワイトラベル機能ドキュメント](./features/white-label.md#broker-allowlist)を参照してください。

## すべてのユーザーの 1 つの Open API アプリを出荷する

ユーザーごとの手間をスキップします: **1 つの cTrader Open API アプリケーション**を提供し、すべてのクライアントがそれを通じてアカウントを承認します — クライアントは独自のものを登録することはできません。単一のリダイレクト URL を登録し、資格情報を設定または所有者設定にドロップし、共有モードがすべての人のために有効になります。cTrader メッセージ制限がより高い交渉されましたか？**メッセージタイプごとのクライアントレート制限**を調整します（またはペーシングを無効にします）。→ [共有 Open API アプリケーション & レート制限](./features/open-api-shared-app.md)

## 現金化する新しい方法

- **AI、クライアントのために摩擦なし。** デプロイメントレベルでデフォルト AI プロバイダーキーを提供し、すべてのクライアントが AI 機能を直ちに取得します — 他の場所でサインアップなし。それをマークアップするか、プレミアムティアにバンドルしてください。クライアントは独自のキーを持参できます。→ [AI 機能](./features/ai.md)
- **Prop-firm チャレンジ。** ライブエクイティトレッキングと実施ルール付き資金提供トレーダーチャレンジを実行し、エントリを請求してください。→ [Prop-firm ルール](./features/prop-firm.md)
- **コピートレード事業。** パフォーマンス手数料とプロバイダーマーケットプレイスがコピートレードを収益に変えます。→ [パフォーマンス手数料](./features/copy-performance-fees.md) ·
  [プロバイダーマーケットプレイス](./features/copy-provider-marketplace.md)
- **機能階層。** [機能トグル](./features/feature-toggles.md)を使用して各クライアントセグメントが見る機能を決定します。

## 規制、監査可能、マルチテナント

- **[コンプライアンス](./features/compliance.md)**ログはあなたの規制当局が尋ねる監査証跡を提供します。
- **[2要素認証](./features/two-factor-auth.md)**はデプロイメント毎に必須にすることができます。
- **クライアントごとのブランディング** — あなた自身の制御プレーンから駆動される、セグメントごとに分離されたブランド化されたインスタンスを実行します。→ [マルチテナントブランディング](./white-label-for-business.md#multi-tenant-per-customer-branding)

## 始め方

1. 60 秒の再ブランド化に関して[ビジネス向けホワイトラベル](./white-label-for-business.md)を読んでください。
2. `App:Accounts:AllowedBrokers` をあなたのブローカレッジに設定し、[機能セット](./features/feature-toggles.md)を選択してください。
3. [デプロイ](./deployment/cloud.md)してください — Docker、Kubernetes、Azure、または AWS。

インフラストラクチャを自分で実行したくないですか？ホスティングプロバイダーはあなたのための管理 cMind を運用できます —
[クラウド & VPS プロバイダー向け](./for-cloud-providers.md)を指摘してください。

## ロードマップを形成する

cMind はオープンソースです。それを上に構築するブローカーはそれがどこへ行くかについて外寸の発言を得ます — あなたが必要とする統合とコントロールをリクエストし、[貢献ガイド](./contributing.md)を通じてそれをバックアップしてください。
