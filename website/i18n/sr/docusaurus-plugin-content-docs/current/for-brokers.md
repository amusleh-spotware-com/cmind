# cMind за cTrader брокере 🏦

Водиш cTrader брокерску кућу. Твоје клијенти могу већ трговати — али и клијенти сваког другог брокера могу.
**cMind те пушта да даш твој трговцима пуну AI-покренуту трговачку операциону платформу, брендирану као
твој** тако да граде, тестирају, покрећу, копирају и надзиру стратегије унутар *твог* екосистема
уместо да скитају на алат треће странке. То су лепљивији клијенти, више волумена, и правих предност над
брокерима који нуде ништа осим терминала.

:::tip TL;DR
Покрени белоетиетну cMind за твоје клијенте. Ограничи налоге на **твој** брокериск ход, укљу на AI и
копирање трговања, и пошаљи га под твојом марком. → [Белоетиетна за бизнис](./white-label-for-business.md)
:::

## Предност коју добиваш над другим брокерима

- **Разлика на алатима, не само на спредовима.** Дај клијентима AI cBot генерисање, тестирање на
  управљаном кластеру, копирање трговања, и prop-firm изазове — способности већина брокери једноставно не
  нуде.
- **Задржи клијенте у твом екосистему.** Када трговци граде и покрећу своје стратегије унутар твоје брендиране
  платформе, они остају. Задржавање је цела игра.
- **Под твојом марком, на твом домену.** Име, логотип, боје, favicon, чак и инсталабилна телефонска апликација —
  сви твоји. Никад не види "cMind." → [Белоетиетна функција](./features/white-label.md)

## Служе само твоје налоге (браборски одобрени список)

Трчи белоетиетну за *твоје* клијенте? Ограничи које брокере трговачке налоге корисници могу додати тако
твој развој само служи твоју књигу:

```json
{
  "App": {
    "Accounts": {
      "AllowedBrokers": ["Your Brokerage Name"]
    }
  }
}
```

Када је дозвољени списак постављен, cMind проверава сваки налог тај корисник покушава додати — обоје преко cTrader Open
API и преко ручног cID пријаве (проверава читањем правог имена брокера налога) — и одбија било какав
налог који није на твој листи. Остави га празан и сви брокери су дозвољени (подразумевано). Видите
[Белоетиетна функција док](./features/white-label.md#broker-allowlist) за цели механику.
