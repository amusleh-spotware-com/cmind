# cBot 빌드 & 백테스트

cTrader cBot (C# **그리고** Python, 모두 .NET)를 브라우저 내 Monaco IDE에서 빌드, 실행, 백테스트하고 공식 `ghcr.io/spotware/ctrader-console` 이미지에서 실행합니다.

## 빌드

- **Builder** 페이지는 Monaco 에디터를 호스트하고, `CBotBuilder`는 **임시 컨테이너에서** `dotnet build`로 프로젝트를 컴파일합니다 (`AppOptions.BuildImage`, 작업 디렉토리는 `/work`에 바인드-마운트됨). 신뢰할 수 없는 사용자 MSBuild 대상이 호스트에 도달할 수 없도록 합니다. NuGet 복원은 공유 볼륨을 통해 빌드 간에 캐시됩니다. 웹 호스트는 Docker 소켓 접근 권한이 필요합니다.
- C# + Python 스타터 템플릿은 `src/Nodes/Builder/Templates/`에 있습니다.

## 실행 & 백테스트

- **Instances** = TPH 상태 계층 구조 (`Run`/`Backtest` × `Pending`/`Scheduled`/`Starting`/`Running`/`Stopping`/`Stopped`/`Failed`). 전환은 엔티티를 대체합니다 (id 변경), 컨테이너 id는 유지됩니다.
- `NodeScheduler`는 최소 부하의 적합한 노드를 선택합니다. `ContainerDispatcherFactory`는 원격 노드 HTTP 에이전트 또는 로컬 Docker 디스패처로 라우팅합니다.
- 완료 폴러는 종료된 컨테이너를 조정합니다 (백테스트 컨테이너는 `--exit-on-stop`을 통해 자체 종료됨). 보고서 존재 → 완료 (ReportJson 저장), 없음 → 실패.
- 라이브 컨테이너 로그는 SignalR을 통해 브라우저로 스트리밍됩니다. 백테스트 equity 곡선은 보고서에서 파싱되어 차트로 표시됩니다.

## 백테스트 시장 데이터는 계정별로 캐시됨

cTrader Console은 과거 틱/바 데이터를 `--data-dir`로 다운로드합니다. 이 디렉토리는 **거래 계정별로 키를 지정한 안정적이고 지속적인 캐시**입니다 (계정 번호) — 노드의 디스크에서 자체 컨테이너 경로 (`/mnt/data`)에 바인드-마운트되며, 인스턴스별 작업 디렉토리와 **별도의 중첩되지 않은 마운트**입니다. 따라서 동일한 계정의 모든 백테스트는 각 실행마다 다시 다운로드하는 대신 **이미 다운로드된 데이터를 재사용**합니다 (이전에는 데이터 디렉토리가 인스턴스별 작업 디렉토리 아래에 있었으므로, 실행할 때마다 id가 변경되어 각 백테스트마다 새로운 다운로드를 강제했습니다). 임시 인스턴스별 작업 디렉토리는 여전히 알고리즘, 매개변수, 비밀번호 및 보고서를 보유하고 있습니다. 공유 데이터 캐시는 노드의 백테스트 데이터 사용량에 계산되며 노드 정리 작업으로 지워집니다.

## 백테스트 설정

**Backtest** 대화 상자는 cTrader Console 백테스트 CLI가 허용하는 모든 설정을 노출하므로 명령줄을 건드릴 필요가 없습니다:

- **From / To** — 백테스트 윈도우 (`--start` / `--end`).
- **Data mode** — `m1` (1분 바) 또는 `tick` (`--data-mode`).
- **Starting balance** — 기본값은 `10000` (`--balance`). **0 잔액은 거래를 하지 않으며 cTrader가 빈 보고서를 내보내고 이에 충돌**합니다 ("Message expected"). 따라서 0이 아닌 잔액이 항상 전송됩니다.
- **Commission** 및 **Spread** (`--commission` / `--spread`, 스프레드는 pips 단위).
- **Advanced options** — cTrader가 지원하는 다른 백테스트 옵션을 위한 자유 형식 `name=value` 라인별 상자 (예: `applyCommissionAutomatically=true`). 각 라인은 `--name value` CLI 인수가 됩니다.

## 인스턴스 상세 페이지

인스턴스를 열면 (`/instance/{id}`) 라이브 상태, 로그 그리고 — 백테스트의 경우 — equity 곡선을 표시합니다. **브라우저 탭 제목**은 특정 인스턴스를 반영합니다 (**cBot 이름 · 종류 · 심볼**, 예: `TrendBot · Backtest · EURUSD`). 라이브 실행 탭과 백테스트 탭을 한눈에 구별할 수 있습니다. 동일한 cBot의 실행과 백테스트는 **계보**로 추적됩니다 (상태 전환 전체에서 유지되는 안정적인 계보 id). 페이지는 정확히 하나의 인스턴스를 따라가며 실행의 데이터와 백테스트의 데이터를 혼합하지 않습니다.

## 인스턴스 라이프사이클 제어

각 인스턴스 행 (및 그 상세 페이지)에는 상태 관련 제어가 있습니다. **활성** 인스턴스는 **Stop**을 표시합니다. **터미널** 인스턴스 (Stopped / Completed / Failed)는 **Start (▶)**를 표시하여 동일한 cBot, 계정, 심볼, 시간 프레임, 매개변수 집합 및 이미지로 다시 시작합니다 (실행은 실행으로 다시 시작하고, 백테스트는 백테스트로). Stop을 클릭하면 "Stopping…" 알림이 표시되고 해결될 때까지 아이콘이 비활성화되며, 새로 생성된 실행이 목록에 즉시 나타납니다 — 페이지 새로고침 없음.

콘솔 로그는 **인스턴스가 종료될 때 지속됩니다** — 실행 (Stop 시) 및 **백테스트** (완료 시) 모두 — 따라서 마지막 실행의 로그는 상세 페이지에서 보이며, 로그 도구 모음을 통해 **클립보드에 복사됩니다** (로그 복사 아이콘) 또는 **다운로드됩니다** (로그 다운로드 아이콘). 컨테이너가 사라진 후에도 마찬가지입니다. 둘 다 화면 상의 꼬리만이 아닌 인스턴스의 전체 콘솔 로그에 작용합니다.

업로드된 `.algo`는 여기서 빌드되지 않았으므로 **Last Build** 열은 cBot 페이지에서 공백으로 남겨집니다 (브라우저에서 빌드한 cBot에만 빌드 시간을 표시합니다).

## 중지된 인스턴스 편집 & 재실행

**중지된** 인스턴스 (실행 또는 백테스트)는 **Edit** 제어가 있습니다 — 목록의 행 위에 있는 아이콘 **그리고** 상세 페이지에서 Start/Stop 옆에 있는 아이콘 — 현재 설정으로 **사전 채워진** 대화 상자를 엽니다. **거래 계정, 심볼, 시간 프레임, 매개변수 집합 및 이미지 태그**를 변경할 수 있습니다 (그리고 백테스트의 경우 **윈도우 및 위의 모든 백테스트 설정**). 그런 다음 **Save & start** 새 설정으로 다시 시작합니다 (중지된 인스턴스를 대체). 인스턴스가 활성화되어 있는 동안 제어는 **비활성화됩니다** — 중지된 인스턴스만 편집할 수 있습니다.

## 코드 에디터에서 실행

코드 에디터에서 **Run**을 클릭하면 블라인드 하드 코드된 실행을 실행하는 대신 대화 상자를 엽니다:

- **거래 계정** (필수) — cBot이 연결하는 cTrader 계정.
- **Parameter set** (선택 사항) — 기존 집합을 선택하거나 비워두면 cBot의 **기본 매개변수 값**으로 실행합니다. 셀렉터 옆에 있는 **+** 버튼은 새 매개변수 집합을 인라인으로 생성하고 (아래 참조) 선택합니다.
- **Symbol / Timeframe** 기본값은 `EURUSD` / `h1`이며 변경할 수 있습니다. **Cancel** 또는 **Run**.

**Run** 시 에디터는 현재 소스를 저장 + 빌드하고, 선택한 계정에서 선택한 매개변수로 인스턴스를 시작한 다음, 라이브 컨테이너 로그를 테일합니다. (로그 스트림은 로그인한 사용자의 인증 쿠키를 `/hubs/logs` SignalR 허브로 전달하므로 연결되며 `Invalid negotiation response received`로 실패하지 않습니다.)

## 매개변수 집합

**Parameter set**은 각 매개변수 이름을 스칼라 값에 매핑하는 플랫 JSON 객체로 저장된 명명되고 재사용 가능한 cBot 매개변수 재정의 집합입니다. 예: `{"Period": 14, "Label": "trend"}`. 실행/백테스트 시점에 cTrader `params.cbotset` 파일로 변환됩니다 (`{ "Parameters": { … } }`). cBot의 **Parameter sets** 대화 상자에서 세트를 원시 JSON으로 생성/편집하거나 Run 대화 상자에서 인라인으로 생성할 수 있습니다.

모든 매개변수 집합은 **cBot에 속합니다**: New Parameter Set 대화 상자는 모든 cBot을 나열하고 **하나를 선택해야 합니다** — cBot이 선택될 때까지 생성이 차단됩니다. 세트의 **이름은 cBot마다 고유합니다**: 세트를 동일한 cBot의 다른 세트가 이미 사용하는 이름으로 생성하거나 이름을 바꾸면 거부됩니다 (대화 상자에서 명확한 오류, API에서 `409 Conflict`). 동일한 이름이 **다른** cBot에서 재사용될 수 있습니다.

JSON은 **저장 시 검증됩니다**: 값이 모두 스칼라인 단일 플랫 객체여야 합니다 (문자열 / 숫자 / 부울). 0이 아닌 객체 루트, 배열, 중첩된 객체, `null` 값 또는 잘못된 형식의 JSON은 거부됩니다 (대화 상자에서 명확한 오류, API에서 `400 Bad Request`). 빈 객체 `{}`는 허용되며 "재정의 없음"을 의미합니다.

## cTrader Console CLI 참고 사항

백테스트는 `--data-mode` (기본값 `m1`)를 필요로 하고, 날짜를 `dd/MM/yyyy HH:mm`로, `params.cbotset` JSON 위치 인수를 필요로 합니다. `run`은 `--data-dir`을 거부합니다 (백테스트만 해당). `ContainerCommandHelpers`를 참조하세요.

## 노드 & 규모

노드 에이전트를 추가하여 실행 용량을 확장합니다 (자체 등록 + 하트비트). [노드 발견](../operations/node-discovery.md) 및 [확장](../deployment/scaling.md)을 참조하세요.

## 거래 계정이 필요합니다

cBot을 실행하거나 백테스트하려면 연결할 cTrader 거래 계정이 필요합니다. **Trading accounts** 아래에 계정을 추가할 때까지 **Run New cBot** / **Backtest New cBot** 버튼은 비활성화되어 있습니다 (도구 팁 포함). 계정 설정 링크를 표시하는 프롬프트 — 더 이상 원시 `stream connect failed` 오류를 받지 않습니다.
