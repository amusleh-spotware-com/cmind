# 0005 — عميل AI يستخدم HTTP خام، وليس Anthropic SDK

## السياق

كل ميزة AI (توليد الإستراتيجية، الإصلاح الذاتي، حماية المخاطر، التحليل اللاحق) تستدعي API Anthropic. تبعية SDK تضيف سطح عابر لا نتحكم فيه، تربط وتيرة إصدارنا بأوقاتهم، وتخفي عقد السلك الدقيق الذي نحتاج إلى التفكير فيه للمرونة والتكلفة.

## القرار

`IAiClient` يستدعي Anthropic عبر **HTTP خام** من خلال **typed `HttpClient`** - بشكل متعمد **لا** SDK. `AiFeatureService` هو المنسق الوحيد المشترك بين نقاط نهاية الويب وأدوات MCP و`AiRiskGuard`. السطح الكامل **مبواب على `AppOptions.Ai.ApiKey`**: بدون مفتاح، كل ميزة تعيد `AiResult.Fail` والتطبيق يعمل دون تغيير.

## النتائج

- لا مفتاح مطلوب للبناء أو الاختبار أو E2E - CI والتطوير المحلي يقوم بتشغيل التطبيق الكامل بدون AI.
- نحن نملك شكل الطلب/الاستجابة، سياسة إعادة المحاولة/المهلة الزمنية، ومحاسبة الرمز بصراحة.
- ميزات Anthropic الجديدة يجب أن تكون مسلكة يدويًا؛ نحن نتاجر الراحة للتحكم والسطح تبعية أصغر. انظر مرجع `claude-api` للمعرفات النموذجية الحالية والمعاملات.
