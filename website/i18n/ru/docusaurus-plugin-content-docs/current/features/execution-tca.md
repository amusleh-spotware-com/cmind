# Transaction Cost Analysis (TCA)

Execution alpha крошечная за сделку и огромна за тысячи — это большая часть того, как банки и проп-дески сохраняют своё преимущество. TCA измеряет, насколько далеко цена, которую вы реально получили, отклонилась от цены, когда вы *решили* торговать.

Открыть **cBots → Execution Cost** (`/quant/tca`).

## Что измеряет

По **цене прибытия (решения)**, **стороне** и **исполнениям** (цена × объём), сообщает:

- **Средняя цена исполнения (VWAP)** — объёмно-взвешенная цена, которую вы реально получили.
- **Проскальзывание (bps)** — дрейф от прибытия до VWAP в базисных пунктах, **со знаком**, чтобы положительное число было **стоимостью** (покупка выше прибытия или продажа ниже), а отрицательное — улучшением цены.
- **Implementation shortfall** — эта стоимость, выраженная в терминах цена × объём: деньги, которые дрейф стоил вам на этом ордере.

```http
POST /api/quant/tca
{ "arrivalPrice": 1.1000, "side": "Buy",
  "fills": [ { "price": 1.1010, "quantity": 100 }, { "price": 1.1020, "quantity": 100 } ] }
```

## Умное разбиение (Almgren-Chriss)

Помимо измерения стоимости, cMind может спланировать большой ордер для *минимизации* стоимости. **cBots → Execution Schedule** (`/quant/execution`) строит **оптимальный график исполнения Almgren-Chriss**: по общему количеству, числу частей, вашему неприятию риска, волатильности и временному рыночному воздействию, возвращает размер для торговли в каждом срезе. Более высокое неприятие риска **фронтирует** график (снижая временной риск); нулевое неприятие риска выравнивает до ровного **TWAP**. Срезы всегда суммируются в общее количество.

```http
POST /api/quant/execution-schedule
{ "totalQuantity": 100, "slices": 5, "riskAversion": 2, "volatility": 0.02, "temporaryImpact": 0.1 }
```

## Почему это надёжно

Чистый, детерминированный доменный код (`Core.Execution`) без инфраструктурных зависимостей и внешних вызовов — модульные тесты покрывают знак стоимости buy/sell, улучшение цены, нулевое проскальзывание, агрегацию VWAP и валидацию входных данных. Это измерительная половина качества исполнения; это тот же показатель shortfall, который использует copy-движок для оценки (и, с умным разбиением, снижения) стоимости зеркальных ордеров.
