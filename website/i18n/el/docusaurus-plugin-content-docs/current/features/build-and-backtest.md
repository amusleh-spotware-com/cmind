# Δημιουργία & backtest cBots

Δημιουργία, εκτέλεση, backtest cTrader cBots (C# **και** Python, και τα δύο .NET) από το Monaco
IDE στο πρόγραμμα περιήγησης, εκτέλεση στην επίσημη εικόνα `ghcr.io/spotware/ctrader-console`.

## Δημιουργία

- Η σελίδα **Builder** φιλοξενεί τον επεξεργαστή Monaco· `CBotBuilder` μεταγλωττίζει το έργο με
  `dotnet build` **σε εφήμερο δοχείο** (`AppOptions.BuildImage`, κατάλογος εργασίας bind-mount
  στο `/work`), ώστε τα αναξιόπιστα στοχεία MSBuild του χρήστη να μην φτάνουν το host. Η ανάκτηση NuGet
  αποθηκεύεται σε κρυφή μνήμη μεταξύ των κατασκευών μέσω κοινού όγκου. Ο ιστός host χρειάζεται
  πρόσβαση στο socket του Docker.
- Τα πρότυπα εκκίνησης C# + Python βρίσκονται στο `src/Nodes/Builder/Templates/`.

## Εκτέλεση & backtest

- **Instances** = Ιεραρχία κατάστασης TPH (`Run`/`Backtest` × `Pending`/`Scheduled`/`Starting`/
  `Running`/`Stopping`/`Stopped`/`Failed`). Η μετάβαση αντικαθιστά την οντότητα (αλλαγή id),
  το id του δοχείου μεταφέρεται.
- `NodeScheduler` επιλέγει τον λιγότερο φορτωμένο ελάχιστα κατάλληλο κόμβο· `ContainerDispatcherFactory` δρομολογεί
  στον απομακρυσμένο κόμβο HTTP agent ή στον τοπικό Docker dispatcher.
- Οι pollers ολοκλήρωσης συμφωνούν τα εξαχθέντα δοχεία (τα δοχεία backtest αυτο-εξέρχονται μέσω
  `--exit-on-stop`)· η αναφορά που υπάρχει → ολοκληρώθηκε (αποθήκευση `ReportJson`), που λείπει → απέτυχε.
- Τα ζωντανά αρχεία καταγραφής δοχείου ρέουν στο πρόγραμμα περιήγησης μέσω SignalR· οι καμπύλες
  equity του backtest αναλύονται από την αναφορά + σχεδιάζονται.

## Τα δεδομένα αγοράς backtest αποθηκεύονται σε κρυφή μνήμη ανά λογαριασμό

Το cTrader Console κατεβάζει ιστορικά δεδομένα tick/bar στον κατάλογο `--data-dir`. Αυτός ο κατάλογος είναι
**σταθερή, μόνιμη κρυφή μνήμη με κλειδί στον λογαριασμό συναλλαγών** (τον αριθμό λογαριασμού του) — bind-mounted από
το δίσκο του κόμβου στη δική του διαδρομή δοχείου (`/mnt/data`), ένα **отдельное, non-nested mount** από το
κατάλογο εργασίας ανά instance. Έτσι κάθε backtest στον ίδιο λογαριασμό **επανα-χρησιμοποιεί** τα
ήδη λήφθεντα δεδομένα αντί να τα κατεβάσει ξανά. (Προηγουμένως ο κατάλογος
δεδομένων βρισκόταν κάτω από το κατάλογο εργασίας ανά instance, του οποίου το id αλλάζει κάθε εκτέλεση, το
οποίο ανάγκαζε μια φρέσκα λήψη κάθε backtest.) Ο εφήμερος κατάλογος εργασίας ανά instance εξακολουθεί να ισχύει
για το algo, τις παραμέτρους, τον κωδικό πρόσβασης και την αναφορά· η κοινή κρυφή μνήμη δεδομένων υπολογίζεται
στη χρήση δεδομένων backtest ενός κόμβου και καθαρίζεται από την ενέργεια καθαρισμού κόμβου.

## Ρυθμίσεις Backtest

Το διάλογο **Backtest** εκθέτει κάθε ρύθμιση που δέχεται το CLI backtest του cTrader Console, ώστε ποτέ να μη χρειάζεται να
αγγίξετε μια γραμμή εντολών:

- **From / To** — το παράθυρο backtest (`--start` / `--end`).
- **Data mode** — `m1` (μπάρες 1 λεπτού) ή `tick` (`--data-mode`).
- **Starting balance** — προεπιλέγεται σε `10000` (`--balance`). Ένα **0 balance δεν πραγματοποιεί συναλλαγές και κάνει
  το cTrader να εκδίδει μια κενή αναφορά που μετά καταρρέει** ("Message expected"), έτσι ένα μη-μηδενικό υπόλοιπο
  αποστέλλεται πάντα.
- **Commission** και **Spread** (`--commission` / `--spread`, spread σε pips).
- **Advanced options** — ένα ελεύθερο `name=value` ανά γραμμή κουτί για οποιαδήποτε άλλη ρύθμιση backtest που
  υποστηρίζει το cTrader (π.χ. `applyCommissionAutomatically=true`)· κάθε γραμμή γίνεται ένα όρισμα CLI `--name value`.

## Σελίδα λεπτομερειών Instance

Το άνοιγμα μιας instance (`/instance/{id}`) δείχνει την ζωντανή της κατάσταση, τα αρχεία καταγραφής και —
για ένα backtest — την καμπύλη equity. Ο **τίτλος της καρτέλας του προγράμματος περιήγησης** αντανακλά την
συγκεκριμένη instance (**όνομα cBot · είδος · σύμβολο**, π.χ. `TrendBot · Backtest · EURUSD`) ώστε μια
καρτέλα ζωντανής εκτέλεσης και μια καρτέλα backtest να είναι διακρίσιμες με μια ματιά. Μια εκτέλεση και
ένα backtest του ίδιου cBot ανιχνεύονται ως ξεχωριστές **γραμμές** (ένα σταθερό id γραμμής που μεταφέρεται
μεταξύ μεταβάσεων κατάστασης), έτσι η σελίδα ακολουθεί ακριβώς μια instance και ποτέ δεν μιγματίζει τα
δεδομένα μιας εκτέλεσης με μιας backtest.

## Ελέγχους κύκλου ζωής Instance

Κάθε σειρά instance (και σελίδα λεπτομερειών της) έχει διακόπτες που σχετίζονται με την κατάσταση. Μια **ενεργή**
instance εμφανίζει **Stop**· μια **τερματική** (Stopped / Completed / Failed) εμφανίζει **Start (▶)** για
να την επανεκκινήσει με το ίδιο cBot, λογαριασμό, σύμβολο, χρονοδιάγραμμα, σύνολο παραμέτρων και εικόνα (μια
εκτέλεση επανεκκινείται ως εκτέλεση, ένα backtest ως backtest). Το κλικ στο Stop εμφανίζει ένα σημείωμα
"Stopping…" και απενεργοποιεί το εικονίδιο έως ότου ολοκληρωθεί, και μια νέα δημιουργηθείσα εκτέλεση εμφανίζεται
στη λίστα αμέσως — χωρίς ανανέωση σελίδας.

Τα αρχεία καταγραφής κονσόλας **διατηρούνται όταν μια instance τερματίζεται** — για μια εκτέλεση (στο Stop) και
για ένα **backtest** (κατά την ολοκλήρωση) ομοίως — έτσι τα αρχεία καταγραφής της τελευταίας εκτέλεσης
παραμένουν προβάσιμα στη σελίδα λεπτομερειών και, μέσω της γραμμής εργαλείων καταγραφής, **αντιγράφονται στο
πρόχειρο** (Copy logs icon) ή **λήφθηκαν** (Download logs icon) ακόμη και μετά την αφαίρεση του δοχείου. Και οι δύο
ενεργούν στο πλήρες αρχείο καταγραφής κονσόλας της instance, όχι μόνο την εμφανιζόμενη ουρά.

Ένα **μεταφορτωθέν** `.algo` δεν χτίστηκε ποτέ εδώ, έτσι η στήλη **Last Build** του στη σελίδα cBots αφήνεται
κενή (δείχνει ώρα κατασκευής μόνο για cBots που κατασκευάζετε στο πρόγραμμα περιήγησης).

## Επεξεργασία & επανεκτέλεση μιας σταματημένης instance

Μια **σταματημένη** instance (εκτέλεση ή backtest) έχει έναν έλεγχο **Edit** — ένα εικονίδιο στη σειρά της στη
λίστα **και** δίπλα στο Start/Stop στη σελίδα λεπτομερειών της — που ανοίγει ένα διάλογο **προ-συμπληρωμένο** με
την τρέχουσα ρύθμιση της. Μπορείτε να αλλάξετε το **λογαριασμό συναλλαγών, το σύμβολο, το χρονοδιάγραμμα, το σύνολο
παραμέτρων και την ετικέτα εικόνας** (και, για ένα backtest, το **παράθυρο και όλες τις ρυθμίσεις backtest**
παραπάνω), στη συνέχεια **Save & start** επανεκκινήσει την με τις νέες ρυθμίσεις (αντικαθιστώντας την σταματημένη
instance). Ο έλεγχος είναι **απενεργοποιημένος ενώ η instance είναι ενεργή** — μόνο μια σταματημένη instance
μπορεί να επεξεργαστεί.

## Εκτέλεση από τον επεξεργαστή κώδικα

Το κλικ **Run** στον επεξεργαστή κώδικα ανοίγει ένα διάλογο αντί να ενεργοποιήσει μια τυφλή, σταθερή εκτέλεση:

- **Trading account** (απαιτείται) — ο λογαριασμός cTrader που συνδέεται το cBot.
- **Parameter set** (προαιρετικό) — επιλέξτε ένα υπάρχον σύνολο ή αφήστε το κενό για εκτέλεση με τις
  **προεπιλεγμένες τιμές παραμέτρων** του cBot. Ένα κουμπί **+** δίπλα στο επιλογέας δημιουργεί ένα νέο σύνολο
  παραμέτρων εσωτερικά (δείτε παρακάτω) και το επιλέγει.
- **Symbol / Timeframe** προεπιλέγονται σε `EURUSD` / `h1` και μπορούν να αλλάξουν· **Cancel** ή **Run**.

Κατά το **Run** ο επεξεργαστής αποθηκεύει + δημιουργεί την τρέχουσα πηγή, ξεκινά την instance στον επιλεγμένο
λογαριασμό με τις επιλεγμένες παραμέτρους, στη συνέχεια ακολουθεί τα ζωντανά αρχεία καταγραφής δοχείου. (Η ροή
αρχείου καταγραφής προωθεί το cookie ελέγχου ταυτότητας του συνδεδεμένου χρήστη στον `/hubs/logs` SignalR hub,
ώστε να συνδέεται αντί να αποτυγχάνει με `Invalid negotiation response received`.)

## Σύνολα παραμέτρων

Ένα **parameter set** είναι ένα ονοματισμένο, επαναχρησιμοποιήσιμο σύνολο παραμέτρων cBot που αποθηκεύονται ως
ένα επίπεδο αντικείμενο JSON αντιστοιχίζοντας κάθε όνομα παραμέτρου σε μια κλιμακωτή τιμή, π.χ.
`{"Period": 14, "Label": "trend"}`. Κατά την εκτέλεση/backtest μετατρέπεται στο αρχείο cTrader `params.cbotset`
(`{ "Parameters": { … } }`). Μπορείτε να δημιουργήσετε/επεξεργαστείτε ένα σύνολο ως ακατέργαστο JSON από
το διάλογο **Parameter sets** του cBot ή εσωτερικά από το διάλογο Run.

Κάθε σύνολο παραμέτρων **ανήκει σε ένα cBot**: το διάλογο New Parameter Set παραθέτει όλα τα cBots σας και
**πρέπει να επιλέξετε ένα** — η δημιουργία αποκλείεται έως ότου να επιλεγεί ένα cBot. Το **όνομα ενός συνόλου
είναι μοναδικό ανά cBot**: η δημιουργία ή μετονομασία ενός συνόλου σε όνομα που ένα άλλο σύνολο του ίδιου cBot
χρησιμοποιεί ήδη απορρίπτεται (ένα σαφές σφάλμα στο διάλογο, `409 Conflict` στο API). Το ίδιο όνομα μπορεί
να επανα-χρησιμοποιηθεί σε ένα **διαφορετικό** cBot.

Το JSON **επικυρώνεται** κατά την αποθήκευση: πρέπει να είναι ένα μόνο επίπεδο αντικείμενο του οποίου οι τιμές
είναι όλες κλιμακωτές (string / number / bool). Μια μη-αντικείμενη ρίζα, ένας πίνακας, ένα φωλιασμένο αντικείμενο,
μια `null` τιμή ή κακής μορφής JSON απορρίπτεται (ένα σαφές σφάλμα στο διάλογο, `400 Bad Request` στο API). Ένα
κενό αντικείμενο `{}` επιτρέπεται και σημαίνει "χωρίς παρακάμψεις".

## Σημειώσεις CLI cTrader Console

Τα Backtests χρειάζονται `--data-mode` (προεπιλέγεται `m1`), ημερομηνίες ως `dd/MM/yyyy HH:mm`, και
όρισμα θέσης `params.cbotset` JSON· το `run` απορρίπτει το `--data-dir` (backtest-only). Δείτε
`ContainerCommandHelpers`.

## Κόμβοι & κλίμακα

Η ικανότητα εκτέλεσης κλιμακώνεται με την προσθήκη κόμβων agent (αυτο-εγγραφή + ηχοσβήνευση). Δείτε
[node discovery](../operations/node-discovery.md) και [scaling](../deployment/scaling.md).

## Ένας λογαριασμός συναλλαγών είναι απαιτούμενος

Η εκτέλεση ή δοκιμή ενός cBot χρειάζεται έναν λογαριασμό συναλλαγών cTrader για σύνδεση. Μέχρι να προσθέσετε
ένα κάτω από τους **Trading accounts**, τα κουμπιά **Run New cBot** / **Backtest New cBot** είναι
απενεργοποιημένα (με μια συμβουλή) και η σελίδα εμφανίζει μια γρήγορη σύνδεση προς τη ρύθμιση λογαριασμού —
δεν θα χτυπήσετε πλέον ένα ακατέργαστο σφάλμα `stream connect failed` από ένα bot χωρίς λογαριασμό.
