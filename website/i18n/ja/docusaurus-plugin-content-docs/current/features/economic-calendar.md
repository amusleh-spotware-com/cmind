# 経済カレンダー

cMindは**独自の**経済カレンダーを出荷 — リリーススケジュール、実績値、予測値、改訂値とデータ駆動インパクトモデル — **一次権限**（中央銀行と国家統計機関）から調達、**ゼロ依存**ForexFactory、FXStreet、Investing.comまたは任意のアグリゲータ。これはポイントインタイム正確、≥10年の履歴を保持、取引、パブリックAPI、MCP、cBot、AI、アラートおよびバックテストにワイアード。これはデカップルモジュール：ゼロ効果でトレーディングコアに無効化できます。

> **ステータス。** ドメインコア（インパクトモデル、国→シンボルマッピング、ニュースウィンドウポリシー、ポイントインタイム改訂チェーン、2層ゲーティング）**および**永続（`calendar` Postgresスキーマ、アペンドのみ読み取り/書き込み側、FREDコネクタおよび設定ゲート摂取ワーカー）は実装および テスト（ユニット + Testcontainers統合）。JWT REST API、MCPツール、UIは以下に説明された後続ロールアウトフェーズで土地。

## 何が異なるのか

主流カレンダーに対する繰り返しの苦情は私たちの設計制約になった：

- **サイレントインパクト評価変更なし。** 私たちのインパクト評価は**決定論的、バージョン付き、監査可能**。すべての変更は記録改訂タイムスタンプ — サイレント上書きがない。ユーザーは正確に**なぜ**イベントがHighなのかを見ることができます。
- **イベントごとの1つのUTCアンカー。** すべてのイベントは一次ソースの公式スケジュール（ソースの独自タイムゾーンは保存される、ユーザーごとレンダリング明示的なIANAタイムゾーンを使用するDSTはゾーンデータベースで処理 — 決してマニュアル±1hトグル）から単一UTC瞬間にアンカー。
- **完全改訂チェーンすべての場所。** 元の値と各改訂は一級、同一にAPI、MCP、cBotサーフェス全体で公開。
- **≥10年の履歴、壁なし。** 制限なし閲覧範囲；60日キャップなし、登録ゲートなし。
- **ポイントインタイム構造で。** すべての事実は`KnownAt`（いつ*私たち*それを学んだ）および`EffectiveAt`（イベント瞬間）を運ぶ。「時間Tのカレンダーが見た」は一級クエリ、バックテストニュースルールはライブと全く同じ振る舞う — 改訂値を使用からの先読みなし。

## インパクトモデル

インパクトスコアは純粋で決定論的な機能[0, 100]、Low / Medium / High / Criticalにバンド。その入力はスコアリング時にのみ知られたデータ（未来リークなし）：

- **シリーズ先行** — インジケータクラスごとのベースライン重み（レート決定はCPIを上回り、マイナー調査を上回る）。
- **実現ボラティリティフットプリント** — 一次影響シンボルの中央絶対リターンこのシリーズの*過去*リリース後のウィンドウ：「このリリースは履歴的に価格をこれだけ動かす」。
- **サプライズ感度** — どの程度強く絶対サプライズ（z-スコア）は履歴的にリリース後の移動と相関。

スコアはこれらを固定の重みで混合、スタンプ`ImpactModelVersion`。再計算は明示的ロギングされた操作新しい改訂を生成 — ミューテートなし — スコアは常に入力から再現可能。

## 国→通貨→シンボルマッピング

最もシタルゴアルゴ統合ペーパーカットは、純粋な機能として一度に解決：国はその通貨にマップ（すべてのユーロ地域メンバーファン IN EUR）、通貨は監視リストシンボルマップするどちらかの脚にそれを引用。**EURUSDは両方EUおよびUSイベント影響を受ける**；XAUUSDはUSD公開；US500はUSDにマップ。これはニュースフィルタ、影響シンボル解決およびブラックアウト数学を駆動。

## ニュースウィンドウポリシー

`NewsWindowRule`は`{ minImpact, beforeMinutes, afterMinutes, currencies?, series? }`。単一、共有、純粋実装は「インスタント T はシンボル S のブラックアウト内か？」に答える — cBotニュースフィルタ、コピー取引一時停止およびAIリスクガードによって使用、それらは決して分岐できない。不確実性の場合、ブラックアウト答えはコンフィグ保守値デフォルト（デフォルトで失敗クローズ）データギャップはサイレント決して高インパクト明細を通じた取引をグリーンライト。

## ポイントインタイム & 改訂

実績値、予測値およびインパクトスコアは**アペンドのみ**。各イベントは改訂の順序チェーンを所有、単調`KnownAt`：

- `Scheduled` — イベントは最初スケジュール（前インパクト、実績なし）。
- `Released` — 最初の印刷実績到着。
- `Revised` — 後の改訂値到着。
- `Rescheduled` — ソースはリリース瞬間を移動（監査可能、アラート可能）。
- `Rescored` — インパクトスコアは新しいモデルバージョンの下で再計算。

過去瞬間として`as of`クエリはその時点で知られた正確な改訂を返す — バックテストニューズルールの先読みを殺す保証。

## 予測/コンセンサス

経済学者のサーベイ中央値は**自由に**一次ソースによって公開されない — アグリゲータの専有付加価値、私たちはそれを作成しません。イベントスキーマは無効の`Forecast`を運ぶ；デプロイメントはライセンスコンセンサスフィードを通じてオプション`IForecastProvider`ポート（あなた独自のキー、デフォルト無効）をワイアできます。前の値と改訂は常に公式ソースから来る。

## データソース

2層デカップル、すべての一次 — 決してアグリゲータ：

- **スケジュール/タイミング：** FREDリリースカレンダー；国家統計機関（BLS、BEA、Census、Eurostat、ONS、Destatis、INSEE、e-Stat、ABS、StatCan）；中央銀行会議カレンダー（Fed、ECB、BoE、BoJ、RBA、BoC、SNB、RBNZ）。
- **実績値：** FRED（改訂用ビンテージ日付とポイントインタイム）、プラスBLS、BEA、Census、ECB SDW、Eurostat およびOECD SDMX API。

死んだソースは**そのソースのみ**カバレッジに低下；カレンダーは他すべてを提供し続け、ギャップを新鮮度メトリクスとして浮上。

## レート制限と バックアップ計画

外部プロバイダーは公開レート制限（FREDは許可～120要求/分）。カレンダーはそれが**決してプロバイダーの制限をトリップしません**、そのスロットルまたはカットオフが決して読み取りを低下させないように構築：

- **プロアクティブスロットル。** 各ソースのHTTPクライアントはスレッドセーフレートゲートを通じて、コンフィグ予算に対するアウトバウンドリクエストを間隔（`App:Calendar:FredRequestsPerMinute`、デフォルト100 — 意図的にプロバイダー天井の下）。リクエストはキューイング、ペース付き、決してバースト。
- **`429 Retry-After`を尊重。** プロバイダーが`429 Too Many Requests`を返す場合、ゲートは全体ソースをバックオフサーバーリクエスト冷却（または`App:Calendar:RateLimitBackoff`、デフォルト60s）次の呼び出し前 — タイトな再試行ループなし。
- **標準耐久性。** 各ソースクライアントはまたアプリ全体耐久性ハンドラを継承（再試行バックオフ+ジッター、回路ブレーカー、タイムアウト）、一時的なブリップは吸収され、永続的に失敗するソースはパーク（カバレッジは古くなる）他に影響なく。
- **バックアップ計画 — 耐久読み取りスルーキャッシュ。** 読み取りは**決して**プロバイダーを呼び出すことで提供；範囲を取得すると、それはPostgresにアペンドのみで永続化、その後永遠に提供（§「オンデマンド負荷」参照）。したがってソースが rate-limited またはダウン時でも、カレンダーはキャッシュ、ポイントインタイム正確なデータから答えを保つ；欠落スパンは単にカバーされたままで、次の摂取サイクルで再試行。ブラックアウト答えはさらに不確実性の下でデフォルト保守に失敗するため、データギャップは決してリリースを通じた取引をグリーンライト。
- **安い投票。** 条件付きフェッチ（ETag / If-Modified-Since / ソースビンテージカーソル）および「スパン一度フェッチ、再びなし」キャッシュは実際のリクエストボリュームを保つはるか下任意の制限を正常運用 — レートゲートはセーフティネット、共通パスではなく。

## 有効化/無効化

2つの独立層、正確に他cMind機能：

- **層1 — ランタイム機能トグル**（`Feature.EconomicCalendar`）機能管理UIからフリップ；再デプロイなし、ライブで有効。
- **層2 — ホワイトラベルハードゲート**（`App:Branding:EnableEconomicCalendar`、デフォルト`true`）。リセラーはそれを`false`に設定して機能を完全に削除；オペレータはその後それを再有効化できません。

有効状態は`Branding.EnableEconomicCalendar && FeatureToggle.EconomicCalendar`。無効化時、ナブエントリは隠され、`/economic-calendar`、`/api/calendar/**`およびMCPカレンダーツールはクリーン機能無効`404`を返す — `500`ではなく。永続履歴はランタイムトグルオフで保持されるため、再有効化は即座。

- **P0 — ドメインコア***(実装)*：アグリゲート、値オブジェクト、ポート、インパクトモデル、国→シンボルマッピング、ニュースウィンドウポリシー、2層ゲーティング、完全ユニットスイート。
- **P1 — 永続 + 1つソース***(実装)*：EF`calendar`スキーマ（独自テーブル、アペンドのみ、ホットインデックス）、読み取りスルー`IEconomicCalendar`リーダーポイントインタイム`asOf`、べき等アペンドのみ書き込みサービス、耐久性タイプクライアントの後ろのFREDコネクタ、設定ゲート摂取ワーカー；Testcontainers統合テスト（永続、PIT、べき等性、ブラックアウト）。
- **P2 — パブリックJWT REST API + Web UI***(実装)*：バージョン付き、JWT保護`/api/calendar/v1` API — クライアント発行、トークン交換、コア読み取りエンドポイント（イベント、履歴、シリーズ、サプライズ、次、ブラックアウト、影響シンボル、ヘルス）スコープ実行および2層ゲーティング、統合テスト。プラス携帯優先**`/economic-calendar` ページ** — ゲート付き、完全ローカライズ（23言語）アップカミングリリースのアジェンダ、携帯フレンドリーカード色バンドインパクトチップおよびMudBlazor**フィルタダイアログ**（通貨+最小インパクト+**日付から**ピッカーをジャンプする**任意**過去日付全履歴を通じて — 60日キャップなし、壁なし）；ナブエントリ、スモーク/携帯/a11y/E2Eテスト。**シリーズ履歴ページあたり**（`/economic-calendar/series/{code}`、各イベントからリンク）シリーズの完全な印刷履歴をリスト。サプライズチャート+無限スクロールブラウザは従う。
