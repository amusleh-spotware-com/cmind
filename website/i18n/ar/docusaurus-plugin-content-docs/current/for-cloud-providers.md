# cMind لمزودي السحابة و VPS

أنت بالفعل استأجر الحوسبة. cMind هو منتج جاهز مفتوح المصدر يمكنك تغليف هذه الحوسبة حوله: **توفير استضافة cMind المُدارة** والهبوط على حمل عمل قيمة عالية لزج حساب الجوع - متاجري الخوارزمية وسطاء وشركات prop ومجتمعات التداول الذين يريدون منصة تعمل دون أن تصبح فريق ops بأنفسهم.

:::tip TL;DR
تشغيل الطبقة بدون حالة + Postgres + أسطول العقد؛ يد العملاء URL مصنف. تحقيق اشتراك والحوسبة والعلامة البيضاء والذكاء الاصطناعي. → [النشر إلى السحابة](./deployment/cloud.md)
:::

## لماذا تقدم cMind المُدارة

- **لا تكلفة البناء.** إنه مفتوح المصدر و MIT-مرخص وموثق واختبر وحاوية بالفعل. أنت تحزم وتشغل - أنت لا تبنيه.
- **منتج متمايز لمكان مربح.** التداول الخوارزمي حساب الجوع: backtests والعقد الحية حرق CPU، وهي *استخدام فواتير* كنت بالفعل بيع.
- **عملاء لزج.** المتاجرون الذين يبنون ويشغلون استراتيجيات داخل المنصة لا تحقن عرضياً.
- **يتحول تحفظ إلى إضافة.** cMind مستضافة ذاتياً بالتصميم - للعملاء الذين "لا تريد أن تكون فريق ops"، *أنت* الجواب.

## من يشتري cMind المُدار منك

- **الكميين الفردين والمتاجرين** الذين يريدونها مستضافة. → [لمتاجرين](./for-traders.md)
- **وسطاء cTrader** تشغيل white-label لعملائهم. → [للوسطاء](./for-brokers.md)
- **Prop firms ونسخ مشاريع التداول** الذين يحتاجون إلى البنية التحتية الموجودة والقابلة للتدقيق.

## ماذا يعني "cMind المُدار" للتشغيل

تشغيل ثلاث طبقات؛ يحصل العميل على URL ويب مصنف:

| الطبقة | ما هي | حيث يعمل |
|---|---|---|
| بدون حالة (الويب + MCP) | التطبيق + API + خادم MCP | أي منصة حاوية، autoscaled |
| قاعدة البيانات | PostgreSQL | Postgres المُدار (RDS / خادم مرن / خاصتك) |
| أسطول العقد | بناء و تشغيل حاويات cTrader | **VMs أو Kubernetes - يحتاج Docker الامتيازات** |

:::warning شيء واحد للنطاق مقدماً
وكلاء العقد تبني وتشغيل حاويات cTrader، لذا يحتاجون **Docker الامتيازات**. هذا يستبعد runtimes حاوية بدون خادم (Azure Container Apps، AWS Fargate) *للوكلاء* - تشغيل تلك على [Kubernetes](./deployment/kubernetes.md)، أو VM، أو EC2. الطبقة بدون حالة تعمل في أي مكان.
:::

أدلة النشر الحقيقية والنسخ واللصق تجعل هذا ملموس: [نظرة عامة على السحابة](./deployment/cloud.md) · [Azure](./deployment/cloud-azure.md) · [AWS](./deployment/cloud-aws.md) · [Kubernetes](./deployment/kubernetes.md) · [التوسع](./deployment/scaling.md).

## كيف تحقيق ذلك

- **اشتراك استضافة مُدار.** خطط شهري Starter / Team / Business الحجم من خلال أسطول العقدة والتزامن الاختبار.
- **الاستخدام والحوسبة المقاسة.** فاتورة ساعات اختبار وساعات عقدة حية والتخزين - طبيعيًا تُقاس عن طريق أسطول الحاوية التي تعمل بالفعل.
- **طبقات مُعيد البيع white-label.** رسوم أكثر للعلامة التجارية الكاملة (الشعار والألوان و PWA و `ShowSiteLink=false`) وللتمكين من قدرات premium عبر [مفاتيح الميزات](./features/feature-toggles.md). → [White-label](./features/white-label.md)
- **الذكاء الاصطناعي المُدار.** استشرق مفتاح موفر AI الافتراضي حتى تحصل أموال العميل على AI مع لا إعداد وعلامة الاستخدام - أو تقديم إحضار الخاص بك. → [ميزة AI](./features/ai.md)
- **Prop-firm ونسخ التداول حصة الإيرادات.** شركات مضيفة تشغيل تحديات والرسوم الأداء واتخاذ منصة قطع. → [Prop-firm](./features/prop-firm.md) · [رسوم الأداء](./features/copy-performance-fees.md) · [سوق الموفر](./features/copy-provider-marketplace.md)
- **الإعداد والتثبيت والدعم.** إرفاق الخدمات المهنية ودعم premium.

## أنماط متعددة المستأجرين

- **نشر لكل عميل (موصى به).** مثيل مصنف واحد لكل عميل - عزلة قوية، علامة تجارية وقاعدة بيانات كل عميل، رمز انضمام عقدة مميز لكل عميل. يتم قراءة العلامة التجارية من `IOptionsMonitor`، لذا يحمل كل مثيل هويته الخاصة.
  → [علامات تجارية متعددة المستأجرين](./white-label-for-business.md#multi-tenant-per-customer-branding) · [اكتشاف العقدة](./operations/node-discovery.md)
- **لوحة التحكم المشتركة (متقدم).** قيادة العديد من الحالات من طبقة الإنشاء الخاصة بك، وبذر العلامة التجارية والميزات لكل عميل برمجياً.

## قياس الاستخدام للفواتير

نقطة نهاية **`GET /api/usage`** الخاص بالمالك/الإدارة فقط تعيد ملخص قراءة فقط موفر يمكن أن استطلاع وفاتورة على - دون أي مجال جديد أو إصرار، يتوقع الحالة الموجودة:

```json
{
  "users": { "total": 42 },
  "nodes": { "total": 6, "online": 5 },
  "instances": { "total": 1280, "backtestsRunning": 3, "runsRunning": 11 },
  "cbots": { "total": 210 },
  "tradingAccounts": { "total": 88 }
}
```

استطلاع لكل نشر عميل لقيادة مقاعد- أساسي أو قائم على الأسطول أو عبء العمل تسعير. الزوج مع [التسجيل والقابلية للمراقبة](./operations/logging.md) لقياس الحوسبة أدق.

## الخطوات الأولى

1. قف على نشر مرجعي من [أدلة السحابة](./deployment/cloud.md).
2. قالب لكل عميل (العلامة التجارية + رمز الانضمام + DB) وسلك الفاتورة الخاصة بك إلى استخدام الحوسبة.
3. قائمة - لديك الآن منصة تداول algo مُدارة لبيع.
