# 0001 — 厳密 DDD と純粋 `Core`

## コンテキスト

このアプリ 移動 実マネー。ビジネスルール 散乱 エンドポイント、背景サービス、Razor コンポーネント 腐る テスト不可能、矛盾 動作 — 正確 バグ コスト ユーザー 資本。

## 決定

ドメインロジック ライブ **集約、値オブジェクト、ドメインサービス上** `src/Core`、どのコンパイル **ゼロインフラ依存** (EF なし、HttpClient、Docker、ASP.NET なし)。エンドポイント、MCP ツール、コンポーネント、`BackgroundService`s **ベル** — 決してしない。ルール:

- 公開セッター なし; 状態変更 意図明確 方法 ガード 不変式。
- 集約 参照 互いに 強ID により、 ナビゲーションプロパティ ではなく。
- 1 個 `SaveChanges` ミューテート **1 個** 集約; クロス集約フロー ドメインイベント 使用。
- プリミティブ クロッシング ドメイン境界 値オブジェクト ラップ。
- 不変式違反 スロー Core `DomainException`、フレームワーク例外 ではなく。

## 結果

- ドメイン ルール ユニットテスト可能 データベース または ウェブホスト なし。
- `Core` 純粋 機械強制 `ArchitectureGuardTests` + ビルド失敗 もし 壊れた。
- セレモニーが増加 (値オブジェクト、強ID、ドメインイベント) 貧弱 モデルより — これ 意図的 コスト マネー移動 ルール 正確 保つ 1 場所。
