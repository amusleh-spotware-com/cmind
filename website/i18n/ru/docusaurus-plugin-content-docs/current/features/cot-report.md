# Commitment of Traders (COT)

cMind поставляется со встроенным отчетом **Commitment of Traders** — еженедельным разбором CFTC того, кто находится
в длинной и короткой позиции на рынке фьючерсов США (коммерческие хеджеры, крупные спекулянты, фонды), с интерактивными
историческими графиками, нормализованным **индексом COT**, аутентифицированным REST API для ботов cBot и инструментами MCP
для клиентов ИИ. Данные поступают прямиком из **общедоступных наборов данных CFTC Socrata** — никакого ключа API, никакого
агрегатора. Как и экономический календарь, это независимый модуль, который можно отключить без каких-либо последствий для
ядра торговли.

## Что он дает

- **Все три семейства отчетов, только фьючерсы и фьючерсы + опционы вместе:**
  - **Legacy** — Non-Commercial (крупные спекулянты), Commercial (хеджеры), Non-Reportable.
  - **Disaggregated** — Producer/Merchant, Swap Dealers, Managed Money, Other Reportables.
  - **Traders in Financial Futures (TFF)** — Dealer, Asset Manager, Leveraged Funds, Other Reportables.
- **Тщательно отобранный каталог рынков** — валютные пары, золото/серебро/медь, сырая нефть и природный газ, ценные бумаги,
  индексы акций, крипто и основные зерновые/мягкие товары — каждый сопоставлен со своим стабильным кодом контракта CFTC и,
  где однозначно, с торгуемым символом (например, Euro FX → `EURUSD`, Gold → `XAUUSD`).
- **Индекс COT (0–100)** — показывает, где текущая чистая позиция спекулянта находится в пределах своего исторического диапазона
  (по умолчанию ~3-летний lookback). Показания вблизи крайностей сигнализируют о переполненном позиционировании, которое часто
  предшествует развороту; отчет помечает **длинную крайность** (≥80) или **короткую крайность** (≤20).
- **Точность момента времени.** Еженедельный отчет измеряется во вторник, но становится общедоступным только в пятницу; каждое
  чтение соблюдает момент выпуска, поэтому сигнал позиционирования при бэктесте никогда не видит отчет до его публикации
  (без опережения).

## Использование страницы

Откройте **Commitment of Traders** из левой навигации. Выберите **рынок**, **тип отчета** (Legacy /
Disaggregated / Financial) и переключите **Фьючерсы + опционы**, чтобы переключаться между только фьючерсами и
комбинированным вариантом. Страница показывает:

- **Чистое позиционирование во времени** — интерактивный линейный график чистой позиции (длинная − короткая)
  каждой категории трейдера в окне истории.
- **Индекс COT** — линейный график индекса 0–100 с последним показанием и его меткой крайности.
- **Последний снимок** — таблица длинной / короткой / чистой / % открытого интереса для каждой категории трейдера,
  плюс общий открытый интерес и дата отчета.

Каждый график имеет кнопки панели инструментов **масштабирования внутрь / наружу** (и сброса), и вы можете перетаскивать по оси времени для масштабирования. **Экспорт CSV** загружает полную еженедельную историю выбранного рынка и типа отчета как файл, готовый для электронной таблицы. Используйте **Сравнить рынки**, чтобы наложить несколько рынков на одну диаграмму — сравнительные диаграммы отображают чистую позицию спекулянтов каждого выбранного рынка и индекс COT рядом, чтобы вы могли читать кроссрыночное позиционирование с первого взгляда.

## Как данные течют

База данных является кэшем. Еженедельный рабочий процесс приема извлекает шесть наборов данных CFTC для отслеживаемых рынков, выполняет upsert каталога рынков и добавляет каждый новый отчет **идемпотентно** (повторное выполнение никогда не дублирует снимок). Кроме того, данные **загружаются по требованию**: в первый раз, когда запрашивается рынок, он извлекается из источника CFTC и сохраняется, а каждый последующий запрос обслуживается прямо из базы данных. Кэш **обновляется по мере выпуска новых еженедельных отчетов** — как только самый новый сохраненный отчет старше недели, следующий запрос прозрачно извлекает и добавляет последние данные (отрегулировано так, чтобы источник никогда не был перегружен). Первая загрузка заполняет несколько лет истории; отказ источника деградирует до обслуживания лучших кэшированных данных. Все работает из коробки без ключа; опциональный токен приложения Socrata только повышает ограничение скорости.

## Конфигурация

Все ключи находятся в `App:Cot` (см. [переключатели функций](./feature-toggles.md) и
[параметры владельца white-label](./white-label-owner-settings.md)):

| Ключ | По умолчанию | Назначение |
|-----|---------|---------|
| `IngestionEnabled` | `true` | Работает ли еженедельный рабочий процесс ingestion. |
| `PollInterval` | `6h` | Как часто рабочий процесс опрашивает наборы данных CFTC. |
| `BackfillYears` | `5` | Лет истории, извлекаемых при первом запуске. |
| `ReconcileLookbackWeeks` | `4` | Недавние недели пересинхронизируются каждый цикл, чтобы поймать ревизии. |
| `SocrataAppToken` | — | Опциональный токен, повышающий анонимный предел скорости. |
| `CotIndexLookbackWeeks` | `156` | Еженедельные отчеты, используемые как диапазон индекса COT (~3 года). |

## Gating

Видимость — это двухуровневый шлюз, идентичный экономическому календарю: жесткий шлюз white-label
`App:Branding:EnableCot` (уровень сборки) **и** переключатель функции времени выполнения `App:Features:Cot`. Если
один из них отключен, ссылка навигации, страница, REST API и инструменты MCP все исчезают (API возвращает `404`).
Поскольку источник данных не требует ключа, нет шлюза ключа источника данных — включено означает видимо.

## Для разработчиков

- Домен: `Core.Cot` — агрегаты `CotMarket` и `CotReport`, объект значения `CotPositions`, сервис домена
  `CotIndexCalculator` и порты `ICotReports` / `ICotSource`.
- Инфраструктура: `Infrastructure.Cot` — парсер против коррупции `CftcSocrataSource`, шлюз скорости, сервис
  только для добавления записей, сторона чтения и еженедельный рабочий процесс ingestion (схема EF `cot`).
- Доступ cBot & ИИ: [COT cBot API](./cot-cbot-api.md) (REST, `market:read` JWT) и инструменты MCP
  `CotMarkets`, `CotLatest`, `CotHistory`, `CotHealth`.
