# 전략 건전성 및 알파 감소

모든 에지는 감소합니다 — 연구는 양적 전략의 반감기가 수년간에서 수개월로 축소되었다고 명확히 합니다, 그래서 *적응이 발견을 능가합니다*. 전략 건전성 모니터는 전략 자체의 수익 이력에서 에지가 여전히 있는지 알려줍니다.

**cBots → Strategy Health** (`/quant/health`)를 엽니다.

## 무엇을 하는가

수익률 시리즈(또는 Equity 곡선, 가장 오래된 먼저)를 주면:

- 이력을 **이전** 및 **최근** 절반으로 분할하고 해당 Sharpe 비율을 비교합니다;
- 평균이 가장 분명하게 이동한 관찰(체제 돌파구)을 찾는 **CUSUM 변경점** 스캔을 실행하고, 편향이 통계적으로 주목할 만한 경우에만 보고합니다;
- 판정을 반환합니다:

| 판정 | 의미 |
|---|---|
| **Healthy** | 최근 성과가 이전 기록과 일치하거나 더 좋습니다. |
| **Degrading** | 최근 Sharpe가 이전 기록보다 현저히 약합니다 — 면밀히 관찰하세요. |
| **Decayed** | 최근 창에서 에지가 사실상 사라졌습니다 — 일시 중지를 고려하세요. |
| **Unknown** | 판단할 이력이 충분하지 않습니다. |

```http
POST /api/quant/health
{ "returns": [...] }   // 또는 { "equity": [...] }
```

## 왜 신뢰할 수 있는가

인프라 의존성 및 외부 호출이 없는 순수 결정론적 도메인 코드 (`Core.Health`)입니다 —衰廃, degrading, healthy 및 너무 짧은 경우와 변경점 지역화에 대해 단위 테스트되었습니다. 이것은 자율 에이전트를 지원하는 항상 켜진 건강 검사의 수동 동반자입니다: 동일한 통계가 실행 중인 전략의 에지가 사라지는 경우 위험을 줄이는 서킷 브레이커를駆動합니다.
