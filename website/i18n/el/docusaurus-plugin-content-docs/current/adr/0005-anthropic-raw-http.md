# 0005 — Ο πελάτης AI χρησιμοποιεί ακατέργαστο HTTP, όχι το SDK της Anthropic

## Πλαίσιο

Κάθε δυνατότητα AI (δημιουργία στρατηγικής, αυτο-επιδιόρθωση, φράχτης κινδύνου, νεκροσκοπίες) καλεί το API της Anthropic. Μια εξάρτηση SDK προσθέτει μια μεταβατική επιφάνεια που δεν ελέγχουμε, συνδέει το καδράρισμα της έκδοσης μας με το δικό τους, και κρύβει το ακριβές σύμβολο σύμβασης που χρειαζόμαστε να σκεφτούμε για ανθεκτικότητα και κόστος.

## Απόφαση

Ο `IAiClient` καλεί Anthropic σε **ακατέργαστο HTTP** μέσω ενός τυποποιημένου `HttpClient` — σκόπιμα **όχι** το
SDK. Ο `AiFeatureService` είναι ο ενιαίος ορχηστράρης που μοιράζονται Web endpoints, τα MCP `AiTools`, και
`AiRiskGuard`. Ολόκληρη η επιφάνεια είναι **περιορισμένη στο `AppOptions.Ai.ApiKey`**: χωρίς κλειδί, κάθε δυνατότητα
επιστρέφει `AiResult.Fail` και η εφαρμογή εκτελείται αμετάβλητη.

## Συνέπειες

- Δεν απαιτείται κλειδί για κατασκευή, δοκιμή ή E2E — CI και τοπική ανάπτυξη εκτελούν την πλήρη εφαρμογή χωρίς AI.
- Κατέχουμε το σχήμα αίτησης/απόκρισης, την πολιτική επανάληψης/timeout και τη λογιστική token ρητά.
- Οι νέες δυνατότητες Anthropic πρέπει να συνδεθούν χειροκίνητα. ανταλλάσσουμε ευκολία για έλεγχο και μικρότερη
  επιφάνεια εξάρτησης. Δείτε την αναφορά `claude-api` για τρέχοντα αναγνωριστικά μοντέλου και παραμέτρους.
