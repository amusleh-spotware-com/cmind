# Функции ИИ

Слой ИИ cMind **не зависит от провайдера**. Каждая функция общается с одним нейтральным к поставщику швом (`IAiClient.CompleteAsync`); **маршрутизирующий клиент** разрешает активное учетное данные поставщика и отправляет на соответствующий адаптер проводки. Вы выбираете поставщика + модель + конечную точку (и, если поставщику это нужно, ключ); каждая существующая функция работает без изменений с той же защитой, шифрованием, устойчивостью и деградацией.

**Батареи включены:** встроенная локальная LLM поставляется с приложением и включена по умолчанию (Microsoft.ML.OnnxRuntimeGenAI, например Phi-3-mini) — поэтому каждое развертывание имеет работающий ИИ **без ключа API и без внешнего сервиса**. Развертывание белого ярлыка может удалить его и ограничить, какие поставщики пользователи могут добавить. За пределами встроенного подключитесь к любому внешнему поставщику.

Поддерживаемые провайдеры:

- **Встроенный локальный ИИ** (`BuiltInOnnx`) — модель ONNX GenAI в процессе, нет ключа, поставляется + включена по умолчанию.
- **Anthropic** (Claude — Messages API)
- **OpenAI** и **Azure OpenAI** (Chat Completions)
- **Google Gemini** (`generateContent`)
- **Любая конечная точка, совместимая с OpenAI**, включая **локальные модели** (Ollama, LM Studio, vLLM, llama.cpp `server`, LocalAI) и облака, совместимые с OpenAI (OpenRouter, Groq, Together, Mistral, DeepSeek) — всё через один адаптер, совместимый с OpenAI, отличающийся только базовым URL + моделью + ключом.

Ровно **один** поставщик активен одновременно. Учетные данные хранятся **зашифрованными** (`AiProviderCredential` агрегат + `IAiProviderStore` + `ISecretProtector`, `EncryptionPurposes.AiApiKey`); локальная конечная точка **не нуждается в ключе**. С **отсутствующим** активным поставщиком, каждая функция возвращает отключённый результат и остаток приложения работает без изменений (ключ не требуется для сборки, тестирования или запуска платформы).

**Обратная совместимость:** существующее развертывание наследует старый `App:Ai:ApiKey` (или старый зашифрованный параметр `ai.api_key`) автоматически как активный поставщик **Anthropic** по умолчанию — ноль действий не требуется.

ИИ не сконфигурирован → страницы ИИ тускнеют действия и показывают баннер плюс одноразовое приглашение добавить поставщика в **Settings → AI** (`AiFeatureNotice`). Статус на `GET /api/ai/status` (`{ enabled, kind, model }`); поставщики управляются (только владельцу) через `GET/PUT /api/ai/providers`, `POST /api/ai/providers/{id}/activate`, `DELETE /api/ai/providers/{id}` и пинг проверки соединения `POST /api/ai/providers/test`.

## Развертывание по умолчанию vs собственный поставщик пользователя

Учетные данные ИИ имеют два масштаба:

- **Развертывание по умолчанию (управляемое владельцем).** Владелец настраивает поставщика (или поставляет один через `App:Ai:Providers[]` / старый `App:Ai:ApiKey`). Это становится **общим по умолчанию для каждого пользователя** — поэтому брокер или хостинг-провайдер может финансировать ИИ для всех своих пользователей **без настройки для каждого пользователя и без лимита для каждого пользователя**. Управляется через маршруты владельца только `/api/ai/providers` выше.
- **Собственный поставщик пользователя (самообслуживание).** Любой подписанный пользователь может добавить своего поставщика в `GET/PUT /api/ai/my-providers`, `POST /api/ai/my-providers/{id}/activate`, `DELETE /api/ai/my-providers/{id}`. При наличии, их **собственный активный поставщик переопределяет развертывание по умолчанию для их собственных функций ИИ**; удаление вернётся к умолчанию.

**Порядок разрешения** (в `AiProviderStore`, за пользователя запроса): собственное активное учетное данные пользователя → развертывание по умолчанию → старый ключ конфигурации → нет (ИИ отключен). Ровно одно учетное данные активно **за масштаб** (частичный уникальный индекс на `OwnerUserId`), и каждый масштаб разрешается независимо, поэтому пользователь, активирующий собственный ключ, никогда не нарушает общее по умолчанию. Фоновые/не-Web контексты (нет пользователя запроса) всегда разрешают развертывание по умолчанию.

## Матрица возможностей поставщика

Возможности по умолчанию на поставщика и переопределяемы владельцем. Когда возможность отключена, функция **деградирует, никогда не выбрасывает**: веб-поиск молча опускается; видение возвращает типизированный отказ неподдерживаемой возможности.

| Поставщик | Тип | URL базы по умолчанию | Ключ требуется | Веб-поиск | Видение | Примечания |
|---|---|---|---|---|---|---|
| Встроенный локальный ИИ | `BuiltInOnnx` | n/a (в процессе) | нет | ✖ | ✖ | поставляемая модель ONNX GenAI, по умолчанию включена |
| Anthropic | `Anthropic` | `https://api.anthropic.com/` | да | ✅ | ✅ | Messages API, инструмент `web_search` |
| OpenAI | `OpenAiCompatible` | `https://api.openai.com/v1/` | да | opt-in | opt-in | Chat Completions |
| Azure OpenAI | `AzureOpenAi` | `https://<resource>.openai.azure.com/` | да | ✅ | ✅ | путь развертывания + `api-version` |
| Google Gemini | `Gemini` | `https://generativelanguage.googleapis.com/` | да | ✅ | ✅ | `generateContent`, обоснование `google_search` |
| Ollama (локальный) | `OpenAiCompatible` | `http://localhost:11434/v1/` | нет | ✖ | зависит от модели | через адаптер совместимый с OpenAI |
| LM Studio (локальный) | `OpenAiCompatible` | `http://localhost:1234/v1/` | нет | зависит от модели | зависит от модели | через адаптер совместимый с OpenAI |
| vLLM / llama.cpp / LocalAI | `OpenAiCompatible` | ваш обслуживаемый URL | нет | ✖ | зависит от модели | через адаптер совместимый с OpenAI |
| OpenRouter / Groq / Together / Mistral / DeepSeek | `OpenAiCompatible` | URL поставщика | да | ✖ | зависит от модели | через адаптер совместимый с OpenAI |

Полные руководства по настройке для каждого поставщика (ключи, URL, id моделей, шаги пользовательского интерфейса): смотрите [Поставщики ИИ — каталог настройки](../deployment/ai-providers.md).
