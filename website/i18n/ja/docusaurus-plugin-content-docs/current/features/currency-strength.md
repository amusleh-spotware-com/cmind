# AIマクロ通貨強度・フォワード展望

cMindは**AI支援付き、数学的に決定論的**なマクロ通貨強度エンジンを提供します。
8主要通貨プラス新興国・エキゾチック通貨まで設定可能な宇宙を**現在**の基礎的強度でランキングし、
選択した期間（1M / 3M / 6M / 12M）にわたるすべてのペアの**フォワード方向性展望**を予測します。
すべてのランキング、すべてのペアバイアス、すべての数値はドメインコアの純粋な決定論的数学で計算されます;
LLMはデータがpublishできないフォワードルッキングな入力を*収集*し、結果をplain Englishで*説明*するだけです。
ランキング、方向、または数値をLLMが発明することは決してありません。

> **正直な制限。** ファンダメンタルは中长期の価値を予測 хорошо、中短期の価値を予測するのは難しいです。
> ポジショニング/コンフルエンスフィルタとして扱い、**短期タイミングシグナルとしては扱わないでください。
> 高インパクトリリース（NFP/CPI/中銀）近くの讀みはノイズが多くなります。
> 金融アドバイスではありません。

## 動作原理

1. **現在のファンダメンタルはLLMではなく経済カレンダーから取得。** ハード numbers — 政策金利、CPI vs 目標、
   GDP、雇用、貿易収支 — とその**サプライズzスコア**は、[経済カレンダー](./economic-calendar.md)モジュール
   （FRED/BLS/BEA/ECB和中銀スケジュール）から**ポイントインタイム**で取得。
   履歴スナップショットはルック-aheadをリークしません。
2. **LLMはカレンダーがpublishできないもののみを収集** — 通貨ごと：フォワード Trajectory
   （期待される政策金利パス（bp）、インフレーション動向vs目標、成長モメンタム）と**地政学的展望**
   （リスクオン/オフ、関税、財政/債務、選挙）、PLUS カレンダーが欠いているEM/エキゾチック現在の数字。
   厳密なJSON、階層対応バリデーション、ウェブ検索オン。
3. **ドメインがランキングとフォワードマトリックスを決定論的に計算。** 各ドライバーは**階層内zスコア**
   でスコアリング（そうでない場合、50%インフレのエキゾチックが主要通貨を歪めることがない）、
   winsorized、加重合計されて複合スコアになり、
   最も強い→最も弱いと安定ISO tie-breakでランキング。
   フォワードレイヤーは各複合体をそのTrajectoryに沿って伝播 —
   `projected = current + horizonScale · Σ trajectoryDriver·weight` —
   各ペアの予測差分を**方向性バイアス**（▲ app房 / ▬中立 / ▼減価）にマッピングし、
   convictionとともに提示。
4. **LLMが**ランキングとトップペア呼叫をplain languageで説明。

## ドライバー

| ドライバー | 強度への影響 | 備考 |
|---|---|---|
| 政策金利とTrajectory | 高い/タカ派 ⇒ 強い | 最も重い权重; 中銀の乖離が最も大きなギャップを駆動。 |
| インフレーション（CPI vs 目標） | 目標以上 ⇒ 弱い | スコアリング逆（Purchasing power drag）。 |
| GDP成長 | 相対成長率高 ⇒ 強い | パネルとの差分。 |
| 雇用 | 労働が強 ⇒ 強い | 政策パスを供給。 |
| 貿易収支 / 経常収支 | 黒字 ⇒ 強い | 構造的需要。 |
| 政策姿勢 | タカ派 ⇒ 強い | 主な長期ドライバー。 |
| サプライズモメンタム | 最近のbeats ⇒ 強い | カレンダーのサプライズzスコアから。 |
| 地政学的/リスク | リスクオフ ⇒ 安全通貨（USD/JPY/CHF）が強い | バウンドされたフォワードリスクデルタ。 |
| 実質利回り/キャリ（PEM/エキゾチック） | 実質金利プラス ⇒ 強い | 穏やかな体制での支配的なEMドライバー。 |
| 外部脆弱性（PEM/エキゾチック） | 赤字/準備金不足/USD建て債務 ⇒ 弱い | 構造的減価压力。 |
| 交易条件（PEM/エキゾチック） | 輸出価格上昇 ⇒ 強い | BRL、ZAR、CLP、NOK、AUD、CAD。 |
| 政治/制度的リスク（PEM/エキゾチック） | 不安定 ⇒ 弱い | より広い dead-band、capped conviction。 |

## 階層付きユニバース（主要 + EM + エキゾチック）

ユニバースは**デプロイメント構成可能**（`App:CurrencyStrength:Universe`） —
通貨追加はコードではなく設定。 各通貨には**階層**（`Major` / `EmergingMarket` / `Exotic`）carry、
weighting、dead-band幅、convictionキャップを調整：

- **主要通貨** — USD, EUR, GBP, JPY, AUD, NZD, CAD, CHF（金利レベル主導）。
- **新興国市場** — CNH, INR, BRL, MXN, ZAR, KRW, SGD, PLN (+ Scandi NOK/SEK);
  キャリ + リスク + 外部脆弱性の权重アップ、medium confidence。
- **エキゾチック** — TRY, HUF, CZK、plus USDペッグHKD/SAR; 低confidence、広い dead-band、capped
  conviction。**ペッグ/ヘビー管理通貨**（HKD, SAR, CNH）はフラグ付けられ、そのTrajectoryは
  ダウンウェイトされ、そのペア展望は`Neutral`に向かってクランプされるため、ペッグが自由浮动シグナルとして読まれることはない。

公式EM/エキゾチック統計は低頻度、改訂族、そして時に不透明なため、AIが収集した数字には**階層ごとのconfidence**が信頼性バッジとして表示されます。

## グレースフル低下

| カレンダー | AI | 結果 |
|---|---|---|
| ✅ | ✅ | 完全ランキング + フォワード予測 + ナラティブ（`CalendarAndAi`）。 |
| ✅ | ❌ | カレンダーからの現在ランキングのみ、フォワード予測なし（`CalendarOnly`）。 |
| ❌ | ✅ | AI収集現在の数字 + フォワード、低confidence（`AiOnly`）。 |
| ❌ | ❌ | スナップショットなし — ウィジェットは非表示、ページは空状態を表示。 |

どちら的路でもappは変更なしで実行されます。 AIはAIキーでゲート; カレンダー肢は自身のホワイトラベルゲート + ランタイムトグルを尊重。

## 使用方法

- **AIを有効化**（Settings → AI）+ 自身のダッシュボードの**Customize**ダイアログから
  「Currency strength」ウィジェットをオン（オプトイン、デフォルトで非表示）。
  ウィジェットはトップの強/弱通貨とトップ3Mペア呼叫を表示; 完全ページにリンク。
- **完全ページ** — `/ai/currency-strength`：期間セレクター（1M/3M/6M/12M）、
  階層フィルター（All/Majors/EM/Exotics）、現在のランキング、フォワード予測、
  ペア展望マトリックス（バイアス + conviction、ペッグ/低confidenceフラグ付き）、
  AIナラティブ。**今すぐ更新**（所有者）を押して再生成。
  バックグラウンドワーカー（`App:CurrencyStrength:RefreshEnabled`、**デフォルト`true`**）がスケジュールで
  更新するため、ページは箱から出してすぐにデータが埋まります; デプロイまたは所有者がオフにします
  （またはAI / 経済カレンダー機能をオフにすると、リ fresherはナ snapshotなしに低下することを尊重）。

## プログラム的アクセス

1つの共有読み取りモデル（`ICurrencyStrengthQuery`）に3つの方法でアクセス可能：

- **アプリ内AI** — AI機能に直接インジェクト（インプロセス）。
- **MCP** — `currency_strength`ツール（パラメータ`horizon`、`tier`）AIクライアント/エージェント向け。
- **cBot REST** — `GET /api/market/v1/currency-strength/{latest,history,pair/{base}/{quote}}`、
  [カレンダーcBot API](./calendar-cbot-api.mdと同じ`CalendarJwt` machinerでセキュリティ保護 +
  追加 **`market:read`**スコープ付き。 cBotは`market:read`でAPIクライアントを登録、
  id + secretを`POST /api/calendar/v1/token`で短期JWTと交換し、`Bearer`トークンでエンドポイントを呼び出す。
  2番目のJWTスキーム、2番目のシークレットなし — 漏洩したトークンは読み取り専用、market-scoped、
  短期、取消可能。

トークンフローとコピペサンプルについては[カレンダーcBot API](./calendar-cbot-api.md)を参照。
