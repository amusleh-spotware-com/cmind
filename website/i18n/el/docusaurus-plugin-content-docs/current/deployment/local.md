# Εκτέλεση cMind τοπικά 🖥️

Αυτός είναι ο ταχύτερος τρόπος για να δείτε cMind για πραγματική — μια πλήρης παρουσία στο δικό σας μηχάνημα. Πάρτε έναν καφέ.
πιθανώς θα έχετε συνδεθεί πριν να κρυώσει.

:::tip Τι θα έχετε στο τέλος
Μια εκτέλεση web app στο **localhost:8080**, ένας MCP server στο **localhost:8081**, μια βάση δεδομένων Postgres,
και έναν τοπικό κόμβο εργαζόμενου έτοιμο να κτίσει και backtest cBots. Όλα στο μηχάνημά σας, όλα δικά σας.
:::

**Πριν να ξεκινήσετε, χρειάζεστε ένα από:**

- **Απλώς Docker** → χρησιμοποιήστε Option A (δεν απαιτείται .NET SDK). Συνιστάται για μια πρώτη ματιά.
- **.NET 10 SDK + Docker** → χρησιμοποιήστε Option B εάν θέλετε να hackάρετε τον κώδικα.

Και οι δύο διαδρομές είναι διασταυρούμενης πλατφόρμας (Windows / macOS / Linux).

## Option A — Docker Compose (δεν απαιτείται .NET SDK)

Προαπαίτητο: Docker Desktop (ή Docker Engine + compose plugin).

```bash
cp .env.example .env        # επεξεργασία PG_PASSWORD, OWNER_EMAIL, OWNER_PASSWORD
docker compose up --build
```

- Web UI: <http://localhost:8080> (συνδεθείτε με ιδιοκτήτη από `.env`. αναγκάστηκε να αλλάξει κωδικό πρόσβασης κατά την πρώτη σύνδεση).
- MCP server: <http://localhost:8081/mcp>.
- Τα δεδομένα Postgres παραμένουν στο όγκο `pgdata`. το σχήμα μεταναστεύει αυτόματα κατά την εκκίνηση.

Το κοντέινερ web προσαρτά την υποδοχή Docker του κεντρικού υπολογιστή (`/var/run/docker.sock`) έτσι ο ενδο-περιηγητής κατασκευαστής και φυτεύτηκε **LocalNode** κτίσουν + τρέχουν κοντέινερ cTrader Console στο μηχάνημά σας.

**Σημειώσεις διασταυρούμενης πλατφόρμας**
- Docker Desktop (Windows/macOS) εκθέτει την υποδοχή στο `/var/run/docker.sock` — το προσάρτημα compose λειτουργεί ως έχει.
- Linux: βεβαιωθείτε ότι ο χρήστης σας μπορεί να έχει πρόσβαση στην υποδοχή, ή τρέχει σύνθεση με επαρκή προνόμια.
- Η εικόνα Web είναι `linux/amd64`. στο Apple Silicon Docker το τρέχει κάτω από προσομοίωση.

Διακοπή και σκούπισμα:

```bash
docker compose down          # κρατήστε δεδομένα
docker compose down -v       # επίσης διαγραφή του όγκου βάσης δεδομένων
```

## Option B — .NET Aspire (για ανάπτυξη)

Προαπαίτητο: .NET 10 SDK + Docker.

```bash
dotnet run --project src/AppHost
```

Aspire ορχηστρώνει Postgres, Web, MCP, pgAdmin. καλώδια connection strings + OTLP. ανοίγει dashboard. Ορίστε διαπιστευτήρια ιδιοκτήτη ως παραμέτρους Aspire (`OwnerEmail`, `OwnerPassword`).

Τρέχτε μόνο το web app ενάντια σε υπάρχουσα Postgres:

```bash
dotnet run --project src/Web
```

## Προσθήκη εργαζόμενων κόμβων τοπικά

Το σπόρο LocalNode ήδη τρέχει εργασία στο μηχάνημά σας. Για άσκηση **αυτο-ανακάλυψη** τοπικά, ξεκινήστε πράκτορα κόμβων που δείχνουν στο Web app (δείτε [ανακάλυψη κόμβων](../operations/node-discovery.md)) με `NodeAgent:MainUrl=http://host.docker.internal:8080` και αντιστοίχιση `JoinToken`.

## Αντιμετώπιση προβλημάτων 🔧

Ο Docker έχει απόψεις. Εδώ είναι οι συνήθεις ύποπτοι:

| Σύμπτωμα | Πιθανή αιτία & διόρθωση |
|---|---|
| `port is already allocated` στο 8080/8081 | Κάτι άλλο χρησιμοποιεί τη θύρα. Σταματήστε το, ή αλλάξτε το mapping στο `docker-compose.yml`. |
| Web ξεκινά αλλά κτίσματα/backtests αποτυγχάνουν | Η υποδοχή Docker δεν προσαρτάται ή δεν είναι προσβάσιμη. Σε Linux, βεβαιωθείτε ότι ο χρήστης σας μπορεί να φτάσει `/var/run/docker.sock`. |
| `permission denied` στην υποδοχή (Linux) | Προσθέστε το χρήστη σας στο `docker` group (`sudo usermod -aG docker $USER`) και ξανά-σύνδεση, ή τρέχει με επαρκή προνόμια. |
| Πολύ αργή πρώτη εκτέλεση | Η πρώτη κατασκευή τραβά εικόνες και μεταγλωττίζει — οι επόμενες εκτελέσεις είναι πολύ πιο γρήγορες. Στο Apple Silicon η `linux/amd64` web εικόνα τρέχει κάτω από προσομοίωση. |
| Δεν μπορείτε να συνδεθείτε | Ελέγξτε `OWNER_EMAIL` / `OWNER_PASSWORD` στο `.env` σας. Η πρώτη σύνδεση αναγκάζει αλλαγή κωδικού πρόσβασης. |
| Περίεργότητες βάσης δεδομένων μετά τις αναβαθμίσεις | `docker compose down -v` σκουπίζει τον όγκο για ένα καθαρό σχέδιο (θα χάσετε τοπικά δεδομένα). |
